# v0.28.0 — RunPod P2 adapter

The primary transport is REST via `httpx.AsyncClient`; the SDK is diagnostic
only. The adapter uses `POST /v1/pods`, `GET /v1/pods/{id}`, `DELETE
/v1/pods/{id}`, and `GET /v1/billing/pods`. It selects in order: RTX 2000 Ada,
RTX 4000 SFF Ada, RTX 4000 Ada, then L4. Blackwell identifiers remain blocked.

Managed pods require both name `vf-auto-<job_id>` and a non-secret owner marker.
The adapter never returns or reaps any other account resource. Pod status is
read afresh on every poll, including public IP and port mapping; an old SSH
endpoint is never cached as truth.

The executor consumes a persisted approval, binds its provider handle
atomically, records all provider mutations in Supabase, and always enters a
termination `finally`. Billing absence may be polled for up to 15 minutes; a
missing receipt fails P2 even after deletion.
