# v0.30.0 — Nebius adapter roadmap marker

The provider-neutral `Provisioner` interface, policy fuses, audit path and P-4
product boundary are ready for another adapter. `NebiusAdapter` is the next
planned provider implementation and must pass the same dry-run, gold,
termination and orphan tests as RunPod.

No Nebius API, SDK, authentication or provider call is implemented in this
wave. Adding it must not require changes above the adapter seam.
