# v0.29.0 — P-4 product wiring

The existing approval remains a durable statement of intent and cannot mutate
provider state. A separate Start Forge command consumes that approval only
after an explicit confirmation, validates the approved config, applies the
smaller config/system budget, resolves a credential for the duration of the
provider call, and records a pollable lifecycle.

`VF_AUTOPROVISION=false` is the safe default. Mock binding proves the complete
state path without cost. The single live P-4 smoke is create/observe/terminate
only, capped at `$1`; it does not rerun product training.

The live proof writes a secret-free local ledger and a delayed billing schedule.
Its synchronous termination gate requires accepted DELETE, target absence and a
fresh raw account inventory with zero `vf-auto-*` names. Billing reads at `+1h`
and `+6h` are supporting asynchronous evidence.
