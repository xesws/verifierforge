# v0.24.0 — Contracts, fuses, and mock lifecycle

P-1 exposes a strict async adapter protocol and a deterministic dry-run
implementation. The orchestrator owns lifecycle transitions and treats
provider state as evidence: after a handle exists, any failure first attempts
termination and records the outcome before reaching a terminal state.

Safety policy is fail closed. Per-job budget, maximum concurrency, maximum
runtime, global kill, orphan reconciliation, and append-only audit each have a
dedicated test. Missing approval/audit context rejects creation. A concrete GPU
model or secret-bearing environment key cannot cross `ProvisionSpec`.

The demo is zero-cost and leaves no active handle. The feature flag remains off
and no production route imports the package in this version.
