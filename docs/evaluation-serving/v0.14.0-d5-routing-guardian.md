# v0.14.0 — Online SQL guardian

The guardian is a sampling sidecar, not a gate on proxy forwarding. On a tuned
`data-pull-sql` request, it receives the original prompt and completion,
locates the matching frozen NL2SQL record, and evaluates it with the unchanged
`NL2SQLVerifier`. Scores are aggregated into timestamped `LivePassRate` points
for the real API.

The default sampling rate is 20% and is configurable for testing or later
operations. Unsupported clusters, unknown prompts, malformed completions, and
SQLite failures create no user-visible failure: the completion has already been
returned. The v0.14.0 acceptance uses only `fake-tuned` to prove plumbing; the
result is operational telemetry, not a held-out evaluation or a training gate.

In local fake-tuned acceptance, 113 sampled frozen SQL requests produced 113
rolling points, each with an exact-pass rate of 1.0. That number demonstrates
the plumbing only: fake-tuned derives deterministic SQL from the frozen fixture
and must not be interpreted as a deployed-model quality result.
