# v0.4.0 — Evaluation Serving: D2 Metric Bridge

## Scope

Map verl file-logger values to the existing append-only `MetricRecord` fields:
trainer step, mean reward, greedy validation accuracy as pass@1, and actor
entropy. The API continues to aggregate those lines into `Metrics` without a
database or contract change.

## Completion Evidence

The trainer writes a text manifest at `artifacts/final/model.txt` only after a
successful final step. It contains checkpoint/model metadata, not weights. A
small plot utility may place `artifacts/curve.png` beside the manifest for the
frontend teammate; both are safe for `vf watch`.

## Validation

Test metric normalization and deduplication on repeated logger reads, then verify
the local API returns the synced curve after the RunPod resume smoke.

## Acceptance Evidence — 2026-07-15

The completed `d2demo` run wrote 100 append-only MetricRecord lines. The
post-kill resume began producing new public entries at step 53 without
overwriting its preexisting rows. After `vf watch`, local `GET
/jobs/d2demo/metrics` returned four aligned 100-element arrays; the final
manifest and curve image were present without syncing weights.
