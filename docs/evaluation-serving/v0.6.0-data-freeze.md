# v0.6.0 — Gate A evaluator and freeze evidence

**Status:** in progress. Linked plan: `docs/versions/v0.6.x/v0.6.0-data-freeze.md`.

## Gate A

`scripts/gate_a.py` consumes candidate JSONL with a prompt and expected result,
uses the shared completion client, and scores `k` independently sampled answers
per prompt through the existing verifier. It reports exactly: baseline pass@1,
pass@k (called pass@8 for the default), and mixed fraction (a prompt's sample
group contains both at least one pass and one fail).

The gate exits nonzero when pass@1 is outside `[0.20, 0.60]` or mixed fraction
is below `0.30`. It never adjusts those limits. Its structured evidence records
input digest, verifier identity/digest, model/base-url identifier (not a key),
sample count, raw metric values, timestamp, and decision.

## Freeze boundary

Only a passing Gate A creates the frozen manifest and Git tag. The later GRPO
run reads the exact tagged data; its metric JSONL and curve remain run artifacts
under `runs/`, not source-controlled data.

## Validation

Tests exercise empty/invalid records, all-pass, all-fail, mixed groups, exact
threshold boundaries, and nonzero exit behavior. They use a fake completion
source and the real NL2SQL verifier path where practical.
