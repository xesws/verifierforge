# v0.7.1 — evaluation evidence routing

The subset diagnostic is immutable evidence, not a live evaluator result to be
reinterpreted in place. The v0.7.1 router reads its sample JSONL and produces a
separate atomic JSON artifact. It retains raw scorer totals while exposing the
runbook-specific operational category `format_parse_failure` for a completion
that is both code-fenced and rejected only by the legacy single-read-only
lexical gate.

No provider call occurs in routing. The artifact carries SHA-256 bindings to
both source JSONL and the completed Gate A evidence, ordered completion
examples, and the derived `D_format_parse_failure_fraction`. This makes the
Branch A/B decision reproducible without changing aggregate Gate A metrics.
