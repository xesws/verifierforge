# v0.11.0 — Evaluation: held-out selection and D4 after snapshot

The frozen 60-row held-out set is the sole D4 comparison instrument. Its
before snapshot is `pass@1=0.5833333333333334`,
`pass@8=0.7666666666666667`, and `mixed_fraction=0.4666666666666667`; the
50-row training-pool curve remains monitoring only.

After M3, each eligible 50-step main checkpoint is evaluated without entering
the training process. Evidence preserves the input/verifier/config/model
identity, per-sample completions and tier scores, metrics, and artifact hashes.
The retained checkpoint is the maximum held-out pass@1 (lower step wins ties).
The final report artifact displays before and after triplets beside the random
control curve, and labels any unavailable after-evaluation as unavailable
rather than a gain.

The evaluator stays pod-local/OpenAI-compatible for model-serving work; no
OpenRouter request is needed. An evaluation, evidence, or hash mismatch failure
stops report promotion but does not rewrite the immutable before snapshot.

The M1′ single-Blackwell hardware change affects only scheduling: the required
random-reward control runs after the main job. It does not change the held-out
identity, evaluator evidence contract, or before/after interpretation.
