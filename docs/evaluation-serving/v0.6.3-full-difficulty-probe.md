# v0.6.3 — full difficulty probe artifact

**Status:** in progress. Linked plan:
`docs/versions/v0.6.x/v0.6.3-full-difficulty-probe.md`.

`scripts/gate_a.py --reference --per-prompt-output <path>` writes an atomic,
newline-delimited record for each completed input group. Each row identifies the
source record and stores only `pass_count` and `k`; no completion text is kept.
The full 276-row run is descriptive, not a Gate A admission decision. Rows with
`1 <= pass_count <= 7` are the morning's candidate pool for a human-approved
reprojection; this document does not authorize that reprojection.

Terminal completion failures still invalidate the probe and write normal v2
failure evidence. A completed probe records its output descriptor in the
evidence file so the counts can be tied to the exact input and verifier.
