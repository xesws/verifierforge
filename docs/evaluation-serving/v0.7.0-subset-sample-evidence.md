# v0.7.0 — sample-level Gate A evidence

**Status:** implementation validated; diagnostic rerun pending. Linked plan:
`docs/versions/v0.7.x/v0.7.0-subset-sample-evidence.md`.

`gate_a` saves sample evidence automatically when its loaded input has exactly
50 records. `--save-samples` explicitly enables it for another reference run;
`--no-save-samples` is retained only for non-diagnostic large runs. The output
is an atomic JSONL artifact, never a best-effort append, so a completed
evidence file cannot point to a partial sample set.

Each sample reports the raw completion and actual stage facts: raw parser
success, safe-read-only acceptance, SQLite execution success, result match,
per-tier scores, any length penalty, and the unchanged final verifier score.
The taxonomy partitions non-full-passes into parse failure, execution error,
and executable-not-full-pass; a separately flagged length-penalized exact
result avoids disguising a formatting penalty as a wrong SQL result.

The diagnostic is run with `--reference`; its metrics do not admit or reject a
training dataset. It only supplies O1/O2's durable evidence and branch input.
