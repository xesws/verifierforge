# v0.6.2 — auditable Gate A execution

**Status:** in progress. Linked plan: `docs/versions/v0.6.x/v0.6.2-pod-eval-recovery.md`.

Gate A requires `--report` and writes schema-v2 evidence on success and failure.
Every report includes its resolved sanitized endpoint/model, input/verifier
hashes, mode, and sampling parameters. Failure reports include request ordinal,
record/sample, up to two attempts, chained exception facts, status, redacted
truncated provider body, terminal-failure distribution, and circuit state.

At most eight logical samples run concurrently. Each gets one retry. Ten
consecutive terminal failures in request-ordinal order stop new work. Any
terminal failure invalidates metrics and exits 2; it is never silently counted
as a verifier failure. Subset mode enforces the fixed admission thresholds;
`--reference` records full-set metrics without threshold-based exit failure.
