# v0.30.0 — Reviewer sandbox full mode

**Status:** implementation complete; public full-mode gate pending — 2026-07-19
**Owner/worktree:** Codex on `main`
**Release type:** minor reviewer-delivery capability

## Problem and scope

The existing one-click sandbox is a loopback artifact/fake fallback. Add an
explicit full mode that composes the real API/UI and proxy behind one local
entrypoint, uses Supabase repositories, routes tuned traffic to the configured
real public model endpoint, runs Forge Agent with deterministic mock binding,
and exposes Start Forge through the zero-cost mock provisioner. A Cloudflare
quick tunnel supplies the temporary public URL. HTTP Basic invitation auth
protects the composite surface; the invite code lives only in an ignored
runtime file or environment.

Full mode uses a hybrid API data mode: immutable committed job/report artifacts
remain reviewable while clusters, decisions, approvals, routes, guardian points
and P-4 execution state use the actual database. The fallback mode remains the
default and requires no secrets, database, endpoint or public listener.

## Non-goals

Do not enable paid LLMs, provision a GPU, modify trainer/frozen data, create a
named Cloudflare tunnel, claim a quick tunnel is an SLA, implement Nebius, or
enable `VF_AUTOPROVISION` outside the sandbox's explicit mock binding.

## Affected areas and contracts

- Composite reviewer ASGI app with `/proxy/v1/chat/completions` and invite auth.
- `scripts/start_reviewer_sandbox.sh --mode fallback|full` lifecycle, readiness,
  cloudflared URL capture and fail-closed configuration checks.
- Additive `hybrid` API data mode and regression coverage.
- JUDGES onboarding and README/design roadmap language.
- Nebius remains the next adapter; no provider implementation is added.

## Validation and stop conditions

Fallback must stay zero-cost and fully green. Full-mode tests use local fake
components only. The live launcher may read Supabase and the configured model
endpoint but sends no paid LLM request and performs no provider mutation.
Missing database/endpoint/invite configuration, failed upstream health, or an
unavailable tunnel is recorded without weakening authentication. Tag
`reviewer-sandbox-full` only after public authenticated health and product-page
readback succeed.
