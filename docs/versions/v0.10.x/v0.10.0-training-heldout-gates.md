# v0.10.0 — training/held-out split and Gate A ceiling

**Status:** deterministic artifacts published; training-pool Gate A pending
**Owner / worktree:** Codex / repository-root `main`
**Predecessor:** `v0.9.0` stopped B3 difficulty reprojection

## Superseding U1–U3 rules

This version appends the human-approved U1–U3 rules to the existing R1–R4
workflow; where they differ, these rules win.

1. **Training-pool projection (U1):** for each seed, choose the pass-count
   candidate nearest to `2` within `[1,4]`. If none exists, choose the lowest
   count within `[1,6]`. Ties are resolved by population ID. If no `[1,6]`
   candidate exists, retain R2's documented discard/backfill rule.
2. **Gate A (U2):** a 50-row training pool passes only at `k=8` when all three
   predicates hold: `0.20 <= pass@1 <= 0.60`, `mixed_fraction >= 0.30`, and
   `pass@8 >= 0.85`.
3. **Freeze and measurement scope (U3):** freeze exactly three identities:
   the 50-row training pool, a disjoint 60-row held-out evaluation set, and
   verifier version/provenance. Before/after and DoD claims use held-out
   measurements only; training-pool measurements remain training monitoring.

## Held-out split rule

Start from the immutable 326-row B1 population after excluding the 50 selected
training source population IDs. Partition the remainder by observed B1
pass-count bucket `0..8`. Each nonempty bucket receives one held-out row; the
remaining positions up to 60 are allocated proportionally to each bucket's
remaining capacity by largest remainder, breaking ties by increasing bucket.
Within a bucket, choose lexicographically by population ID. The split report
must prove exactly 50/60 rows, zero source-ID overlap, bucket allocation, and
verifier-v2 exact reference scores.

## Scope and non-goals

- Update Gate A evidence/validation and projection/splitting tooling with tests.
- Reuse the immutable B1 population/count artifact; do not call OpenRouter or
  alter verifier tier scoring, trainer logic, model configuration, or thresholds
  beyond U2.
- Do not overwrite `trainer/data/nl2sql_v1.jsonl`, freeze, tag, or train until
  the new three-predicate training-pool Gate A passes and the held-out split is
  validated.
- A future freeze implementation must bind the three artifacts; it must not use
  a training-pool metric as a before/after or DoD result.

## Validation and stop conditions

Focused tests cover U1 fallback/ties, U2's required `k=8`/pass@8 gate, and U3
allocation/overlap/version checks. Then run the deterministic split and one
pod-local training-pool Gate A. Any failed U2 predicate stops before freeze or
training; no threshold relaxation or replacement sampling is authorized.
