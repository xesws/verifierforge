# v0.10.1 — Gate A entrypoint recovery and baseline capture

**Status:** planned before implementation
**Owner / worktree:** Codex / repository-root `main`
**Predecessor:** `e3227c8` recorded a pod launch failure before Gate A import

## Problem and decision

The first v0.10.0 U2 launch invoked `python -m scripts.gate_a`, but `scripts/`
was not a Python package on the pod. Python exited before evaluation with
`No module named scripts.gate_a`; no completion, metric, or evidence artifact
exists from that attempt.

The human authorized one narrow recovery. Codex's implementation choice is to
add an empty `scripts/__init__.py`, retaining one canonical root-relative
entrypoint: `python -m scripts.gate_a`. This keeps imports and command behavior
identical on laptop and pod, without duplicating an absolute-path launcher.

## Scope

- Add the package marker and a remote preflight import to `scripts/vf` before
  it detaches a training job.
- Verify `python -c "import scripts.gate_a"` on RunPod before each direct Gate
  A tmux launch; an import failure must return synchronously and create no
  detached session.
- Run the 50-row training pool at `k=8` with full sample evidence, then run
  the 60-row held-out set at `k=8` in reference mode as the D4-before snapshot.
- If and only if the training pool satisfies all fixed U2 predicates, create
  the U3 three-piece freeze and launch the existing detached 0.5B Gate B.

## Explicit non-goals

- No verifier scoring, threshold, dataset, trainer logic, model, or external
  provider change.
- No reuse of the failed launch as a metric result and no whole-run retry beyond
  this explicitly authorized entrypoint recovery.
- The held-out result is archival/baseline only; it is never a Gate A predicate.

## Affected areas and validation

- `scripts/`, `scripts/vf`, and `docs/infrastructure/`: package/import and
  detached-job preflight; validate `bash -n scripts/vf` and a failing-import
  regression test.
- `scripts/gate_a.py` usage under `docs/evaluation-serving/`: validate local
  module invocation, pod preflight, evidence hashes, and the exact two metric
  triplets.
- `docs/model-trainer/`: Gate B remains gated on U2 plus U3 only.

The recovery stops on import/preflight failure, an infrastructure/evidence
failure, or any failed U2 predicate after the requested held-out baseline has
been captured. It must then report raw numbers and not freeze or train.
