# v0.7.0 — durable subset sample evidence

**Status:** in progress
**Owner / worktree:** Codex / repository-root `main`
**Predecessor:** `v0.6.3` reference per-prompt pass-count probe

## Problem and scope

The first 50-row Gate A kept only aggregate metrics, making the failure mix
impossible to inspect after the fact. v0.7.0 adds durable, atomic sample-level
evidence to the existing evaluator and Gate A: every saved sample retains its
full completion, progressive NL2SQL tier facts, final score, and a stable
failure class. A 50-record invocation saves samples by default; callers can
also explicitly pass `--save-samples`.

The immediate execution is one local-only diagnostic rerun of the 50-row
subset at `k=8` on pod-local Qwen2.5-1.5B-Instruct. It is reference-only: its
triplet is evidence, not a Gate A admission decision.

## Explicit non-goals

- No OpenRouter request, `.env` transfer, model change, threshold change,
  dataset mutation, freeze, tag, or trainer edit during the diagnostic.
- No scoring or verifier behavior change in v0.7.0; diagnostic tier reporting
  must reproduce the existing final score exactly.
- No raw prompt persistence; only model completions, scorer facts, and record
  identifiers are stored in the sample artifact.

## Affected areas

- `core/rewards/nl2sql.py`: additive score-breakdown API without changing
  score semantics.
- `core/eval_runner.py`: retains successful sample outcomes for evidence.
- `scripts/gate_a.py`: `--save-samples`, atomic JSONL, and taxonomy summary.
- `tests/`: verifier, runner, and Gate A evidence coverage.
- Matching documents: evaluation serving, verifiers, infrastructure, and GPT
  integrations; `docs/p0-run-sheet.md` is the operational authority.

## Evidence and validation

The JSONL sample artifact has one row per successful logical sample with
`record_index`, `record_id`, `sample_index`, `request_ordinal`, full
`completion`, `tier_scores`, diagnostic stage booleans, `final_score`, and
`failure_class`. Its SHA-256/count and an aggregate taxonomy appear in the
normal evidence JSON. The taxonomy includes `D = parse_failure / all failed`
and up to three ordered parse-failure completions verbatim. Tests prove atomic
publication, default subset behavior, score parity, and taxonomy accounting.
