# v0.7.1 — auditable format-failure routing

**Status:** implemented and routed to Branch A
**Owner / worktree:** Codex / repository-root `main`
**Predecessor:** `v0.7.0` durable subset sample evidence

## Problem and scope

The v0.7.0 subset diagnostic preserved every completion and reported a raw
verifier taxonomy. Its raw `parse_failure` count is zero, but direct inspection
shows that 322 of 341 non-full-passes are complete SQL statements wrapped in a
Markdown SQL fence. The legacy lexical read-only gate classifies those as
`not_single_read_only_statement`; operationally, they are the runbook's
format/SQL-extraction failure class.

v0.7.1 adds a deterministic, atomic routing artifact derived from the immutable
sample JSONL. It retains the raw taxonomy and hashes, records a separate
`format_parse_failure` category only when the completion is fenced and has that
legacy failure detail, writes up to three full matching completions, and
computes the runbook D value from that operational category. It does not alter
the verifier, model request, candidate data, or original evidence artifact.

## Explicit non-goals

- No retry, model request, OpenRouter use, candidate mutation, threshold
  change, freeze, tag, or trainer change.
- No rewrite of the original v0.7.0 JSONL/evidence files or their raw scorer
  classifications.
- No heuristic classification of arbitrary non-fenced execution errors.

## Affected areas

- `scripts/`: deterministic sample-evidence router and atomic JSON output.
- `tests/`: routing predicate, accounting, full-completion retention, and
  failed-publication coverage.
- `docs/evaluation-serving/`, `docs/verifiers/`, and `docs/infrastructure/`:
  routing semantics, raw/derived taxonomy boundary, and artifact handling.
- `docs/p0-run-sheet.md`: actual O1 evidence and O2 branch decision.

## Acceptance and route rule

For every non-full-pass, the artifact must account for exactly one operational
category. A `format_parse_failure` requires both
`failure_detail == "not_single_read_only_statement"` and a leading Markdown
code fence. `D = format_parse_failure / failed_sample_count`. The artifact
must bind its result to the input sample JSONL SHA-256 and source evidence SHA-
256, use atomic publication, and retain the first three ordered full
completions. `D >= 0.50` selects Branch A; otherwise Branch B.

## Result

The router published `runs/p0-gate-a/v0.7.1-format-route.json` atomically with
SHA-256 `dfbe6c2c4f7ef4b80b556f6af959c2774f0cd16f5c722d2600190d3ac0822d3f`.
It verified the v0.7.0 sample/evidence hashes, retained three complete fenced
SQL examples, and produced `D=322/341=0.9442815249266863`. The resulting route
is Branch A. Focused routing tests passed before artifact generation; no model
request was made.
