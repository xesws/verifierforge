# v0.17.2 — reviewer sandbox and overnight handoff

**Status:** planned before implementation, 2026-07-17.
**Owner/worktree:** Codex on `main` in `/Users/tangyiq/dev/verifierforge`.

## Problem and scope

The submission bundle needs a reproducible, zero-cost review surface and a
factual morning handoff. This patch may package a one-command local sandbox
that starts the existing artifact-mode API and fake-upstream proxy, and may
timebox a Cloudflare quick-tunnel experiment on the existing serving pod
without stopping its vLLM process. It also records the final S1--S10 status,
commits, and owner actions in the run-sheet.

## Non-goals

No `trainer/` or frozen-data change, model deployment change, paid model call,
credential publication, new contract field, custom authentication system, or
claim of public availability is permitted. A public tunnel is not a substitute
for invite authentication: if no pre-existing, reviewable authorization
mechanism is available, package the local launcher and record that boundary.

## Affected areas

- `scripts/`: a non-secret reviewer-sandbox launcher only if the timeboxed
  external path cannot be safely completed.
- `app/api/` and `app/proxy/`: consumed unchanged in artifact/fake modes.
- `docs/infrastructure/`, `docs/backend-api/`, `docs/submission/`, and
  `docs/p0-run-sheet.md`: operating instructions, limitations, and handoff.

There is no contract or mock impact.

## Validation plan

1. Inspect existing service/auth/tunnel capabilities without interrupting
   `vfserve`.
2. Spend at most 30 minutes on a safe, no-paid-call tunnel experiment. Record
   the exact blocker once and switch to the local launcher if it cannot provide
   a secure invitation boundary.
3. Smoke the launcher against `VF_API_DATA_MODE=artifacts` and
   `VF_PROXY_UPSTREAM=fake`; verify `/jobs` and a fake OpenAI-compatible
   completion response locally.
4. Run focused tests and `pytest -q`, then commit/push the sandbox wave.
5. Append the dated morning report after all S1--S10 outcomes are factual;
   include an ascending-time owner checklist, commit list, and raw blockers.
