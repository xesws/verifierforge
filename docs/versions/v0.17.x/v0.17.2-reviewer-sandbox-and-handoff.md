# v0.17.2 — reviewer sandbox and overnight handoff

**Status:** completed with explicit owner actions, 2026-07-17.
**Owner/worktree:** Codex on `main` in `/Users/tangyiq/dev/verifierforge`.

## Problem and scope

The submission bundle needs a reproducible, zero-cost review surface and a
factual morning handoff. This patch may package a one-command local sandbox
that starts the existing artifact-mode API and fake-upstream proxy, and may
timebox a Cloudflare quick-tunnel experiment on the existing serving pod
without stopping its vLLM process. It also records the final S1--S10 status,
commits, and owner actions in the run-sheet.

## Non-goals

No `trainer/` or frozen-data change, model deployment change, paid model call,
credential publication, new contract field, custom authentication system, or
claim of public availability is permitted. A public tunnel is not a substitute
for invite authentication: if no pre-existing, reviewable authorization
mechanism is available, package the local launcher and record that boundary.

## Affected areas

- `scripts/`: a non-secret reviewer-sandbox launcher only if the timeboxed
  external path cannot be safely completed.
- `app/api/` and `app/proxy/`: consumed unchanged in artifact/fake modes.
- `docs/infrastructure/`, `docs/backend-api/`, `docs/submission/`,
  `docs/p0-run-sheet.md`, and `JUDGES.md`: operating instructions,
  limitations, and handoff.

There is no contract or mock impact.

## Validation plan

1. Inspect existing service/auth/tunnel capabilities without interrupting
   `vfserve`.
2. Spend at most 30 minutes on a safe, no-paid-call tunnel experiment. Record
   the exact blocker once and switch to the local launcher if it cannot provide
   a secure invitation boundary.
3. Smoke the launcher against `VF_API_DATA_MODE=artifacts` and
   `VF_PROXY_UPSTREAM=fake`; verify `/jobs` and a fake OpenAI-compatible
   completion response locally.
4. Run focused tests and `pytest -q`, then commit/push the sandbox wave.
5. Append the dated morning report after all S1--S10 outcomes are factual;
   include an ascending-time owner checklist, commit list, and raw blockers.

## Sandbox result

`cloudflared` was absent on both the laptop and the still-running `vfserve`
pod. The repository scan found no invite/authentication implementation, so no
public tunnel was started and no outbound model call was made. This is a secure
early stop rather than a 30-minute wait: an unauthenticated quick tunnel would
violate the requested invitation boundary.

`scripts/start_reviewer_sandbox.sh` is the packaged fallback. It binds only
`127.0.0.1`, starts existing artifact-mode API and fake proxy processes, keeps
their logs and SQLite traffic file under ignored `runs/reviewer-sandbox/`,
checks both ports before launch, and tears both children down on exit. Its
integration test exercises `/jobs` and a deterministic OpenAI-compatible fake
completion; conflict and shell-syntax tests cover failure paths. Focused
validation passed `10 passed`; full validation passed `246 passed, 1 skipped`.

## Handoff result

`docs/p0-run-sheet.md` ends with the required dated morning report. Its owner
checklist is ordered by estimated time and distinguishes the supported local
review path from two unresolved boundaries: public ingress/invitation
authentication and the incomplete timed S3 node-loss proof.

During a read-only remote service-status inspection, the endpoint key appeared
as a process argument in agent tooling output. The value was not copied into
the repository, evidence, or documentation, but it must be treated as exposed.
Because rotating it requires a deliberate short vLLM restart, the final owner
checklist records rotation as the first operational action rather than silently
disrupting the running service.
