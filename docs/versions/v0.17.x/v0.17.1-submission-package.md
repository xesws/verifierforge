## Problem and scope

The reviewer-facing repository needs an honest submission packet and a
deliberate repository-visibility check. This patch covers: a private-repository
attempt through GitHub CLI, a Devpost draft, a three-minute video script, a
ten-minute judge path, and an owner-action record for any authorization block.

## Non-goals

This version does not deploy a public endpoint, change model weights, modify
the training pipeline or frozen data, make paid model calls, or alter the
existing vLLM service.

- `JUDGES.md`: repository-root quick-start path.

There is no contract or mock change.

## Result

`gh repo edit xesws/verifierforge --visibility private
--accept-visibility-change-consequences` succeeded, and the read-only
verification returned `xesws/verifierforge visibility=PRIVATE`. The first local
CLI invocation was rejected before any API request because GitHub requires the
explicit consequences flag; it was a local syntax/preflight correction, not an
authorization retry. The Devpost draft, exact three-minute script, and
GPU-free judge path were then written with only committed evidence and stated
limitations.
