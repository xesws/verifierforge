# v0.6.1 — P0 data-freeze execution

**Status:** in progress
**Owner / worktree:** Codex / repository-root `main`
**Predecessor:** `78912f1` (`v0.6.0 Data: discard malformed expansion responses`)

## Problem and scope

The D3 implementation needs one bounded, reproducible live path from 50 reviewed NL2SQL seeds to a frozen subset that the existing 0.5B GRPO job can consume. This version adds only operational tooling needed to run that path: malformed-response retry/timebox accounting, deterministic projection, Gate A evidence hardening, freeze evidence, and the detached Gate B record.

The authoritative execution state is `docs/p0-run-sheet.md`. It defines the fixed S1–S5 order and explicit stop gates.

## Non-goals

- No change to contracts, mock data, frontend, API shapes, verifier scoring, trainer Python logic, GRPO config, or trainer data-path configuration.
- No copied `.env`, API key, vendor-specific client, OpenAI-platform test request, automatic threshold relaxation, automatic retry after a stop gate, or model artifact in Git.
- No training before a passing subset Gate A and a pushed freeze tag.

## Affected areas

GPT integrations, evaluation serving, verifier operations, infrastructure, and model-trainer operations each receive a matching v0.6.1 document. The frozen data commit contains the full candidate JSONL, a deterministic 50-row projection at the existing trainer fixture path, Gate A evidence, and a manifest tied to `NL2SQLVerifier` source identity.

## Contract impact

None. Candidate/freeze records are local data artifacts, not public API contracts. The trainer continues to consume its existing 50-row fixture format; projection metadata is additive.

## Validation

Unit coverage must prove retry, timeout, deterministic selection, fallback accounting, verifier rechecking, Gate A threshold/evidence behavior, and existing loader compatibility. Each scoped commit runs relevant tests and `pytest -q`; only Gate B uses RunPod after the tagged freeze is pushed.
