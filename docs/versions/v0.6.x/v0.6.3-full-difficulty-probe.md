# v0.6.3 — full NL2SQL difficulty probe

**Status:** in progress
**Owner / worktree:** Codex / repository-root `main`
**Predecessor:** `v0.6.2` pod-local Gate A recovery

## Problem and scope

The completed 50-row Gate A evidence preserved only aggregate metrics, so it
cannot support a later deterministic difficulty projection. v0.6.3 adds one
probe-only artifact path: a successful reference-mode evaluation can atomically
write one JSONL row per input prompt with its exact full-pass count out of `k`.
Tonight it will be used only for the already-generated 276-row candidate set
against pod-local Qwen2.5-1.5B-Instruct.

## Explicit non-goals

- No rerun or reconstruction of the prior 50-row subset's missing completions.
- No threshold change, dataset projection, freeze, tag, training, model change,
  verifier change, trainer change, or external/OpenRouter request.
- No raw prompt or completion persistence in the probe artifact.

## Affected areas

- `scripts/gate_a.py`: optional reference-only per-prompt count output.
- `tests/test_gate_a.py`: artifact, atomicity, and reference-mode coverage.
- `docs/evaluation-serving/`, `docs/infrastructure/`, and
  `docs/gpt-integrations/`: execution and configuration record.

## Contract and validation

The existing Gate A contract remains additive. The new JSONL rows contain
`record_index`, `record_id`, `pass_count`, and `k`; `pass_count` is in
`[0, k]`. The command remains `--reference`, so measured thresholds do not
control its exit status. Validate focused tests plus `pytest -q`, then launch a
detached pod job with only `VF_EVAL_BASE_URL` and `VF_EVAL_MODEL` set.
