# v0.6.0 — D3 data freeze and Gate A

**Status:** in progress  
**Owner / worktree:** Codex / `feature/v0.6.0-data-freeze` in `../vf-v0.6.0`  
**Predecessor:** `d543705` (`v0.5.0 Contracts: unlock frontend mock clusters and arena`)

## Problem and scope

D3 needs a reproducible path from NL2SQL seed examples to a verifier-screened
training set, with an evidence-based difficulty gate before GPU time is spent.
This version adds one environment-configured OpenAI-compatible LLM client,
augmentation tooling, and `scripts/gate_a.py`. Gate A samples `k=8` completions
per candidate and fails closed unless baseline pass@1 is in `[0.20, 0.60]` and
the fraction of prompts containing both pass and fail samples is at least `0.30`.

On a passing run, it records immutable input/verifier evidence and creates a
Git tag before the existing 0.5B GRPO command is launched through `vf` in tmux.
The tag is the training admission boundary: no frozen tag, no main training.

## Explicit non-goals

- No changes to `core/contracts.py`, `mock/`, frontend contracts, or API shapes.
- No change to trainer Python training logic, reward semantics, or checkpoint bridge.
- No vendor-specific HTTP client, hard-coded API base URL, model name, key, or
  committed secret; `.env` remains local-only.
- No automatic threshold relaxation or training launch after a failed Gate A.

## Contract impact

None. `Job`, mock data, and frontend-facing endpoints stay on the v0.5.0
contract. Gate evidence is a local JSON artifact, not a new API contract.

## Validation and merge criteria

1. `pytest -q` and focused unit tests cover configuration, augmentation parsing,
   evaluator accounting, and passing/failing Gate A boundaries without a network.
2. A live, secret-safe run reads only environment variables, prints no key, and
   writes final candidate/evidence JSONL outside Git-tracked source paths.
3. Gate A prints the three raw metrics and exits nonzero on either threshold
   failure. A passing candidate produces a committed freeze manifest and an
   annotated/tagged dataset-verifier revision.
4. Only after (3), `vf train <job> <cfg>` detaches and its run log contains the
   frozen tag, pass@1 curve location, and an explicit smoke-only conclusion.
