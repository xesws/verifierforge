# v0.40.0 — Technical Deep Dive PDF export

**Status:** complete
**Date:** 2026-07-21
**Owner/worktree:** primary Codex agent on `main`
**Release type:** submission capability

## Problem and scope

The public `/tech` article contains 12 expanded chapters, but judges need one
portable attachment that preserves the entire narrative rather than only the
first visible chapter. Add a deterministic print mode and exporter that turns
the same versioned Markdown and six evidence-derived SVG figures into a single
submission-ready PDF.

The PDF must include the title/summary, a numbered table of contents, all 12
chapter headings and bodies, all six figures, formulas, tables, source links,
page headers/footers, and PDF metadata. The website and PDF must continue to
share one content source.

## Explicit non-goals

- No new metric, claim, figure, training result, or product behavior.
- No screenshots standing in for selectable PDF text.
- No manual copy of chapter content into a second document.
- No change to the interactive `/tech` experience outside print/export mode.

## Affected areas

- `frontend/src/pages/TechPage.tsx` and print CSS: deterministic PDF mode.
- `scripts/export_technical_deep_dive.py`: reproducible build/serve/print and
  header/footer post-processing.
- `output/pdf/verifierforge-technical-deep-dive.pdf`: stable final artifact.
- README and submission documentation: direct download path and provenance.
- Focused tests for chapter count, source identity, and export contract.

## Validation plan

1. Assert the canonical source has exactly 12 H2 chapters and six figures.
2. Build the frontend and export through local Chromium with all chapters open.
3. Use `pypdf` to verify metadata, page count, selectable text, and all chapter
   headings; use `pdfimages` to verify figure resources are embedded.
4. Render every page with Poppler and visually inspect contact sheets plus
   representative full-resolution pages for clipping and overlap.
5. Run frontend test/lint/build, secret scan, `git diff --check`, and full
   pytest before committing.
