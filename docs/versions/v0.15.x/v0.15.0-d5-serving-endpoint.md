# v0.15.0 — D5 real serving endpoint

**Status:** in progress
**Owner / worktree:** Codex / repository-root `main`

## Problem and scope

v0.14.0 proved the proxy, routing switch, and guardian entirely with local
`fake`/`fake-tuned` upstreams. D5 needs to replace that final fake component
with the M6-selected step-350 standard-HF export on a dedicated L4 serving pod.
The pod is an empty, serving-only executor: no trainer, repository clone,
bootstrap, or training workload is allowed.

The deployment must use vLLM 0.10.2 in a dedicated virtual environment,
bfloat16, maximum model length 2048, OpenAI-compatible port 8000, and a secret
API key held only in ignored `.env`. The archive's selected model identity is
the 13-file `actor/serveable_huggingface` step-350 tree with SHA-256
`7bde853af7c82405fd1356de9bad9b6c421de45a45ce747f63ea2f8a27eda658`.

The local proof script uses the official `openai` SDK and changes only
`base_url` and `api_key` to use the public RunPod proxy URL. After that proof,
the local product proxy can use the endpoint through
`VF_PROXY_TUNED_UPSTREAM`; 50% canary traffic is evaluated by the existing
NL2SQL guardian, then reset to 0%.

## Explicit non-goals

No trainer code/configuration, frozen dataset, verifier, contract field,
checkpoint conversion, real-model retraining, RunPod training bootstrap, or
README change is in scope. The endpoint API key is never printed, committed,
or copied into the pod as a file. Real endpoint deployment beyond this L4 pod
is deferred to the D5 owner.

## Affected areas and validation

Affected repository files are `assets/` proof material, `NOTES.md`, and
`docs/p0-run-sheet.md`; the matching infrastructure, evaluation-serving, and
backend API documents describe the operational integration. Contract impact is
**none**.

Validation is ordered E1--E7: validate L4 connectivity; install only vLLM and
its bundled torch; rsync and verify the M6 tree hash; serve and test locally;
verify the public `proxy.runpod.net` route with the official SDK; route 200
synthetic requests at 50%; confirm guardian live pass rate is at least 0.5;
and restore canary to 0%. Stop immediately on absent/mismatched archive,
vLLM load failure, public route failure, or live rate below 0.5.
