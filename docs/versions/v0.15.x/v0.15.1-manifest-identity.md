# v0.15.1 — M6 manifest identity repair

**Status:** completed
**Owner / worktree:** Codex / repository-root `main`

## Problem and scope

The v0.12.7 M6 archive recorded only an aggregate tree SHA-256. During D5
transfer recovery, a diagnostic implementation encoded the tree separators as
the literal bytes `\\0` and `\\n`, rather than NUL (`0x00`) and newline
(`0x0a`), producing a false alternate identity. The original v0.12.7
`_tree_identity` function recomputed the frozen step-350 export as
`7bde853af7c82405fd1356de9bad9b6c421de45a45ce747f63ea2f8a27eda658`.

This patch makes a new, additive M6 identity manifest for all eight standard-HF
checkpoint exports. It records the canonical tree-hash algorithm and every
relative filename, byte size, and file SHA-256. The rejected diagnostic value
`22b9e85a049551114bf72e6695d462001c0ec93787d6e3ae79cfa404e7560a62` is
preserved as an audit note, never promoted to a model identity.

## Explicit non-goals

Do not retrain, convert weights, alter a checkpoint, change a frozen dataset or
verifier, alter shared contracts, start vLLM, or make an external model call.
The original v0.12.7 manifest remains immutable evidence.

## Affected areas and validation

Code: `trainer/m6_archive.py` and `tests/test_m6_archive.py`. Documentation:
this file plus `docs/model-trainer/`, `docs/infrastructure/`,
`docs/evaluation-serving/`, and `docs/p0-run-sheet.md`. Contract impact:
**none**.

Validation creates a temporary completed M3/M4 pair and asserts stable
per-file identities, byte sizes, all discoverable serving exports, and the
canonical NUL/newline tree digest. On the training pod, the new manifest must
re-identify all eight exports and retain `7bde…` for step 350 before D5
serving resumes.

## Result

The historical v0.12.7 implementation was replayed on the training pod and
confirmed `7bde…` for step 350. The earlier `22b9…` value is reproducible only
with literal backslash separators and is rejected. The new v0.15.1 manifest
contains eight 13-file inventories and has SHA-256
`182601fcb1906a3446ab5f6797c9333b944034a6d9b335d9ae7691ab026ea39c`.
