# v0.25.0 — Supabase landing and historical import (DB-2)

**Status:** in progress — 2026-07-18
**Owner/worktree:** delegated migration area, integrated by Codex on `main`
**Release type:** minor database deployment capability

## Problem and scope

DB-2 applies DB-1's Alembic schema to Supabase, imports every existing
relational row plus local job summaries, proves idempotence and content/count
parity, and runs the repository/API suite against true Postgres. The source
SQLite file is opened read-only from an immutable snapshot; local `runs/` logs
are never copied into the database.

Application code continues to require `SUPABASE_DB_URL`. For this migration
only, the operator shell maps the existing
`SUPABASE_DB_DIRECT_CONN_STRING` into that process-local variable without
printing either value. `.env` is not changed.

## Non-goals

Do not enable Postgres by default, delete/archive the source SQLite file, write
provider credentials, modify S3 objects, call a GPU/LLM/vendor API, or add a
dual-write period. The owner alone performs the persistent backend switch.

## Affected areas and interfaces

- Add a read-only SQLite/runs importer with deterministic source identities,
  dry-run/apply/verify modes, all-or-nothing transactions, and sanitized errors.
- Seed the shared three-cluster catalog and migrate traffic, routing, guardian
  scores/points, jobs, Agent decisions, and approvals.
- Run one parameterized repository suite against Postgres using an outer
  rollback transaction so tests do not pollute imported history.

No API/Pydantic response field changes.

## Validation plan

Freeze source SHA/counts; migrate schema; import twice; require the second pass
to insert zero rows; compare every table's count and canonical content digest;
run Postgres repository tests, the complete suite, and a process-local product
smoke. A permission/403 error is attempted once and becomes `OWNER-ACTION`.
`db-2-complete` is withheld until the owner persists `VF_DB_BACKEND=postgres`
and the full product regression passes in that configuration.
