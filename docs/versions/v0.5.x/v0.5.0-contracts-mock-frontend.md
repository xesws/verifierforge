# v0.5.0 — Contracts + Mock Frontend Unlock

**Status:** Complete

**Owner/worktree:** trunk branch `feature/v0.5.0-contracts-mock-frontend`
**Type:** Contract + mock capability (minor)

## Problem and Scope

Teammates need a stable additive contract surface and a richer mock server
tonight so frontend work (cluster list, job create, report arena) can start
against fixtures without waiting for real training data.

## Non-goals

- Full real-API alignment of `GET /jobs` list shape with mock.
- Writing `POST /jobs` to Storage / `runs/`.
- Renaming or removing existing contract fields.
- Real live routing or pass-rate ingestion.

## Affected Areas and Contract Impact

- `core/contracts.py` (additive only): `Cluster*`, `RoutingState`,
  `LivePassRate*`, `Arena*`, optional `Report` fields; optional nested
  `routing` / `live_pass_rate` on `Cluster`.
- `mock/server.py`: clusters endpoints, four job statuses, `POST /jobs`,
  arena/savings on done jobs.
- `app/api/main.py`: minimal `GET /jobs/{id}` returning a `Job` from files.
- Docs: `docs/frontend/mock-guide.md`, `docs/backend-api/`, this file.
- Tests: contracts, mock server, minimal API job detail.

## Validation and Merge Gate

1. `pytest -q` fully green.
2. Existing Job fixtures validate without new Report fields.
3. Live cluster embeds routing + `pass_rate` (not `pass_at_1`) points.
4. Merge to `main`, push, delete feature branch the same day.
