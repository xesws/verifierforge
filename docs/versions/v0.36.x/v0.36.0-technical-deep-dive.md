# v0.36.0 — evidence-backed technical deep dive

**Status:** complete (2026-07-20)

## Problem and scope

VerifierForge has an auditable training, Agent, database, provisioning, and
serving record, but the technical story is distributed across the run sheet,
design notes, evidence JSON, and product UI. This version publishes one English
technical narrative, six evidence-backed SVG figures, a public `/tech` route,
and a small reusable brand lockup.

The implementation starts from deployed frontend commit `212cda7` on the
short-lived branch `feature/v0.36.0-technical-deep-dive`. The Markdown article
is the only manually maintained prose source; the website consumes an exact
generated mirror because the Vercel project uploads only `frontend/`.
Data-driven figures are generated deterministically from tracked demo artifacts
and embed their input paths and SHA-256 values.

## Explicit non-goals

- No trainer, verifier, frozen-data, backend, database, or API-contract change.
- No GPU, provider, paid LLM, or new benchmark request.
- No claim that the M3/M4 comparison is a strict single-variable experiment:
  the runs also differ in base-model size and duration.
- No claim that the incomplete timed S3 node-death drill passed.
- No exposure of hidden model chain-of-thought; Agent evidence remains tool
  calls, structured rationale, and the validated terminal decision.

## Affected areas and ownership

- `docs/blog/`: canonical article, evidence map, and generated SVG figures.
- `frontend/`: public route, Markdown/KaTeX rendering, navigation, and favicon.
- `assets/brand/`: SVG mark and wordmark.
- `README.md`: top lockup and Technical Deep Dive entry.
- Owner: primary Codex integration agent; no sub-agent implementation writes.

## Contract impact

None. `/tech` is a static, public frontend route and performs no product API
call. Existing invitation auth and Discover → Forge → Runs → Proof → Ship
journey guards remain unchanged.
