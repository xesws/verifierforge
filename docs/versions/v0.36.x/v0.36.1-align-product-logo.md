# v0.36.1 — align report branding with the product VF mark

**Status:** planned

## Problem and scope

The v0.36.0 technical report introduced a new graphite/checkmark logo that does
not match the established `VF` mark in the product navigation. This patch makes
the generated brand mark, wordmark, README header, `/tech` header, and favicon
use the same blue-to-green rounded-square `VF` treatment already shipped in the
frontend.

## Non-goals

- No page layout, report content, product navigation, API, or contract change.
- No redesign of the existing frontend mark.
- No raster asset or external image generation.

## Affected areas

- `scripts/build_blog_assets.py` and generated brand/deployment assets.
- `assets/brand/`, `frontend/public/`, and the generated `/tech` bundle.
- `tests/test_blog_assets.py` for a product-mark identity regression test.
- Documentation: `docs/frontend/` and `docs/blog/`.

**Owner/worktree:** primary agent on `fix/v0.36.1-align-product-logo`.
**Contract impact:** none.

## Validation

Regenerate/check all assets, assert the SVG contains the product gradient and
separate V/F glyphs, run focused pytest plus frontend test/build, visually
inspect the rendered SVG, deploy Vercel, and verify `/tech` and favicon.
