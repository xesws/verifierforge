# v0.24.0 — Provisioner contracts and dry-run orchestration (P-1)

**Status:** in progress — 2026-07-18
**Owner/worktree:** delegated P-1 area, integrated by Codex on `main`
**Release type:** minor platform capability behind a disabled flag

## Problem and scope

VerifierForge needs a provider-neutral layer that can turn a separately
approved training configuration into a bounded GPU lifecycle. P-1 implements
only strict contracts, a deterministic mock adapter, the lifecycle state
machine, six safety fuses, and durable audit calls through DB-1 interfaces.

The public state chain is REQUESTED → PROVISIONING → BOOTSTRAPPING → RUNNING →
COLLECTING → TERMINATED/FAILED. Provider credentials are never present in a
`ProvisionSpec`, and the GPU abstraction exposes only Ada, Ampere, and H100
classes; Blackwell has no valid mapping.

## Non-goals

Do not call RunPod, Nebius, SSH, `scripts/vf`, trainer, S3, or any GPU. Do not
connect approval to execution, add a product API route, accept a provider key,
or enable automatic provisioning. `VF_AUTOPROVISION` remains false.

## Affected areas and contracts

- Add strict Pydantic `ProvisionSpec`, `ProvisionHandle`, `ProvisionStatus`,
  provider/GPU/state enums, and secret-name validation for injected env keys.
- Add async `Provisioner` Protocol, deterministic `MockAdapter`, policy
  resolver, lifecycle orchestrator, kill switch, and orphan reaper.
- Audit every mutation with actor, approval/job IDs, handle, reason, and
  before/after state through DB-1's append-only audit interface.
- Add a mock-only CLI demonstrating the gold path and fuse/failure paths.

Existing Job/Agent/Approval HTTP contracts do not change.

## Validation plan

Use the same adapter consistency suite for the mock and dry-run binding. Test
the full state path, illegal transitions, idempotent terminal ticks, Blackwell
and secret-env rejection, budget/concurrency/runtime/kill/orphan/audit fuses,
and create-timeout/SSH-unreachable/mid-run termination convergence. Grep the
P-1 package for vendor SDK, trainer, subprocess, and `scripts.vf` imports. Run
the full suite; passing creates `provisioner-p1-complete`.
