# v0.11.2 — Py-spy Blackwell hang diagnosis

**Status:** planned; documentation gate complete before execution  
**Owner / worktree:** Codex / repository-root `main`  
**Starting point:** `a8a7a91`, after v0.11.1 exhausted its two functional
Blackwell retests without a rollout or metric.

## Problem and scope

The 1.5B single-GPU Blackwell smoke has stalled either at `ray::WorkerDict` or
before GPU work, while prior logs do not identify the blocking Python frame.
v0.11.2 is a diagnostic-only, 20-minute track. It will start the unchanged
`grpo_v1_1p5b_blackwell_smoke`, treat an unchanged `WorkerDict` state for a
full three minutes as reproduction, then install the sole permitted new
diagnostic dependency, `py-spy`, in the pod virtual environment.

For every Python process belonging to the job's launch or verl/Ray session,
the run captures a raw `py-spy dump` into ignored run evidence and copies that
complete raw text into `docs/p0-run-sheet.md`. This must identify the concrete
file and line where each worker is executing, including an import, extension
compile, or lock wait if present.

## Decisions, non-goals, and stop conditions

- The current model, frozen data, reward, verifier, FSDP/TP declarations,
  eager-mode setting, and `TORCH_SDPA` override remain unchanged for the first
  reproduction. No real model API request or secret handling is involved.
- A single repair and one retest are pre-authorized only when a captured stack
  clearly shows a one-step remedy: disable/remove a JIT-compiled extension, or
  apply a documented environment-variable import bypass. The run sheet and
  matching area documents are updated before that mutation.
- At 20 minutes from job launch, or after one hanging retest, use the v0.11.1
  `vf kill` process-group cleanup, verify zero GPU memory with `nvidia-smi`,
  preserve raw evidence, and stop for human direction. No additional backend,
  retry, trainer change, or M3–M6 step is authorized.

## Affected areas and validation

- `docs/p0-run-sheet.md` records timestamps, process manifest, every raw stack,
  any classified single repair, final teardown, and conclusion.
- `docs/model-trainer/` records the unchanged smoke inputs and what stack
  evidence qualifies as a repair. `docs/infrastructure/` records remote
  sampling, evidence paths, and W1 teardown verification.
- Validate the 3-minute stall before installing `py-spy`; ensure each selected
  Python PID has a raw dump artifact; execute `vf kill`; and require
  `nvidia-smi` to report `0 MiB`. Preserve all runtime artifacts under ignored
  `runs/`; do not stage them or print/copy `.env` values.
