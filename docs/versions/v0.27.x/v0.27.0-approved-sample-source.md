# v0.27.0 — Approved sample source and production Analyze

**Status:** complete — 2026-07-19
**Owner/worktree:** Codex on `main`
**Release type:** minor Agent/product capability

## Problem and scope

Traffic storage intentionally retains prompt hashes and aggregate metrics, not
request bodies. The Forge Agent therefore cannot inspect samples unless a user
explicitly attaches a governed source. This wave adds an additive approved
sample-source contract, persistence and API controls, deterministic read-only
inspection of the frozen 50-row NL2SQL pool, and a P2 execution profile that can
propose the approved 0.5B/100-step RunPod smoke configuration.

Two one-shot production analyses will preserve the evidence boundary: first the
zero-sample `need_more_data` trace, then a source-backed decision and approval.
The exact OpenAI model ID is the already provider-discovered
`gpt-5.6-luna`; the new-call reservation is capped at USD 0.50 with no retry.

## Non-goals

Do not provision a GPU in this version, change frozen data, store traffic
bodies, enable the Agent flag by default, alter existing contract fields, or
silently repair an invalid model-produced config.

## Contract and affected areas

- Add `ApprovedSampleSource` and optional `Cluster.approved_sample_source`.
- Extend the training-model whitelist additively with the 0.5B Qwen model; the
  default remains Qwen2.5-1.5B-Instruct.
- Add repository/API support and an Alembic migration for source metadata only:
  kind, repository-relative URI, SHA-256, row count, approver and timestamp.
- Make `inspect_samples` and `check_verifiability` consume validated,
  deterministic approved JSONL samples; include source identity in cache
  evidence.
- Preserve mock/API schema compatibility and update matching tests/docs.

## Validation plan

Run contract, repository, tool, API and Agent tests, then `pytest -q`. Verify an
invalid path/SHA/row count is rejected and no sample body enters the database.
Run one source-less and one source-backed production Analyze call, preserve both
traces, approve only a legal forge decision, and reconcile the cost ledger.
