## M3--M6 implementation design (before code)

M3 and M4 use all 50 rows of the frozen training-pool alias as their training
input. Their small deterministic validation slice is copied from that same
pool and is explicitly monitoring-only; the immutable 60-row held-out file is
never passed to verl. M3 asks verl to export a self-contained Hugging Face
model beneath each eligible 50-step checkpoint in addition to its resumable
native state. M5 starts a pod-local, loopback-only vLLM server for each such
checkpoint, invokes the existing evaluator with local `VF_EVAL_*` variables,
and retains 480 full completion/tier rows per checkpoint. It selects maximum
held-out pass@1, with lower step as the tie-breaker. A missing export, serving
failure, evaluation failure, or identity/hash mismatch writes an unavailable
report and forbids a gain claim.

The entropy brake is a pure append-only-metric policy: the first ten bridged
entropy values establish the median; after step 20, ten consecutive values
below one quarter of that median atomically write `early_stopped` plus an
evidence artifact naming the values, threshold, and latest Storage checkpoint,
then terminate the child without publishing a final-model artifact. The M4
reward adapter is seed-derived Bernoulli(0.5), separate from the verifier.

## E1 implementation result

The launch target now exports the five diagnostic values and the GRPO command
adds matching `ray_kwargs.ray_init.runtime_env.env_vars.*` overrides, so they
survive Ray worker construction. `capture_runtime_evidence()` atomically writes
`evidence/runtime-environment.txt` before every GRPO child, with the effective
diagnostic values, `pip freeze`, and an `nvidia-smi` driver/GPU probe. Focused
tests and `pytest -q` passed (`187 passed, 1 skipped`), and the launch shell
passed `bash -n`. E1 may now use the regular `vf` path.

**First remote E1 result:** `e1-h100-1p5b-smoke-v0124` wrote its runtime
evidence but stopped before a Ray worker or GPU allocation. The explicit
numeric-looking Hydra values were coerced to integers, and Ray rejected them:
`runtime_env['env_vars'] must be of type Dict[str, str]`. The approved narrow
repair quotes all propagated values in the Hydra override, then reruns E1 with
a new job ID. This is a launch-boundary typing failure, not an engine verdict.

**E1 admission result:** `e1-h100-1p5b-smoke-r1-v0124` passed and completed
30 steps, with 30 bridged metrics and Storage checkpoints at 10/20/30. The
worker saw sleep-mode (`23:08:58Z`), spawn (`23:09:55Z`), EngineCore
(`23:10:59Z`), and safetensors before the T+15 diagnostic gate. Stable steps
2--30 averaged 1198.881 tokens/s and 7.828 seconds, while median values were
1302.839 tokens/s and 6.351 seconds. The run exited successfully and released
