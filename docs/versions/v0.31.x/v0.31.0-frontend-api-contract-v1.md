# v0.31.0 — Frontend API contract v1 freeze

**Status:** complete — 2026-07-19
**Owner/worktree:** Codex on `main`
**Release type:** minor integration contract freeze

## Problem and scope

Freeze the concrete HTTP surface the Monday frontend consumes. Every documented
route must exist in both the real and mock API with the same Pydantic shape.
Cluster detail aggregates the latest analyzer decision, routing state and
Guardian curve. Jobs support queued submission and full result retrieval;
completed reports expose before/after pass@1, main/control curves, optional
arena, projected savings and verdict. Analyze, Approve and Start Forge stay
three distinct action boundaries.

The full reviewer sandbox uses HTTP Basic Auth at its outer composite layer;
the application routes themselves remain unchanged for loopback deployment.

## Non-goals

Do not rename/remove any existing contract field, trigger training from
`POST /jobs`, alter trainer/frozen data, expose secrets, invent a new frontend
framework, or enable any default-off feature flag.

## Affected areas and contracts

- Add optional `Cluster.analyzer_decision` and strict `JobCreateRequest`; all
  existing `Cluster`, `Job`, `Report`, routing and Agent fields are unchanged.
- Implement real `POST /jobs` as queued metadata only and DB-backed readback.
- Aggregate cluster product state in real/mock GET responses.
- Add real/mock OpenAPI shape parity tests covering the frozen endpoint list.
- Complete `docs/frontend/api-contract-v1.md` with executable examples, field
  notes, auth, launch commands and feature-flag defaults.

## Validation and stop conditions

The additive core contract wave landed first, followed by the real/mock route
wave. Contract parsing, actual response-key parity, full pytest, secret scan
and documented command smokes are required before tag `frontend-api-v1`. Any
destructive field change or documented route without an implementation blocks
the tag.

## Result

`Cluster.analyzer_decision` and `JobCreateRequest` are additive. `POST /jobs`
persists queued metadata only. Real cluster reads aggregate repository-backed
decision, routing and Guardian state; the mock exposes the same workflow,
including write-only credential status and separately confirmed Start Forge.
Validation: `433 passed, 1 skipped`; focused contract suite `11 passed`; secret
scan, compile and diff checks passed.
