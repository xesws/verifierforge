# v0.26.0 — Database production hygiene (DB-3)

**Status:** complete — 2026-07-18
**Owner/worktree:** Codex on `main`
**Release type:** minor security and operations capability

## Problem and scope

The unified repository needs encrypted provider credential values, explicit
disconnect behavior, automated secret scanning, and a concise operating guide.
DB-3 adds a Fernet service around `CredentialStore`, database error redaction,
CI scanning, and documented migration/backup/fallback commands.

`VF_CRED_KEY` is required only when credentials are written/read. Absence never
creates a default key and never prevents unrelated reviewer-safe API startup.

## Non-goals

Do not expose a credential API, rotate a live secret automatically, modify
`.env`, enable Postgres, call a provider, or silently fall back after a database
failure. This stage is timeboxed and may close as deferred without a milestone
tag if DB-1/DB-2 prerequisites or validation remain incomplete.

## Affected areas and validation

- Fernet encrypt/decrypt service with provider/user scoping and ciphertext-only
  repository writes.
- Bounded Postgres pool/connect settings with validated environment overrides;
  SQLite keeps its small local timeout and does not accept pooler knobs.
- Sanitized database exceptions and explicit 503 behavior; proxy completion
  remains available when observability persistence fails.
- CI secret scan plus tests that inject sentinel DSNs/passwords and assert they
  never appear in logs, exception strings, traces, or committed evidence.
- README operations section for migrations, backup, explicit SQLite fallback,
  and Fernet key generation.

Passing focused security/disconnect tests, secret scan, and the complete suite
creates `db-3-complete`.

## Result

Fernet credentials are bound to both user and provider; plaintext is absent
from repository rows and object representations. Postgres pool settings are
bounded before engine creation. Disconnect tests prove fixed sanitized 503s on
control routes and a successful completion plus generic warning on the proxy.
The tracked-file scanner reports only path/category/line, never a matched
value, and runs in CI. Focused security tests and the full suite pass.
