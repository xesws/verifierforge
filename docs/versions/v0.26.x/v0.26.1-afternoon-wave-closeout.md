## Problem and scope

The new security workflow failed before dependency installation because
`setup-python` could not infer the nonstandard `requirements-app.txt` cache
input. After that correction, its second run reached the test step but exposed
a separate packaging gap: the repository had no development requirements, so
`pytest` was absent. This patch declares cache inputs explicitly, adds a small
`requirements-dev.txt`, reruns the security workflow, and records final Lane
A, DB-1/2/3, and P-1 status.
