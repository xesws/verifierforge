# v0.2.0 — D1 Remote-Control Smoke Test

**Status:** Complete — D1 remote-control smoke passed on 2026-07-14.

**Owner/worktree:** `feature/v0.2.0-d1-remote-smoke`

**Type:** infrastructure and training-control feature

## Problem and Scope

Prove the D1 path from laptop to a disposable RunPod worker: `vf bootstrap` creates or fast-forwards `/workspace/verifierforge`, installs trainer dependencies plus `tmux` and `rsync`, and initializes `/workspace/hf-cache`; `vf train demo1 fake_smoke` starts a detached five-minute fake job in tmux; and `vf watch` returns metrics/logs to the local `runs/` directory. Align `vf status` with §6.6 by showing both tmux sessions and `nvidia-smi`, without starting real GRPO training.

## Non-goals

No verl/vLLM integration, real model download, S3Storage, real GPU training, automatic recovery, or contract changes. Those follow in a separately documented and timeboxed version after this gate passes.

## Validation and Merge Gate

Require `ssh runpod` connectivity, a persistent `/workspace`, and a read-only deploy key. Run bootstrap, detached fake training, watch/log retrieval, local API JSON checks, `pytest -q`, and `bash -n scripts/vf`; document results in the affected-area files before merge.

## Evidence Record

- The laptop SSH alias was verified with `ssh runpod "nvidia-smi"`; the worker exposes an NVIDIA L4 and `/workspace` is the persistent volume.
- A read-only deploy key named `runpod-readonly` authenticated a private-repository probe clone. No write permission or secret was added to the repository.
- `bash scripts/vf bootstrap` cloned/fast-forwarded `/workspace/verifierforge`, created its virtual environment, installed trainer requirements, and created `/workspace/hf-cache`. A second run completed successfully, establishing idempotency.
- `bash scripts/vf train demo1 fake_smoke` returned immediately after starting tmux session `demo1`. The detached trainer ran 150 steps at a two-second interval after the SSH caller exited.
- `bash scripts/vf watch demo1` synchronized 150 JSONL records and the final artifact to the laptop. Local and remote `metrics.jsonl` SHA-256 values matched: `a30b250de8932c6ffef67ab14ade294d51d881df201a2640eafe37e11295ab00`.
- `vf logs`, `vf status`, `vf kill` (using an isolated `killcheck` job), and `vf model` were exercised successfully. `killcheck` was removed after producing two metrics records.
- The local API, served on port 8010 because port 8000 was already occupied on the laptop, reported `demo1` as `done` and returned four 150-element metric arrays ending at step 150.
