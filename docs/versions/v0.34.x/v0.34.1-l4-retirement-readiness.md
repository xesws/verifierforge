# v0.34.1 L4 retirement readiness

**Status:** complete — 2026-07-20

## Problem and scope

The scale-to-zero path passed, but the hosted environment and older local-full
launcher still retain static tuned-upstream variables from the retired L4 era.
This patch makes the production proxy resolve every physical tuned endpoint
from `serving_endpoints`, verifies the complete reviewer surface while the
registry is cold, and documents the owner-safe L4 shutdown boundary.

## Non-goals

- Do not connect to, restart, stop, or mutate any process on the old L4.
- Do not wake a serving pod or make a paid LLM/provider request.
- Do not change frozen Job/report fields, training code, or model data.
- Do not remove historical evidence URLs whose purpose is explicitly archival.

## Affected areas and ownership

- Proxy/runtime: eliminate active `VF_PROXY_TUNED_*` physical endpoint inputs;
  keep deterministic fake-tuned behavior only for injected tests.
- Hosted/local-full configuration: cold start with the Supabase registry and no
  physical endpoint secret or URL.
- Frontend/reviewer: make cold fallback observable and verify all serving-state
  messages.
- Documentation: JUDGES, infrastructure, backend API, frontend, and run-sheet.

Owner/worktree: primary Codex agent on main in the repository root. Contract
impact is additive headers/documentation only; JSON response fields are frozen.

## Validation plan

Inspect Railway variable names without values, Supabase registry/routing state,
and repository references. Remove stale hosted variables, redeploy, run focused
and full tests plus secret scan, then exercise the public Basic-auth reviewer in
cold state: three Discover cards, flagship Job/report/metrics, routing,
LivePassRate, and a forced tuned request that must visibly fall back. The old L4
is safe to close only if the registry is cold, provider inventory is zero, all
reviewer reads succeed, and no active static endpoint input remains.

## Result

Railway and the ignored local `.env` no longer contain any of the four legacy
physical-endpoint variables. The production proxy resolves only the Supabase
registry, which is cold with `url=NULL`; managed provider inventory is zero.
Every cold-state public reviewer read returned HTTP 200. A forced logical tuned
request returned HTTP 200 through explicit `X-VerifierForge-Route:
default-fallback`, then routing was restored to disabled/canary zero.

The audit also fixed failure cleanup calling a synchronous drain from the
async serving loop and reduced the three-card Discover hydration from about
9.4 seconds to 2.5–2.8 seconds. Final validation was 474 passed, one skipped,
plus secret scan and shell syntax.
