# v0.34.2 — Nora frontend branch review

**Status:** complete

**Date:** 2026-07-20

**Owner / worktree:** Codex / repository-root `main`

## Problem and scope

Audit Nora's `feature/v0.18.0-static-frontend` branch without merging or
modifying it. The review covers its stack, five product pages, frozen frontend
API compatibility, credential and dependency hygiene, local buildability,
judge-path gaps, merge strategy, and Vercel settings.

The canonical result is `docs/frontend/branch-review.md`. Existing branch-owned
screenshots are copied byte-for-byte into `assets/frontend-review/` with
explicit provenance because a fresh interactive browser is unavailable in the
current execution environment.

## Explicit non-goals

- No frontend integration work, UI redesign, or API client implementation.
- No merge, rebase, push, or modification of Nora's branch.
- No backend, contract, mock, database, provider, or deployment mutation.
- No representation of branch-owned screenshots as fresh audit captures.

## Affected areas

- `docs/frontend/branch-review.md`: canonical evidence-backed review.
- `docs/frontend/v0.34.2-nora-frontend-branch-review.md`: area record.
- `assets/frontend-review/`: screenshot evidence copied from the reviewed SHA.

No public API, Pydantic contract, application source, or frontend source is
changed by this version.

## Validation plan

- Identify the branch and commit through GitHub and compare it with current
  `main` from an isolated temporary clone.
- Run `npm ci`, lint, build, development-server, production-preview, dependency
  audit, and HTTP route checks without modifying tracked branch files.
- Compare all 21 frozen frontend operations and highlighted response fields.
- Scan the branch delta for secrets, environment files, heavy dependencies,
  and backend or contract edits.
- Run the repository test suite, secret scanner, and `git diff --check` before
  committing only the review deliverables.
