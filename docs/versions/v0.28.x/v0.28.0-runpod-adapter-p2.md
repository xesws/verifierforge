# v0.28.0 — RunPod adapter and P2 live path

**Status:** in progress — 2026-07-19
**Owner/worktree:** Codex on `main`
**Release type:** minor provisioning capability

## Problem and scope

P-1 proves provider-neutral lifecycle policy with a mock. P-2 adds a real
RunPod REST adapter and one approval-driven executor that provisions the least
expensive allowed Ada-class pod, streams the exact committed revision by Git
bundle, bootstraps a volume-free S3-backed trainer, monitors progress through
S3, verifies final object hashes, terminates the pod, and confirms billing has
stopped.

The wave-wide spend cap is `$5`; managed concurrency is one; full-run runtime is
at most 180 minutes. Any stop/failure trigger starts a 30-minute cleanup SLA.
Provider resources are managed only when both the `vf-auto-` name prefix and
the VerifierForge owner marker match. Existing user pods are out of scope.

## Non-goals

Do not change the training algorithm, attach a network volume, add a provider
credential to a pod, enable autoprovision by default, wire the product button to
execution, support Nebius, or call any LLM.

## Affected areas and contracts

- Add `RunPodAdapter` using `httpx.AsyncClient` and current official REST pods
  and billing endpoints. Every status call refreshes SSH data.
- Add durable approval lookup/handle binding and a DB-backed active registry.
- Add a provider-neutral workload runner around P-1 lifecycle states while
  keeping the existing mock adapter path compatible.
- Add `scripts.provision_runpod --approval-id <id> --execute-live`.
- Add one 0.5B P2 config: frozen 50-row pool, 100 steps, k=8, checkpoints 50/100,
  serviceable HF publication and S3-only state.
- Add HTTP contract tests, lifecycle/fuse tests, S3 collection tests, and a live
  evidence artifact/run-sheet ledger.

## Validation and stop conditions

Before creating a paid pod: full pytest, secret scan, clean Git status, pushed
HEAD, and read-only account inventory. Then prove create/status/delete/billing,
orphan reap, and the approval-driven 100-step run. Final collection requires
metric step 100, checkpoint manifest step 100, final model marker, curve PNG,
and per-file SHA-256. Missing billing-stop evidence, spend `$5`, runtime 180
minutes, a kill switch, or any unsafe identity mismatch forces termination and
prevents the `provisioner-p2-live` tag.
