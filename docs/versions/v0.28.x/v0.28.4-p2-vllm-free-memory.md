# v0.28.4 — P-2 vLLM free-memory ceiling

**Status:** implementation complete; live `-r4` pending — 2026-07-19
**Owner/worktree:** Codex on `main`
**Release type:** patch correction to v0.28.3

## Problem and scope

The v0.28.3 `-r3` attempt reached step 50, published 49 metrics, converted the
checkpoint, and exercised the new remote-failure detector. Its 0.90 serving
ceiling was incorrect for vLLM 0.10.2: the engine first compares the requested
fraction of total memory with free memory at startup. The preserved provider
log reports `Free memory on device (14.51/19.67 GiB)` and a requested 0.90
target of 17.71 GiB, then instructs the caller to decrease utilization.

The serving smoke must use 0.70. That requests about 13.77 GiB, remains below
the observed 14.51 GiB free-memory guard, and leaves substantial capacity for
the 0.927 GiB model plus the tiny 64-token/one-sequence KV cache. The repository
also previously validated 0.70 for its local vLLM evaluator.

The chained retry validator must generalize from `-r3` evidence: the approval
root remains the first terminal handle, while `retry_of` and the admission's
previous handle identify the immediately preceding replacement. One clean
`-r4` S3 prefix is allowed under the same cumulative `$5` fuse.

## Non-goals

- No trainer algorithm, config, data, model, provider allowlist, checkpoint
  cadence, API contract, feature flag or S3 persistence change.
- No reduction of the serving load/completion acceptance gate.
- No reuse of the three failed prefixes or pod-local staging.
- No fifth attempt if `-r4` fails.

## Affected areas and contract impact

- `trainer/serving_smoke.py`: fixed 0.70 vLLM ceiling.
- `scripts/provision_runpod.py`: immediate-previous versus approval-root retry

No public contract changes.

## Validation and stop conditions

Focused and complete tests, tracked-secret scan, raw prefix zero, absent prior
target, actual chained-evidence admission, empty `-r4` S3 prefix and clean
pushed HEAD are mandatory before create. Live completion still requires 100
metrics, checkpoint publication, final marker, curve, SHA-verified collection,
DELETE and raw prefix zero. Any `-r4` failure ends P-2 without a semantic tag.
