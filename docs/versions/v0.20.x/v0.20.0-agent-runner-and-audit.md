# v0.20.0 — Guarded Forge Agent runner and audit persistence

**Status:** completed
**Owner/worktree:** Codex, repository `main`
**Stage:** Forge Agent A-2

## Problem and scope

Run the A-1 tool graph through a bounded OpenAI-compatible tool-calling loop.
Only the runner-owned `submit_decision` tool may terminate a run. Unknown tools,
bad JSON, fabricated dependency IDs, illegal configs, and budget-policy breaches
reject the entire run without correction. Every attempt must remain auditable.

The design document names `AgentDecisionStore` but supplies no methods. This
version records the Codex implementation decision: `put`, `get`, and
`latest_for_cluster`. SQLite implements the summary store for tonight; full
sanitized traces use a dedicated S3 object store.

## Non-goals

No evaluator, HTTP/UI integration, approval, trainer, provisioner, subprocess,
GPU SDK, or training action. The trace contains no hidden chain-of-thought.

## Persistence and failure semantics

The runner writes the complete trace to S3 first and the SQLite summary second.
S3 failure is fail-closed: a failure summary is written with no decision and the
caller receives an error. SQLite failure after S3 may leave an orphan trace, but
the caller still receives an error. A decision without a durable trace can never
be cached or approved.

## Validation plan

Use scripted model turns and moto to cover a full trace, tool-call order,
step/token/timeout guards, all invalid action classes, dynamic budget lowering,
S3-first ordering/failure, SQLite latest/fingerprint semantics, and import scans
that prohibit every execution-side dependency.

## Result

The bounded runner, strict terminal submission, dynamic budget guard, SQLite
summary store, and S3 trace store are implemented. Focused Agent validation
passed 25 tests; the full suite passed 280 tests with one expected skip. Tests
proved S3-first publication, failure-summary behavior, checksum metadata,
dependency/early-submit/token/budget rejection, and absence of execution-side
imports. No live provider request was used in A-2.
