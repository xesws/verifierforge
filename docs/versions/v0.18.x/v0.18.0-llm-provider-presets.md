# v0.18.0 — LLM provider presets and governance

**Status:** implementation complete; live preset gate blocked  
**Owner/worktree:** Codex, repository `main`  
**Stage:** Forge Agent A-0

## Problem and scope

VerifierForge has one OpenAI-compatible transport but only an OpenRouter-shaped
default. Add explicit `openrouter` and `openai` presets while retaining generic
environment overrides and existing callers. The OpenAI preset for this wave is
exactly `gpt-5.6-luna-xhigh`; any model containing `sol` or `terra` must be
rejected before a network request. The owner reports a $10 OpenAI credit balance;
this work may spend at most $3 and must preserve a $7 safety reserve. OpenRouter
may spend at most $2.

This version also admits the owner-authored `docs/design/` system documents and
the four repository governance rules into Git under stable filenames.

## Non-goals

No Forge Agent contracts, tools, runner, evaluator, product UI, provisioner,
GPU action, or training change belongs to A-0. Secrets and raw completion text
must not enter committed evidence.

## Affected areas and contract impact

- `app/gpt/`: additive provider resolution, tool-turn response metadata, usage,
  timeout, and token limits.
- `docs/design/`, `AGENTS.md`, `docs/gpt-integrations/`, and
  `docs/infrastructure/`: durable policy and evidence rules.
- No existing public Pydantic contract changes.

## Validation plan

Unit-test provider defaults, explicit override precedence, legacy augmentation
compatibility, Luna allowlisting, Sol/Terra rejection, usage parsing, and secret
redaction. Run one bounded live request per provider only after a cost preflight;
record safe metadata and exact token/cost evidence in the run-sheet.

## Execution result

Focused tests passed. The one authorized OpenRouter request resolved to
`https://openrouter.ai/api/v1` / `z-ai/glm-5.2` but returned no usable message
content (`LLMResponseError`). The one authorized OpenAI request resolved to
`https://api.openai.com/v1` / `gpt-5.6-luna-xhigh` and returned HTTP 404. Both
failed calls were charged at a conservative $0.25 reservation upper bound in
the local ignored ledger; neither provider supplied a billable cost value.
No Sol/Terra request, fallback, retry, raw completion, or secret was recorded.
