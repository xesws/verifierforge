## Problem and scope

The README explains how the repository was built with Codex but does not give
GPT-5.6 its own concise account. Add a section immediately before “How we
worked with Codex” that separates GPT-5.6's two roles in VerifierForge:

1. collaborative development alongside Soul Ultra;
2. product intelligence through Luna in the Agent Layer and GPT-5.6-assisted
   C-code verifier generation.

The section must distinguish these roles from training and serving the final
small model. It must also preserve the repository's evidence boundary: Agent
decisions use a strict schema, and generated verifier candidates are reviewed
and validated rather than trusted blindly.
