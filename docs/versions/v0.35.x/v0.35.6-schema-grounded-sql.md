# v0.35.6 — Schema-grounded SQL generation

Status: deployed

## Problem and scope

Ship loaded the frozen SQLite fixture correctly, but sent the tuned model only
a generic "return SQL" instruction. The model therefore had no knowledge of
the actual columns and could invent names such as `employees.department` and
`employees.status`; the correctly loaded SQLite database then rejected them.

Export a schema-only prompt asset from the same canonical frozen source as the
browser fixture. Include that exact schema in every tuned SQL generation
request so generation and execution share one schema identity.

## Non-goals

- Changing frozen data, the model, training, verifier scores, or API contracts.
- Hiding genuine model mistakes or replacing failed output with reference SQL.
- Automatically retrying or rewriting SQL after an execution error.

## Contract impact

None. The request remains an OpenAI-compatible chat completion and the frozen
public HTTP boundary remains 22 operations. Only prompt content changes.

## Validation plan

- Export/check one schema-only asset from the canonical 50-row training pool.
- Assert that the model request contains all four table definitions and exact
  fields needed by the sample questions, without fixture INSERT rows.
- Execute the correct Engineering query against the same browser fixture.
- Run frontend tests, lint/build, frozen-asset checks, and the repository suite.
- Deploy to Vercel and confirm the public bundle contains the grounded schema.

## Result

The deterministic exporter now emits a DDL-only prompt schema with SHA-256
`7aa3240d901cd982b381aa3e75e538d5136f72bdf60ca405ede3d667cd073878`.
The generated manifest records that identity alongside the unchanged complete
fixture identity. Generation requests include the DDL but no INSERT rows or
reference SQL.

Validation on 2026-07-20: 26 frontend tests, ESLint, the Vite production build,
the asset check, and 480 repository tests with one skip passed. Vercel
deployment `dpl_FpLH7DgBhVP1CAKNExUzrE4CuDG6` reached READY at the stable
alias, whose public bundle contains the schema instruction and employee DDL.

With the existing registry already ready, one live tuned-only Engineering
probe generated the correct `employees`/`departments` join with `e.active = 1`.
Executing that exact response against the frozen fixture returned Ada,
Frances, and Grace. No wake or new GPU was requested.
