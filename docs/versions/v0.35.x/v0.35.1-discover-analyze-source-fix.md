## Problem and scope

The deployed Discover form sent the repository JSONL selected by **Input** as
`AgentAnalyzeRequest.data_source`. That request field is a guarded internal
override for the proxy traffic SQLite path, so production correctly rejected
the request with `data_source must match the configured VF_PROXY_DB_PATH`.
Because Analyze never produced a decision, the journey correctly kept Forge
and every downstream stage locked.

Fix the frontend boundary so **Input** continues to approve the governed
training/evaluation sample source, while Analyze omits the internal traffic DB
override and lets the API use its configured traffic store.

## Non-goals

- No weakening or removal of the API path-validation guard.
- No contract, database, trainer, provisioner, or serving changes.
- No automatic stage unlock after an Analyze error.
- No GPU wake or paid model request.

## Affected areas

- `frontend/src/pages/DiscoverPage.tsx`: separate the two source meanings.
- `frontend/src/test/pageFlows.test.tsx`: reject regression to a client-supplied
  traffic path and prove successful Analyze unlocks Forge.
- `docs/frontend/v0.35.1-discover-analyze-source-fix.md` and run-sheet:
  document diagnosis and public acceptance.

## Contract impact

None. `data_source` remains an optional, protected operator override in the
existing API contract. The reviewer frontend no longer populates it.
