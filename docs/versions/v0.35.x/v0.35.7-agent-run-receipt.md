# v0.35.7 — Auditable live Agent receipt

Status: deployed

## Problem and scope

Discover currently renders only a decision and rationale. In the public
sandbox those words can come from the deterministic mock binding, yet the UI
does not disclose the provider or show the actual read-only tool trace. A
reviewer therefore cannot distinguish a fresh model run, a cached run, and a
mock fixture.

Extend the existing Analyze response additively with an auditable run receipt:
trace ID, provider, model, timestamps, token counts, cache status, guard events,
exact validated tool arguments/outputs, and the terminal structured decision.
Render those records in a compact expandable panel. A mock or cached result
must be labelled as such and must never be described as a live model run.

## Non-goals

- Exposing hidden chain-of-thought, system-private reasoning, credentials, or
  provider internals.
- Inventing prose between tool calls or presenting a normalized trace as raw
  provider bytes.
- Provisioning GPUs, starting training, or changing Agent decision policy.

## Contract impact

`AgentAnalysisResponse` gains additive provenance and a full validated
`AgentTrace`. No field is renamed or removed and no HTTP operation is added;
the frozen operation count remains 22. Mock and real APIs must remain shape-compatible.

## Validation plan

- Contract round-trip and mock/real shape tests cover the receipt.
- API tests prove fresh vs cached identity and exact tool trace persistence.
- UI tests prove mock/cached/live labels and verbatim structured decision/tool
  JSON are visible after Analyze.
- Full frontend tests, lint/build, repository pytest, and public deployment
  smoke must pass.

## Result

The public Analyze path now runs with `VF_AGENT_BINDING=real`, provider
`openai`, and the `/v1/models`-verified `gpt-5.6-luna`. `sol` and `terra` were
not configured. One production `force_refresh=true` acceptance returned HTTP
200 with `cached=false`, trace `5595903532224e9b91d6e4d5dc640bd4`, 7,254 input
tokens, 536 output tokens, the required four-tool chain, and a `forge`
decision at 0.98 confidence. A no-cost reload returned the same trace with
`cached=true`, proving persistence.

An OpenRouter `z-ai/glm-5.2` compatibility probe was rejected by the existing
guard at `estimate_economics.base_model must equal the issued evidence value`;
it was therefore not deployed. This failure is evidence that the guard, rather
than presentation code, controls the action boundary.

Validation: 27 frontend tests, ESLint, production build, and 480 repository
tests with one skip passed. Railway deployment
`3fef3ee5-e4a6-422f-907a-5fdb396072ab` succeeded. Vercel deployment
`dpl_5uHsdR44p8tbxapJMf5zkwuVSeqK` reached READY at the stable alias; the
public bundle contains the live/mock labels and receipt UI.
