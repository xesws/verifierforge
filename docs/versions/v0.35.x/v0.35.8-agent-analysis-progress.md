# v0.35.8 — Agent analysis progress

Status: deployed

## Problem and scope

Discover currently changes the Analyze button label but leaves the reviewer
without an explicit request state while the live Agent is running. Add a
visible, accessible progress panel with elapsed time and honest request stages.

The panel may report only states the browser can observe: request dispatched,
Agent response pending, response received, and failure. It must not invent a
percentage, claim a particular server-side tool has completed before the
persisted receipt arrives, or expose hidden chain-of-thought.

## Non-goals

- No API, contract, Agent runner, provider, or training changes.
- No fabricated token stream or server-side tool progress.
- No changes to the completed run receipt.

## Affected areas

- `frontend/src/pages/DiscoverPage.tsx`
- `frontend/src/styles/index.css`
- `frontend/src/test/pageFlows.test.tsx`
- `frontend/package.json`
- `docs/frontend/v0.35.8-agent-analysis-progress.md`

Owner/worktree: primary agent on `fix/v0.35.8-agent-analysis-progress`.
Contract impact: none.

## Validation

- A delayed Analyze response immediately renders an accessible live status.
- Elapsed time advances while the request is pending.
- The request, Agent, and receipt stages are visibly distinguished without a
  fake completion percentage.
- Success replaces progress with the audited receipt; failure shows the real
  error and clears the busy state.
- Frontend tests, lint, and production build pass.

## Result

The production Discover page now renders the progress panel immediately after
Analyze is clicked. It reports live elapsed seconds, disables conflicting
actions, and distinguishes request dispatch, server-side Agent evaluation, and
the pending validated receipt without inventing a completion percentage.
