## Problem and scope

Ship exposes one free-form tuned SQL query, but gives a reviewer no guidance
about the frozen employee/department/project schema. Its customer-order default
is not part of that schema. Add several one-click examples drawn from the
existing NL→SQL training-pool question vocabulary, spanning simple filters,
joins, aggregation, and grouped ranking.

## Non-goals

- No API, model, verifier, frozen-data, serving, routing, or trainer change.
- No automatic request when an example is selected.
- No paid completion or GPU Wake during implementation acceptance.
- No claim that sample selection is an evaluation result.

## Affected areas

- `frontend/src/data/presentation.ts`: reviewed prompt catalog.
- `frontend/src/pages/ShipPage.tsx`: accessible prompt selector.
- `frontend/src/styles/index.css`: responsive chips matching the current UI.
- `frontend/src/test/pageFlows.test.tsx`: selection and request-body coverage.
- `docs/frontend/v0.35.2-ship-sample-prompts.md` and run-sheet.

## Contract impact

None. A selected sample only populates the existing natural-language textarea;
`POST /serving/tuned-completion` is unchanged.
