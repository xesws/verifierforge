# v0.35.0 — Frontend API integration and Vercel delivery

**Status:** F0–F3 and F5 complete; paid Wake evidence and owner merge gate pending

**Date:** 2026-07-20

**Owner / worktree:** Codex / repository root branch
`feature/v0.35.0-frontend-api-integration`

## Problem and scope

Nora's `feature/v0.18.0-static-frontend` is the accepted visual base, but it
does not call the frozen frontend API. This feature preserves that visual
language while wiring all 21 `frontend-api-v1` operations, runtime invitation
authentication, explicit loading/error/cold states, the separate
Analyze → Approve → Start Forge boundaries, scale-to-zero serving status, and
Vercel delivery.

The integration starts from current `main` and cherry-picks Nora's exact
`642e3354457529d0c51d65a10bb2e2b4b36008ec` commit so authorship remains
visible. One budget-fused real serving wake is authorized for acceptance; no
Forge training is authorized.

## Explicit non-goals

- No backend, Pydantic contract, mock, trainer, frozen-data, provisioner, or
  database schema changes.
- No redesign or replacement component library.
- No real Start Forge action while `VF_AUTOPROVISION=false`.
- No invite code, provider credential, database URL, or cloud key in source,
  build variables, screenshots, recordings, logs, or commits.

## Affected areas

- `frontend/`: typed client, view-model mapping, UI workflows, state handling,
  tests, and Vercel SPA configuration.
- `scripts/frontend_contract_smoke.py`: secret-safe public acceptance of the
  21-operation boundary, with paid Wake excluded unless explicitly armed.
- `assets/frontend/v0.35.0/`: sanitized screenshots and the authorized wake
  recording.
- `docs/frontend/`, `docs/infrastructure/`, `JUDGES.md`, and the run-sheet:
  integration, deployment, and acceptance evidence.

## Contract impact

None. The frontend consumes the 21-operation boundary frozen by
`frontend-api-v1.2`. Unknown response fields are ignored. The OpenAI-compatible
proxy completion used for the live tuned proof remains outside the frozen UI
operation count.

  and production preview.
- Full `pytest -q`, secret scan, diff check, and explicit no-backend-change
  inventory.
- Browser walkthrough of all five pages and all 21 operations against the
  Railway public API, including explicit disabled Start Forge behavior.
- Exactly one real serving wake, one tuned NL→SQL completion, immediate drain,

- Vercel preview, direct deep-link checks, owner CORS handoff, owner visual

## Stop conditions

Do not merge or tag on a contract mismatch, leaked credential, unproven GPU
deletion, failed production deep link, unresolved CORS failure, or missing
owner visual acceptance. A blocked independent item is recorded after 30
minutes and skipped; it does not silently become accepted.
