# v0.38.1 — user-story causal context

**Status:** planned; documentation gate reserved before script revision

## Problem and scope

The v0.38.0 opening names the workload volume and cost but does not explain why
the requests exist or how the bill accumulates. This patch grounds the opening
in a concrete workflow: employees use an internal data assistant for recurring
department, people, project, and assignment-hour questions; its legacy path
sends every request to a token-priced external large-model API.

The script will connect 95,000 monthly requests to the catalog's $5,500 monthly
baseline, then explain why repeated and automatically verifiable SQL work is a
candidate for a specialist small model.

## Truth boundary

- The repository supports an OpenAI-compatible, operator-configured real
  upstream. OpenRouter can be that upstream, but it is not the only one.
- `$5,500` is a committed Discover business baseline. It is not derived from
  `config/proxy_pricing.json` and is not represented as an OpenRouter invoice.
- The internal assistant is the user-story framing for the demonstrated Data
  Pull SQL workflow; no industry, company, or unsupported data object is added.

## Explicit non-goals

- No API, proxy, pricing, contract, frontend, dataset, model, or runtime change.
- No provider-specific attribution of the $5,500 figure.
- No change to the 25-second or three-sentence limit, or the 3:00 video total.
