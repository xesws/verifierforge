# v0.4.2 — Validation Batch-Size Fix

**Status:** In progress

**Owner/worktree:** Codex integration owner, `fix/v0.4.2-validation-batch-size`

**Type:** D2 trainer configuration bug fix

## Problem and Scope

After TransferQueue installation, the real two-step GRPO preflight loaded the
40/10 Parquet split and failed during verl validation DataLoader creation:

```text
ValueError: batch_size should be a positive integer value, but got batch_size=0
```

Set the explicit small-run override `data.val_batch_size=1` in the GRPO command
builder and assert it in the configuration test, then repeat the same preflight.

## Non-goals

Do not change the reviewed data split, train batch size, model, vLLM settings,
reward, checkpoint cadence, or TransferQueue installation.

## Affected Areas and Validation

- `trainer/grpo_config.py`
- `tests/test_trainer_bridge.py`
- `docs/model-trainer/v0.4.2-validation-batch-size.md`

Run the narrow config test, full pytest, and detached remote two-step preflight.
