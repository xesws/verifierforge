# v0.4.3 — SDPA Attention Compatibility Fix

**Status:** In progress

**Owner/worktree:** Codex integration owner, `fix/v0.4.3-sdpa-attention`

**Type:** D2 trainer runtime compatibility bug fix

## Problem and Scope

The detached two-step `d2preflight` reached model construction on the RunPod
L4 after TransferQueue and the validation-batch fixes. It then failed before
training with:

```text
ImportError: FlashAttention2 has been toggled on, but it cannot be used because
the package for FlashAttention2 does not seem to be installed.
```

verl 0.8 defaults its model configuration's `attn_implementation` to
`flash_attention_2`. Set its explicit model override to PyTorch `sdpa`, which
is available in the proven torch 2.8.0+cu128 pod environment. Add a focused
command-construction test and repeat the same detached preflight.

## Decision

Codex chose SDPA rather than compiling FlashAttention2. The human's D2 goal is
a bounded real-training smoke test, not an attention-performance benchmark;
building a CUDA extension would add a new, unbounded compatibility branch to
the already time-boxed runtime validation.

## Non-goals

Do not change the model, data split, batch sizes, LoRA settings, vLLM memory
budget, checkpoint cadence, or install FlashAttention2. This does not claim
SDPA is the intended performance setting for a later production run.

## Affected Areas and Validation

- `trainer/grpo_config.py`
- `tests/test_trainer_bridge.py`
- `docs/model-trainer/v0.4.3-sdpa-attention.md`

Run the narrow configuration test, full pytest, then the detached two-step
RunPod preflight. Its log must pass model construction without the
FlashAttention2 import error.
