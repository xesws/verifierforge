# v0.4.4 — Hydra Map-Override Syntax Fix

**Status:** In progress

**Owner/worktree:** Codex integration owner, `fix/v0.4.4-hydra-map-override`

**Type:** D2 trainer configuration bug fix

## Problem and Scope

The v0.4.3 preflight passed the `sdpa` decision into the real verl command but
Hydra rejected the first key added to its structured empty map:

```text
Could not override 'actor_rollout_ref.model.override_config.attn_implementation'.
To append to your config use
+actor_rollout_ref.model.override_config.attn_implementation=sdpa
Key 'attn_implementation' is not in struct
```

Use Hydra's documented `+` add-key form for this map entry and update the
focused command assertion. Repeat the same detached two-step preflight.

## Non-goals

Do not alter the v0.4.3 SDPA decision, runtime packages, model, data, or
resource settings. This patch only corrects the serialization syntax required
to express that existing setting to verl 0.8.
