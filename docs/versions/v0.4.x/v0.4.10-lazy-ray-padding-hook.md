# v0.4.10 — Lazy Ray Padding Hook

**Status:** In progress

**Owner/worktree:** Codex integration owner, `fix/v0.4.10-lazy-ray-padding-hook`

**Type:** D2 trainer runtime bug fix

## Problem and Scope

The v0.4.9 direct compatibility check resolved to the intended Transformers
padding helpers. Its next preflight did not reach the actor: Ray's raylet
aborted while waiting 15 seconds for the dashboard metrics-agent port file.
The eager `sitecustomize` hook imported verl and Transformers in every Ray
auxiliary process, including the dashboard agent, which made that startup path
too slow.

Replace the eager import with a lazy import hook. It must patch only
`verl.utils.attention_utils` when a training worker actually imports that
module; Ray control-plane processes must not import verl or Transformers merely
because the compatibility environment variable is present.

## Non-goals

Do not tune Ray timeouts, suppress the dashboard, alter model/reward/GRPO
settings, or install FlashAttention. This patch fixes the lifecycle of the
existing v0.4.9 compatibility hook only.

## Affected Areas and Validation

- `trainer/flash_attn_compat.py`
- `trainer/sitecustomize.py`
- `tests/test_flash_attn_compat.py`
- `docs/model-trainer/v0.4.10-lazy-ray-padding-hook.md`

Validate with a unit test for lazy resolver installation, then run the detached
two-step preflight. Evidence must include successful Ray startup, initial
validation, real training metrics, and a Storage-published step-2 checkpoint.

**Contract impact:** None.
