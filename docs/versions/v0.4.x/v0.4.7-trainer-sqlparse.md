# v0.4.7 — Trainer Verifier Dependency Fix

**Status:** In progress

**Owner/worktree:** Codex integration owner, `fix/v0.4.7-trainer-sqlparse`

**Type:** D2 trainer runtime dependency bug fix

## Problem and Scope

The v0.4.6 preflight passed data filtering, loaded the LoRA model, and started
the vLLM server. The custom reward actor then failed to import
`trainer/grpo_reward.py` because its dependency chain reaches
`core.rewards.nl2sql`:

```text
ModuleNotFoundError: No module named 'sqlparse'
```

`sqlparse` was correctly declared for the app, but the GPU trainer uses its own
independent requirements file. Add the same lightweight verifier dependency to
`requirements-trainer.txt`, install it in the persistent RunPod venv, and
repeat the preflight.

## Non-goals

Do not add a second SQL implementation, change the verifier, pin a new CUDA
package, or alter the runtime stack. This is an omitted existing direct Python
dependency.

## Affected Areas and Validation

- `requirements-trainer.txt`
- `docs/model-trainer/v0.4.7-trainer-sqlparse.md`

Install with the trainer requirements on the pod, assert `import sqlparse`, and
run the detached two-step preflight through custom reward-worker creation.
