# v0.4.6 — Prompt Schema Token-Budget Fix

**Status:** In progress

**Owner/worktree:** Codex integration owner, `fix/v0.4.6-prompt-schema-budget`

**Type:** D2 dataset/runtime compatibility bug fix

## Problem and Scope

The v0.4.5 standard PPO entrypoint passed configuration validation, then
reported `dataset len: 40` followed by `filter dataset len: 0` and failed with:

```text
AssertionError: Train dataloader is empty!
```

Each NL2SQL prompt embedded the full execution schema, including all `INSERT`
fixtures. That is useful to the verifier but exceeds the D2 1024-token prompt
budget after chat templating, so verl correctly filters every training case.

Add a concise DDL-only prompt schema while retaining the complete schema with
fixture rows in each case's `schema_sql` for reward execution. Assert that
prompt text has no data inserts and stays within a conservative character
budget, then repeat the preflight.

## Decision

Codex separated model context from verifier state. The task asks a model to
write SQL from table names and columns; it does not need a duplicate copy of
the execution rows to form those queries. Keeping the full schema only in the
reward payload preserves deterministic expected-result scoring.

## Non-goals

Do not change the 50 reviewed questions, reference SQL, expected results,
train/validation split, execution schema, or the 1024-token D2 budget. Do not
raise the token limit merely to preserve redundant prompt data.

## Affected Areas and Validation

- `trainer/data/nl2sql_v1.py`

- `docs/model-trainer/v0.4.6-prompt-schema-budget.md`

Run data and full pytest, then the detached two-step preflight. The log must
