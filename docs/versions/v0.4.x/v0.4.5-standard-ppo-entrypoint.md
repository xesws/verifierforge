# v0.4.5 — Stable PPO Entrypoint Fix

**Status:** In progress

**Owner/worktree:** Codex integration owner, `fix/v0.4.5-standard-ppo-entrypoint`

**Type:** D2 trainer runtime compatibility bug fix

## Problem and Scope

After the v0.4.4 configuration fix, the real preflight loaded the Qwen 0.5B
LoRA model with SDPA on the L4, then failed in installed verl 0.8.0's
`main_ppo_sync.py`:

```text
KeyError: 'actor_rollout_ref'
```

The failure follows its LoRA/reference-role setup. `main_ppo_sync` selects an
`ActorRollout` worker when LoRA embeds the reference policy, but later
unconditionally looks up `ActorRolloutRef`. The standard
`verl.trainer.main_ppo` entrypoint in the same pinned installation contains the
missing `not self.ref_in_actor` guard and assigns the embedded reference worker
correctly. Switch the D2 launcher to that entrypoint; it uses the same Hydra
configuration, file logger, and `latest_checkpointed_iteration.txt` checkpoint
publication marker consumed by the Storage bridge.

## Decision

Codex made this compatibility judgment after comparing the installed source
paths. Disabling LoRA or KL merely to work around the sync-entrypoint defect
would change the reviewed D2 run. Using the standard entrypoint preserves the
specified LoRA rank 16 and KL coefficient 0.001 without adding a package patch
or changing the locked verl version.

## Non-goals

Do not change the data, model, LoRA, KL, rollout budget, checkpoint cadence,
or dependency versions. Do not patch vendored site-packages on the disposable
pod.

## Affected Areas and Validation

- `trainer/grpo_train.py`
- `tests/test_trainer_bridge.py`

Run focused and full local tests, then repeat the detached two-step preflight.
