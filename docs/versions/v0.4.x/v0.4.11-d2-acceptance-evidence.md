**Status:** Complete

**Owner/worktree:** Codex integration owner, `docs/v0.4.11-d2-acceptance-evidence`
**Type:** Documentation and acceptance-evidence patch

## Problem and Scope

D2 needs an auditable record of the real, remote GRPO smoke rather than a
claim based on unit tests or a fake trainer. This patch records the RunPod
preflight and 100-step kill/resume run, Storage publication, laptop sync,
local API response, and the related sandbox/Copilot/verifier validation.

## Non-goals

- No training algorithm, model, contract, dataset, or runtime dependency change.
- No quality, benchmark, or spurious-control claim from the 100-step smoke.
- No generated run/model/cache data, API key, host address, or key material in Git.

## Affected Areas and Contract Impact

- Documentation under `docs/model-trainer/`, `docs/infrastructure/`,
  `docs/evaluation-serving/`, `docs/backend-api/`, `docs/gpt-integrations/`,
  and `docs/verifiers/`, plus `README.md`.
- No API or shared-contract change.

## Observed Evidence — 2026-07-15

- **Source/runtime provenance:** D2 source spans `c346f15` through `1b6effb`.
  The pod used Python 3.12.3, torch 2.8.0+cu128, vLLM 0.10.2, verl 0.8.0,
  Ray 2.56.0, Transformers 4.57.6, and an NVIDIA L4 with driver 570.195.03.
  Python 3.12 is an intentional pod-runtime deviation from the repository's
  Python 3.11 target; local tests in this session used Python 3.13.5 and do
  not replace a dedicated 3.11 CI run.
- **Preflight:** `vf train d2preflight grpo_v1_0p5b_preflight` detached into
  tmux, ran real validation (initial validation reward 0.36, accuracy 0.20),
  bridged steps 1 and 2, atomically published
  `ckpt/step_2/global_step_2`, and emitted the final manifest and curve.
- **100-step kill/resume:** `d2demo` published step 50, then `vf kill d2demo`
  ended the tmux session. Restarting with the same command logged
  `Resuming d2demo from Storage checkpoint runs/d2demo/ckpt/step_50/global_step_50`.
  The first new bridged metric was step 53; the existing step-51/52 JSONL rows
  were retained without duplication. It completed with `Published Storage
  checkpoint step_100` and `Finished d2demo`.
- **Laptop handoff/API:** `vf watch d2demo` produced a local 100-line
  `metrics.jsonl` whose SHA-256 matched the pod exactly:
  `9aea9fb7a6ffea9d0463934c6e689020ab6b1c5f3e6cb2437b62d7b7d7537cf8`.
  Local `ckpt/` and `.verl-staging/` were absent; the local API on port 8010
  returned aligned 100-element Metrics arrays and marked `d2demo` done.
- **Local validation:** `pytest -q` passed 45 tests with 1 expected skip;
  `VF_RUN_DOCKER_INTEGRATION=1 pytest -q tests/test_docker_sandbox.py` passed
  7; focused Copilot/OpenRouter/data/verifier tests passed 28; and
  `bash -n scripts/vf` passed.

## Interpretation Boundary

This was an operational 100-step smoke. Its observed validation accuracy rose
from 0.20 at initial validation to 0.60 at final validation, but there was no
spurious-control experiment or repeated evaluation, so it is not evidence of a
real model-quality gain.
