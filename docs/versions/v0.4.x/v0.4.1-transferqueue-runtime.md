# v0.4.1 — TransferQueue Runtime Fix

**Status:** In progress

**Owner/worktree:** Codex integration owner, `fix/v0.4.1-transferqueue-runtime`

**Type:** D2 runtime dependency correction

## Problem and Scope

The first real `grpo_v1_0p5b_preflight` reached Ray and verl 0.8.0 but stopped
before model loading with:

```text
RuntimeError: transfer_queue is not installed. Cannot use tq.init().
Please install it by calling pip install TransferQueue==0.1.7
```

Pin `TransferQueue==0.1.7` in `requirements-trainer.txt` and install it into
the persistent `/workspace/verifierforge/.venv` on RunPod, then repeat the same
two-step `vf` preflight.

## Non-goals

Do not change CUDA, torch, vLLM, verl, Ray, model, GRPO hyperparameters, or
Hydra overrides. Do not disable the transfer queue merely to hide the failed
runtime initialization.

## Affected Areas and Validation

- `requirements-trainer.txt`
- `docs/model-trainer/v0.4.1-transferqueue-runtime.md`
- `docs/infrastructure/v0.4.1-transferqueue-runtime.md`
