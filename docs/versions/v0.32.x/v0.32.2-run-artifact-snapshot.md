# v0.32.2 — Run artifact snapshot

**Status:** complete — 2026-07-19
**Owner/worktree:** Codex on `main`
**Release type:** patch artifact publication

## Problem and scope

The owner and frontend teammate need the existing run evidence on another
machine and branch. This patch publishes the current `runs/` and `models/`
snapshot so metrics, logs, reports, manifests, sample evidence, and lightweight
model markers are available through GitHub.

## Non-goals

No training, provider request, contract change, database mutation, or generated
artifact rewrite. Multi-gigabyte `.safetensors` weights remain excluded because
ordinary GitHub Git rejects files over 100 MB and this checkout has no Git LFS;
their identity manifests remain in the snapshot. The reviewer sandbox invite
code is also excluded because it is an access credential.

## Affected areas and contracts

- `runs/`: current non-weight run evidence, force-added as a bounded snapshot.
- `models/`: current lightweight fake-model artifacts.
- `docs/frontend/` and `docs/infrastructure/`: consumption and publication
  notes.

There is no API or model contract impact. `.gitignore` remains unchanged so new
runtime output is still ignored unless explicitly reviewed and force-added.

## Validation plan

Scan all candidate text for credential-shaped content, verify no file selected
for ordinary Git exceeds GitHub's 100 MB limit, inspect the staged inventory,
run the repository secret scanner, commit, push `main`, and confirm the remote
HEAD. Stop rather than publish any secret or oversized model weight.

## Result

The reviewed snapshot contains 286 files under `runs/`, two files under
`models/`, and three version/area documents: 291 files and 36,385,007 bytes in
total. Credential scanning found zero matches, the committed database fixture
contains zero provider credentials, and no staged file reaches 100 MB. The two
ignored `.safetensors` shards and sandbox invite code remain local.
