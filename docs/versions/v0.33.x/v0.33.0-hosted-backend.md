# v0.33.0 — Hosted backend separation

**Status:** complete — 2026-07-20
**Owner/worktree:** Codex on `/Users/tangyiq/dev/verifierforge` `main`
**Release type:** minor hosted-delivery capability

## Problem and scope

The reviewer stack currently depends on laptop processes and an ephemeral
Cloudflare quick tunnel. This version separates the lightweight reviewer/API/
proxy service from GPU inference, packages it in a sub-1GB container, and
deploys exactly one always-on Railway Hobby service with a platform-provided
fixed subdomain. The old sandbox stays live until the new public contract
matrix passes.

## Non-goals

No trainer, dataset, weight, provisioner, DNS, domain purchase, paid LLM call,
or GPU lifecycle change. Railway receives secrets only through its secret
manager. GPU vLLM remains independently hosted and is never bundled into the
API image.

## Affected areas and contract impact

- Runtime dependency split, root container definition, and deploy scripts.
- Reviewer composition: top-level CORS/preflight and invitation boundary.
- Tuned-endpoint health/fallback behavior and deployment/rollback docs.

The 19 frontend operations and their response schemas remain frozen. Internal
health output may add a sanitized degraded indicator.

## Validation plan

Prove no ML imports, image size below 1GB, local Docker access to Supabase and
the configured tuned endpoint, all 19 operations, invitation auth, local and
Vercel CORS, endpoint-unavailable artifact fallback, and the same matrix from
the Railway public URL. Stop H3 unless Railway is publicly green.

## Result

The local gates passed in a 98,709,804-byte image with zero imported training
modules and `455 passed, 1 skipped`. The existing single Railway service was
reused, not duplicated, and accepted from its fixed public domain with 19/19
operations, invitation auth, API/proxy CORS, a complete flagship report,
twelve real tuned completions, and a new Guardian point. H3 then stopped the
obsolete laptop reviewer/tunnel while preserving vLLM. The RunPod-native
port-8000 URL still returns 404; endpoint rotation changes only
`VF_PROXY_TUNED_UPSTREAM`, with artifact/fake fallback on outage. Railway still
reports `trial`, so Hobby/usage-alert setup remains owner work.
