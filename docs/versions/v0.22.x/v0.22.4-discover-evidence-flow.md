# v0.22.4 — Discover evidence and approval flow

**Status:** complete — validated 2026-07-18
**Owner/worktree:** Codex, repository `main`
**Release type:** patch-level product/UI completeness fix

## Problem and scope

The feature-flagged Discover demo jumps from a hard-coded `95k monthly calls`
label directly to an Agent rationale, raw configuration JSON, and an ambiguous
`Approve & Forge` button. The Data Pull SQL path must instead show its evidence,
accept the configured traffic database as input, explain the proposed Forge
configuration, and state exactly what approval does.

The fixed demo facts are `95,000 SQL queries/month` and `$5,500/month`; `95k`
is volume, not cost. Input defaults to the configured `VF_PROXY_DB_PATH`, whose
repository default is `app/proxy/traffic.db`. Only Data Pull SQL receives the
expanded flow in this patch.

## Non-goals

Do not change core contract fields, Gate C scenarios or thresholds, the default
feature-flag state, training code, frozen data, provisioning, routing, or paid
LLM behavior. Approval remains an idempotent database record; it cannot create a
job, allocate a GPU, start training, charge a provider, or change traffic.

## Affected areas and interfaces

- Add real-API parity for the existing `Cluster` contract through `GET
  /clusters` and `GET /clusters/{cluster_id}`.
- Allow an optional `data_source` body on Agent analyze while preserving the
  existing bodyless request. Only the configured traffic database is accepted.
- Add `GET /agent-decisions/{decision_id}/approval` with the existing
  `ApprovalRecord` response.
- Move stable cluster profile values to one shared application catalog used by
  the real API, mock, and Agent fixtures.
- Replace the Data Pull SQL card's compressed result with an evidence-first,
  three-step Solution → Schema → Approve detail flow.

No existing response field is renamed or removed.

## Validation

Require real/mock cluster-shape parity, exact `95_000/$5_500` SQL metrics,
valid/invalid input-source coverage, bodyless Analyze compatibility, approval
idempotency and reload recovery, feature-flag-off behavior, and proof that no
execution-side module is invoked. Run the full pytest suite and a zero-cost mock
browser walkthrough at 390px and desktop widths. Preserve new overview, schema,
and approval-receipt screenshots with SHA-256 evidence.

## Result

The true and mock APIs now share one cluster catalog, and the Data Pull SQL flow
renders Evidence → Solution → Schema → Approve without extending a core
contract. `pytest -q` passed with `311 passed, 1 skipped`; desktop and 390px
browser walkthroughs passed against the mock binding. Approval persisted and
reloaded through the store-only route, with no trainer, provisioner, GPU, paid
LLM, routing, or job-creation call.

Evidence is stored in `assets/forge-agent/v0.22.4-discover-overview.png`,
`v0.22.4-discover-schema.png`, and
`v0.22.4-discover-approval-receipt.png`; hashes are recorded in
`docs/p0-run-sheet.md`.
