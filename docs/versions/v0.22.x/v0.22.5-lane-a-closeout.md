# v0.22.5 — Lane A endpoint closeout and repository cleanup

**Status:** complete — 2026-07-18
**Owner/worktree:** Codex, repository `main`
**Release type:** patch-level serving proof and compatibility cleanup

## Problem and scope

The serving lane has a locally healthy step-350 endpoint but no accepted public
proof, and four old workspace paths were never dispositioned. This wave first
checks the existing `vfserve` loopback endpoint through an SSH tunnel, then
sends at most one public completion attempt with SDK retries disabled. A public
success unlocks an isolated 200-request SQL canary proof whose route is always
reset to zero.

The old `freeze_nl2sql.py` name is retained only as a deprecated compatibility
entrypoint delegating to the authoritative three-piece freezer. It must not
retain the obsolete lexicographic projection or two-file freeze semantics.

## Non-goals

Do not alter model weights, call a paid LLM or GPU
provider, change frozen data, modify trainer code, or relax any public or
guardian gate. A failed local or public serving check becomes `OWNER-ACTION`
and skips canary traffic without blocking later database work.

## Affected areas and interfaces

- Add a bounded public endpoint proof using `VF_ENDPOINT_BASE_URL`,
  `VF_ENDPOINT_API_KEY`, and a model ID discovered from `/v1/models`.
- Give an HTTP tuned upstream its own `VF_PROXY_TUNED_API_KEY`; never borrow a
  copilot/evaluator key.
- Allow `traffic_gen` to emit a single request family for the SQL-only canary.
- Replace the untracked legacy freezer with a compatibility wrapper, delete the
  superseded vLLM execution sheet, and ignore/remove downloaded `tq/` wheels.
- Resolve `.env` defaults without mutating the process environment. Test-time
  LLM settings must not leak `VF_DB_BACKEND=postgres` into later API tests and
  thereby write fixture routing state into the real Supabase database.

No existing API response field changes.

## Validation plan

Require local endpoint model discovery and one completion before the public
request. The public client uses zero retries and a 20-second timeout. If public
serving passes, require 200 successful SQL requests, 40--60% tuned routing, new
guardian points with final pass rate at least 0.5, and a verified canary reset
to zero. Run focused proxy/traffic/freeze tests, the complete pytest suite,
`bash -n scripts/vf`, secret scans, and finish with a clean worktree.

## Result

The original non-invasive SSH preflight failed before model discovery with
`ssh: connect to host 157.157.221.29 port 43314: Operation timed out`.
The owner later supplied a replacement L4 pod. Codex installed the pinned
serving stack, transferred the canonical 13-file step-350 export, and matched
tree SHA-256 `7bde853af7c82405fd1356de9bad9b6c421de45a45ce747f63ea2f8a27eda658`.
Loopback model discovery and SQL completion passed.

The replacement pod exposed only port 8888 through RunPod; the required native
8000 proxy URL returned HTTP 404 even though vLLM listened on `0.0.0.0:8000`.
Codex made the explicit operational judgment to use an outbound Cloudflare
quick tunnel rather than request another pod rebuild. The official SDK public
proof returned `SELECT name FROM users;` from the same step-350 bytes.

The 200-request SQL-only canary completed with zero failures: 120 default and
80 tuned requests, a 0.40 tuned fraction within the fixed 0.40--0.60 gate.
Thirteen new Guardian points had a final LivePassRate of 0.85. The route was
then reset to zero; 20 additional requests produced 20 default and zero tuned
routes. These results permit the `lane-a-closeout` tag.

The repository cleanup completed. The obsolete freezer is now a 23-line
deprecated delegate to the U3 three-piece implementation; its old projection
and manifest builders are absent. The downloaded TransferQueue wheel and the
superseded destructive vLLM execution sheet were removed, `/tq/` is ignored,
and the bounded endpoint proof plus dedicated tuned-key/single-family seams are
ready for a later retry. Focused validation passed `19 passed`; the complete
suite passed `366 passed, 1 skipped`; `bash -n scripts/vf` passed.

The first final-suite run exposed a canary-safety regression: shared LLM
settings loaded the repository `.env` by mutating `os.environ`, so a later API
fixture wrote 50% routing state into real Supabase. Codex classified this as a
release blocker, reset canary to zero, and changed both LLM and Gate C settings
resolution to merge dotenv defaults into a local mapping. Focused regression
passed `48 passed`; after the full suite, a direct database query and the API
both still reported `canary_percent=0`.
