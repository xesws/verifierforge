# v0.22.5 — Lane A endpoint closeout and repository cleanup

**Status:** complete with public endpoint OWNER-ACTION — 2026-07-18
**Owner/worktree:** Codex, repository `main`
**Release type:** patch-level serving proof and compatibility cleanup

## Problem and scope

The serving lane has a locally healthy step-350 endpoint but no accepted public
proof, and four old workspace paths were never dispositioned. This wave first
checks the existing `vfserve` loopback endpoint through an SSH tunnel, then
sends at most one public completion attempt with SDK retries disabled. A public
success unlocks an isolated 200-request SQL canary proof whose route is always
reset to zero.

The old `freeze_nl2sql.py` name is retained only as a deprecated compatibility
entrypoint delegating to the authoritative three-piece freezer. It must not
retain the obsolete lexicographic projection or two-file freeze semantics.

## Non-goals

Do not restart the serving process, alter model weights, call a paid LLM or GPU
provider, change frozen data, modify trainer code, or relax any public or
guardian gate. A failed local or public serving check becomes `OWNER-ACTION`
and skips canary traffic without blocking later database work.

## Affected areas and interfaces

- Add a bounded public endpoint proof using `VF_ENDPOINT_BASE_URL`,
  `VF_ENDPOINT_API_KEY`, and a model ID discovered from `/v1/models`.
- Give an HTTP tuned upstream its own `VF_PROXY_TUNED_API_KEY`; never borrow a
  copilot/evaluator key.
- Allow `traffic_gen` to emit a single request family for the SQL-only canary.
- Replace the untracked legacy freezer with a compatibility wrapper, delete the
  superseded vLLM execution sheet, and ignore/remove downloaded `tq/` wheels.

No existing API response field changes.

## Validation plan

Require local endpoint model discovery and one completion before the public
request. The public client uses zero retries and a 20-second timeout. If public
serving passes, require 200 successful SQL requests, 40--60% tuned routing, new
guardian points with final pass rate at least 0.5, and a verified canary reset
to zero. Run focused proxy/traffic/freeze tests, the complete pytest suite,
`bash -n scripts/vf`, secret scans, and finish with a clean worktree.
