# v0.22.0 — Flag-contained Forge Agent product integration

**Status:** complete; production flag remains disabled
**Owner/worktree:** Codex, repository `main`
**Stage:** Forge Agent A-4

## Problem and scope

The repository contains no frontend application, so satisfy the Discover-card
DoD with a small FastAPI-hosted vanilla HTML/CSS/JS page rather than introducing
an npm/Next stack. Add analyze/latest/approval endpoints using separate additive
contracts. Approval writes one auditable SQLite record and performs no execution.

`VF_AGENT_ENABLED` is false unless its normalized value is exactly `true`.
While false, every new route returns 404 before constructing LLM, S3, or store
dependencies. The existing reviewer path and Cluster responses remain unchanged.

## Cache and gate behavior

An existing completed decision is reused only when the canonical traffic
fingerprint is byte-identical. The real LLM binding additionally requires an
explicit Gate C deployment receipt; this run has no such receipt because Luna
live-eval was incomplete. The local demo therefore uses deterministic mock
LLM/tools with the real SQLite/S3 audit path and makes no paid LLM request.

## Approval boundary

Only completed `forge` decisions with a durable trace key may be approved.
Approval is idempotent by decision ID and stores `approved_by/approved_at` only.
There is no trainer, provisioner, shell, GPU, or job launch call.

## Validation plan

Test flag-off zero construction, flag-on analysis, exact-fingerprint caching,
cache invalidation, approval eligibility/idempotence, mock/real response-shape
identity, API failure mapping, UI presence, and static execution-side import
prohibition. Run a local browser demo and commit a tracked screenshot only if the
full click-to-approval path succeeds.

## Result

The flag-contained API, deterministic mock binding, exact-fingerprint cache,
SQLite approval store, and dependency-free Discover page are complete. A local
mock-binding walkthrough exercised Analyze through Approve: the durable trace
was written to S3 before the decision summary, the approval was stored in
SQLite, and the browser rendered the idempotent approved state. The tracked
screenshot is `assets/forge-agent/v0.22.0-analyze-approve.png` (SHA-256
`521c34d70e3a5dbe566334ceecad76690ca502e5881dec390d32490b33e7cbc4`).

Validation passed 294 tests with one expected skip. No paid LLM request,
training launch, provisioner call, or GPU action occurred. Because formal live
Gate C remains incomplete, `VF_AGENT_ENABLED` is still false by default and the
real binding additionally requires `VF_AGENT_GATE_C_PASSED=true`.
