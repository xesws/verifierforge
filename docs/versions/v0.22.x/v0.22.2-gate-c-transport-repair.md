## Problem and scope

The v0.22.1 live run preserved only the exception class, so 12 requests failed
without an actionable HTTP status or provider body. Preserve a bounded provider
error payload in `LLMRequestError`, reproduce one scenario, then run a minimal
Chat Completions/Responses function-calling matrix. Select the OpenAI transport
from that evidence while leaving the OpenRouter Chat Completions path unchanged.

Add a mandatory two-request live-eval preflight: one plain response and one
function-call response. A failed preflight must stop before the frozen
12-scenario batch.

## Non-goals

Do not change scenario data, Gate C thresholds, tool semantics, public Agent
contracts, feature-flag state, OpenRouter behavior, training code, or the
Sol/Terra prohibition. Do not remove function calling to accommodate a legacy
transport.

## Validation and stop conditions

First run exactly one scenario and record its HTTP status/provider body. Then
issue three probes: Chat Completions plain, Chat Completions with one dummy
function, and Responses with the same function. Add the bare `gpt-5.6` Chat
Completions tool probe only if authenticated `/v1/models` lists that exact ID.

Run focused tests and `pytest -q`. Before the formal batch, require both the
plain and function-calling preflight probes to pass. If they fail, emit no batch
requests. If the batch runs, apply unchanged thresholds and create
`agent-gate-c-pass` only when all four metrics pass. Never enable the feature
flag automatically.

## Result

G1 preserved the exact HTTP 400 and provider body in both the raised
`LLMRequestError` and durable Agent trace. The three-probe matrix showed plain
Chat Completions passing, Chat function calling failing, and Responses function
calling passing. Accordingly, OpenAI Agent turns now use Responses while
OpenRouter remains on Chat Completions. Responses continuations use
`previous_response_id` plus `function_call_output`, preserving reasoning context.

The mandatory two-probe flight check is now part of the evaluator, with a test
proving a failed probe constructs zero scenario runners. Both production probes
passed before the second formal batch.

The unchanged live Gate C tuple was
`0.6666666666666666 / 0.8333333333333334 / 0 / 0.5` (decision accuracy /
chain success / illegal actions / config legality). Ten scenarios produced
evaluable completed traces; two were durably rejected by dependency guards and
therefore appeared as missing traces in the current report. No retry, scenario,
or threshold change occurred. No admission tag was created and the feature flag
remains off. Validation passed 305 tests with one expected skip.
