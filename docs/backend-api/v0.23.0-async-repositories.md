# v0.23.0 — Async API repositories

FastAPI routes and the OpenAI-compatible proxy switch to injected async
repositories backed by one SQLAlchemy session factory. Existing route paths and
response models do not change. Artifact mode remains file-only and independent
of database availability.

Local SQLite is initialized exclusively through Alembic, not ad-hoc `CREATE
TABLE` calls. Postgres configuration errors return a stable sanitized 503; the
proxy still returns its upstream completion when best-effort accounting or
guardian persistence fails.

Jobs remain sourced from local `runs/` in SQLite development mode but are
materialized through `JobStore`; database summaries cap UI metric points while
raw JSONL stays in Storage.

Synchronous HTTP and Agent entrypoints cross into the async stores through one
long-lived `RepositoryGateway`. Database-dependent control routes return a
sanitized 503 on failure; proxy accounting and guardian writes remain
best-effort, and static cluster/file-backed job reads remain available.
