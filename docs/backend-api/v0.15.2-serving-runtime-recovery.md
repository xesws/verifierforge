# v0.15.2 — proxy handoff to the verified endpoint

No API or contract field changes are made. The physical endpoint is supplied
only through `VF_PROXY_TUNED_UPSTREAM`; the existing logical `tuned` route and
`RoutingState` API stay unchanged. The proof uses the API to set the
`data-pull-sql` cluster to 50%, examines persisted `route_path` values and
guardian output, then restores `canary_percent` to zero.

All traffic is synthetic and local to the product path. The proxy must remain
available if the guardian or tuned upstream fails.
