# v0.13.0 — Product traffic proxy

`app/proxy/` is deliberately separate from `app/api/`: it exposes only
`POST /v1/chat/completions` and does not alter job/report contracts. It accepts
the OpenAI request body, forwards all request fields unchanged in real mode,
and returns the upstream JSON/status unchanged. Fake mode is the default and
uses a content-hash-derived response so development and tests never spend money
or require networking.

The local `traffic.db` schema stores only operational metadata: UTC timestamp,
system-prompt SHA-256, model, input/output token counts, latency milliseconds,
and estimated USD cost. Price lookup comes from `config/proxy_pricing.json`,
with a default rate and optional per-model override. SQLite writes are strictly
best effort: forwarding succeeds even if the database is locked or unavailable.

Validation proves request/response shape, fake determinism, price calculation,
metadata recording, and the non-blocking failure path. `traffic.db` and SQLite
sidecar files are ignored by git.

P1 completed with those tests green. Fake mode was the only executed upstream;
real-mode validation used an injected forwarding seam and canonical
`VF_LLM_BASE_URL`/`VF_LLM_API_KEY` configuration, so no paid request occurred.
