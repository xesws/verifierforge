# v0.31.0 — Backend support for frontend API v1

The real API gains queued metadata submission through `POST /jobs` and returns
those rows through the existing Job shape; submission does not train or
provision. Cluster list/detail responses aggregate the latest persisted Agent
decision, routing switch and Guardian points into existing/additive fields.

The mock must implement every frozen action and read endpoint with identical
request/response keys. Contract parity is tested from OpenAPI and actual
responses rather than asserted only in prose.

Implementation result: DB-submitted jobs round-trip through the full `Job`
contract and remain metadata-only. Cluster list/detail combines the latest
valid Agent decision, route and LivePassRate without making a malformed audit
row take Discover down. Hybrid reviewer mode keeps immutable D4 jobs while
reading product state and newly submitted metadata from the configured
repository. The focused contract suite passed 11 tests; the full suite passed 433
with one credential-gated skip.
