# v0.39.1 — serving sleep API

`POST /serving/sleep` is the explicit inverse of Wake for the reviewer demo.
It requires the invitation, accepts only a stable `model_id`, and returns
`ServingStatus` after the endpoint is durably cold.

The operation is idempotent. Missing/already-cold registry state performs no
provider call. Active state is serialized through the coordinator, transitions
through draining, proves provider deletion and managed-inventory zero using the
existing runtime, then persists cold. Provider errors are sanitized and return
a non-success response; callers must not clear their local session on failure.

A real persisted provider handle may not be drained by a process configured
with the mock binding. That mismatch is rejected before any registry mutation;
this prevents a local process connected to production Supabase from recording
a fake cold state.
