# v0.27.0 — Cluster sample-source API

Add authenticated-style repository controls (within the current local product
boundary) to read and attach a cluster's approved sample source:

- `GET /clusters/{cluster_id}/sample-source`
- `PUT /clusters/{cluster_id}/sample-source`

PUT accepts source metadata, validates a repository-relative JSONL file beneath
the allowed data root, recomputes SHA-256 and row count, then stores metadata on
the cluster. It never stores sample bodies. Missing clusters, path traversal,
non-JSONL content and identity mismatches fail before persistence.

Agent Analyze remains behind `VF_AGENT_ENABLED` and Gate C. The cache key adds
the approved source identity. Existing cluster fields, mock shapes and reviewer
behavior remain additive and backward compatible.

Result: both endpoints are live behind the Agent flag, the Discover Input flow
attaches the source before Analyze, and `force_refresh` exists only as an
explicit evidence-run option. Normal product calls continue to cache by the
combined traffic, source-identity and execution-profile fingerprint.
