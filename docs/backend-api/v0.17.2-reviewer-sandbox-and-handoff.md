# v0.17.2 — reviewer sandbox API boundary

The sandbox composes existing components without changing their public shapes:
the FastAPI API reads committed `data/demo-artifacts/` in read-only artifact
mode and the proxy uses its deterministic `fake` upstream. Reviewers can query
`/jobs`, `/jobs/{id}/metrics`, and `POST /v1/chat/completions` locally without
an API key or model-provider request.

No routing mutation, guardian claim, live traffic database, or real tuned
endpoint is exposed. The sandbox must reject a conflicting port rather than
silently attaching to an unrelated process, and its smoke test validates the
two local endpoints independently.

**Implemented result:** `scripts/start_reviewer_sandbox.sh` launches both
existing ASGI apps with the fixed local-only modes. Its integration test checks
the committed D4 jobs and the fake completion response, while a separate test
proves identical API/proxy ports are rejected before either server starts.
