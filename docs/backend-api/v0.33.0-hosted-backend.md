# v0.33.0 hosted backend

**Status:** packaged, deployed, and publicly accepted — 2026-07-20

The deployable service is `app.reviewer.main:app`: one Uvicorn process exposing
the frozen API, OpenAI-compatible proxy, invitation-protected demo surface, and
Guardian side work. The container must not import torch, vLLM, verl, Ray, or
Transformers.

Top-level CORS must cover both `/` API routes and `/proxy`, allow configured
local/Vercel origins, and let unauthenticated OPTIONS preflight pass while
retaining Basic Auth on actual product requests. If the tuned endpoint is
unavailable, report reads remain complete from artifacts and proxy traffic uses
the deterministic fallback without exposing a 500 page.

Validation uses the 19-operation contract matrix plus proxy, Guardian, auth,
CORS, and health probes locally and on the final public URL.

H0/H1 result: runtime import and container probes found no training modules;
top-level unauthenticated preflight works for both mounted apps while actual
calls remain invitation-protected. Tuned connection/HTTP failure changes the
recorded path to `default-fallback`, suppresses Guardian scoring for that
response, and reports degraded health without breaking Job/report reads.

The production topology is frontend (Vercel) → reviewer API/proxy (Railway) →
Supabase/S3 and the independently hosted GPU vLLM endpoint. The API container
contains no GPU runtime. The frozen 19-operation surface and Basic-auth boundary
are unchanged by hosting. `VF_AUTOPROVISION=false` and mock Agent/provisioner
bindings keep the public reviewer unable to spend.

Public origin: `https://verifierforge-production.up.railway.app`. Railway maps
the domain to its injected `PORT=8080`; the service remains one replica with
Serverless disabled. The public 19-operation/auth/CORS/proxy matrix matches the
local container result.
