# v0.37.0 demo traffic API

The reviewer API adds an invitation-protected, feature-flagged demo extension:
`POST /demo/traffic` starts one bounded mixed replay and
`GET /demo/traffic/status` reports its live counters. The local request model
defaults to 200 requests at 5 requests per second, caps total at 500 and rate at
20, forbids non-positive/non-finite rates, and exposes only sanitized task
errors.

The process reserves the singleton task under a lock before starting a daemon
thread. A concurrent POST returns the active status instead of starting another
thread. Status is intentionally in memory: the production topology is one
Railway replica and one Uvicorn worker, and a restart returns to idle.

`traffic_gen` remains the source of the equal three-family request stream. The
API calls the same in-process completion dispatcher used by
`/proxy/v1/chat/completions`; it does not start a subprocess, call itself over
HTTP, forward the invitation secret, or duplicate proxy decisions. The
extension is disabled by default with `VF_DEMO_TRAFFIC_ENABLED=false`.

Local API and generator coverage passes for invitation and flag boundaries,
input limits, singleton adoption, live counters, sanitized fatal failure, and
direct entry through the production dispatcher.

Production authenticated acceptance returned 202 for the new cold run and 200
for a duplicate start while preserving the original 200/5 task. Cold and ready
runs each finished at `sent=200`, `success=200`, `failed=0`, `running=false`.
Anonymous status remained 401, flag-off authenticated status was 404, and the
enabled final status plus Vercel-origin CORS preflight returned 200.
