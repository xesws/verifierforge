# v0.32.3 backend report reconciliation

**Status:** complete — 2026-07-20

`GET /jobs/{id}` must return the complete D4 presentation in artifact, hybrid,
and Supabase modes: before/after, main/control curves, ten held-out arena
samples, projected savings, and verdict. The arena is generated only from the
frozen held-out 60-row set and the aligned M5 baseline/step-350 sample evidence.

The public operation ledger becomes 19 operations by freezing the three
already-implemented reads for Agent decision, approval, and job metrics.
Internal discovery, copilot, OpenAPI, and health routes remain available but
are explicitly outside the frontend contract.

Validation covers response-model round trips, missing-ID behavior, real/mock
OpenAPI parity, and complete report parity for all public data modes.

The public modes are `artifacts`, `hybrid` (default), and `supabase`. The old
`runs` mode remains a deprecated internal compatibility path for the local fake
trainer. Hybrid reads the flagship report from immutable artifacts first and
uses Supabase for dynamic relational facts; Supabase mode is strict and returns
only stored deterministic projections.
