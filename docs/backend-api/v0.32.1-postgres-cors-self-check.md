# v0.32.1 — Postgres API and CORS self-check

The real API will run with `VF_DB_BACKEND=postgres`, deterministic mock Agent,
disabled autoprovision, and hybrid product data. The 16 frozen operations will
be exercised over HTTP and their bodies validated against the public Pydantic
models. Direct repository readback and an intentionally unused SQLite sentinel
path will distinguish Supabase-backed state from artifact, static, and mixed
responses.

CORS will read `VF_CORS_ORIGINS`. Its default is the six HTTP origins formed by
localhost/127.0.0.1 and ports 3000/5173/8080; an explicit comma-separated value
replaces that list, and only an explicit `*` restores wildcard access. The
Start Forge flag remains default-off and must return a clear disabled detail
without reaching persistence or a provider.

Validation covers actual preflight responses, hostile-origin rejection, the
frozen OpenAPI parity suite, Postgres repository contracts, and the full test
suite. No public success payload changes in this patch.

Implementation result: real and mock now share the environment-resolved CORS
policy. Local ports 3000/5173/8080 on both localhost and 127.0.0.1 returned
200 and echoed the exact origin; an untrusted origin returned 400 without an
allow-origin header. Disabled Start preserved status 404 but now names the
disabled flag and exits before repository/provider access.
