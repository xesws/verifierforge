# v0.26.0 — Database failure contract

The API exposes no database URL or driver message. Repository failures map to
stable service-unavailable responses, while artifact reviewer mode stays
database-independent. The proxy's primary completion path remains available
when traffic/guardian writes fail, but a sanitized operational error is
emitted so data loss is never silent.

No HTTP response schema changes in this version.

Verified behavior: a routing-store disconnect returns
`{"detail":"Routing state is unavailable"}` with HTTP 503. A proxy database
initialization failure logs `proxy database initialization failed` without the
driver text and still returns the deterministic upstream completion.
