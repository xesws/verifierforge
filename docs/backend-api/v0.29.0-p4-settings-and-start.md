# v0.29.0 — P-4 settings and Start Forge API

This wave adds write-only provider settings plus a Start Forge boundary for an
already-approved Agent decision. Approval and Start remain distinct requests.
The API returns only identifiers, provider names, credential presence/source,
and lifecycle state. It never serializes a credential or provider exception
containing request authorization.

The new response shapes are additive and will be frozen for frontend consumers
in v0.31.0 after the implementation and mock/real shape tests pass.
