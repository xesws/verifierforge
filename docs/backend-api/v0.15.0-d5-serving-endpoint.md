# v0.15.0 — Proxy endpoint handoff

No API contract changes are required. `RoutingState.target_model` remains the
logical `tuned` label; the proxy resolves that label through the operator's
`VF_PROXY_TUNED_UPSTREAM` environment setting. Replacing the fake physical
target with the RunPod public base URL is therefore the promised one-line
handoff.

The routing API is used to set `data-pull-sql` to 50% for the proof and back to
0% afterward. Existing traffic rows retain `route_path`, and existing
`LivePassRate` endpoint responses retain the shared contract shape.
