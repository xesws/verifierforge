# v0.22.0 — Agent and approval API

New response contracts are independent of `Cluster`, preserving all existing
mock and real API fields while `VF_AGENT_ENABLED=false`. Unknown/missing
decisions return 404, ineligible approvals return 409, guard failures return 422,
and model/persistence failures return 503/502 without provider bodies.

The SQLite approval table is an interim implementation of the design document's
`ApprovalStore`; the later DB wave may replace the backend without changing the
route models.

The real binding is fail-closed behind both `VF_AGENT_ENABLED=true` and
`VF_AGENT_GATE_C_PASSED=true`. The mock binding is intended only for the
flagged demo and contract tests; it still exercises the production SQLite and
S3 persistence boundary.
