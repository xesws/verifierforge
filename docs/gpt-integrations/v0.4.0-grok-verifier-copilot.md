# v0.4.0 — GPT Integrations: Grok Verifier Copilot

## Scope

Verifier Copilot uses the existing OpenRouter adapter with the explicit
`x-ai/grok-4.5` model, never a `~openai/gpt-latest` alias or direct OpenAI call.
It asks for structured JSON in five batches of ten NL-to-SQL seed candidates and
can make one repair request for malformed output. `OPENROUTER_API_KEY` is read
only from the laptop process environment and is never copied to RunPod.

## Review Boundary

The API returns proposal text, rubric, tests, cases, selected model ID, and
sandbox validation metadata. It does not auto-write a verifier or dataset; a
human reviews the candidates before committing or training.

## Validation

Injected-client tests verify the selected model and structured-output handling.
No test requires a key or a live provider request.

## Acceptance Evidence — 2026-07-15

The focused test suite passed with an injected OpenRouter client and verified
the explicit model/structured-output behavior. The real D2 GPU run used the
committed V1 fixture, not a generated provider response; no live OpenRouter
request is represented as evidence here.
