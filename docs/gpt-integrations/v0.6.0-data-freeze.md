# v0.6.0 — shared augmentation-model client

**Status:** in progress. Linked plan: `docs/versions/v0.6.x/v0.6.0-data-freeze.md`.

## Decision

All external completions for D3 use one OpenAI SDK client configured from
`VF_LLM_BASE_URL`, `VF_LLM_API_KEY`, and `VF_AUGMENT_MODEL`. Its central
configuration defaults are `https://openrouter.ai/api/v1` and
`z-ai/glm-5.2`; the local `.env` is loaded without overriding exported shell
values and may override them, while callers never embed a model or provider
URL. The client must accept an injected transport/client in tests so tests do
not make network calls.

## Secret and failure policy

`.env` is ignored and never copied to the pod, committed, logged, or included
in exception text. Missing key/model configuration fails before a request with
a field-name-only diagnostic. HTTP failures identify status/category but redact
authorization values and prompts. No provider-specific request format or
headers are added.

## Validation

Unit tests verify defaults, explicit environment overrides, missing-key errors,
and that all callers receive the same configured client. A live augmentation
run is allowed only after those tests pass and reports counts, never secrets or
raw credentials.
