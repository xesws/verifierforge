# v0.27.0 — Production Analyze evidence run

The approved-sample rollout uses exactly two fresh product Agent runs through
the real binding and the provider-discovered `gpt-5.6-luna` model. The first run
has no approved source and must preserve a complete `need_more_data` trace. The
second follows a validated source attachment and may be approved only if it
returns `forge` with the strict `p2_gate_b` configuration.

The run entrypoint forces a fresh decision without changing normal UI cache
behavior. It authorizes USD 0.25 per Agent run against the durable existing cost
ledger, records success or failure once, and never retries. The total new
reservation is therefore capped at USD 0.50. OpenAI is used only for this
production evidence; development and tests remain mock/OpenRouter-safe.

No key, request body, connection URL or sample body is written to logs. Full
Agent traces use the existing S3 trace store, while the run artifact contains
only decision IDs, source identity, public config, token totals and ledger
receipts.

Result: both one-shot runs passed. The source-less run used 6,860 input / 515
output tokens and chose `need_more_data`; the source-backed run used 7,528 input
/ 514 output tokens and chose `forge`. Each was charged at its `$0.25`
reservation upper bound because provider-reported cost was absent. No retry,
Sol, Terra, OpenRouter, or extra paid request was used.
