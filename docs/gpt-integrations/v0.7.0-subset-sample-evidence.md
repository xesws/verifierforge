# v0.7.0 — diagnostic eval-client isolation

**Status:** in progress. Linked plan:
`docs/versions/v0.7.x/v0.7.0-subset-sample-evidence.md`.

The sample-evidence diagnostic uses the existing `EvalSettings` boundary only.
It requires `VF_EVAL_BASE_URL` and `VF_EVAL_MODEL`, does not load `.env`, and
cannot inherit `VF_LLM_*` or OpenRouter credentials. The missing eval key uses
the local non-secret SDK placeholder.
