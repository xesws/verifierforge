# v0.6.1 — bounded OpenRouter augmentation operations

**Status:** in progress. Linked plan: `docs/versions/v0.6.x/v0.6.1-p0-data-freeze-execution.md`.

S1 uses the existing shared OpenAI-compatible client configured only by the local environment. The live command never embeds a base URL, model slug, or key; it therefore uses the configured OpenRouter model. `.env` stays on the laptop and is never printed, committed, or transferred to RunPod.

Each seed receives one structured completion request, with exactly one immediate retry only when JSON decoding or the required `variants` shape fails. A second malformed response drops that seed. Transport, authentication, and configuration failures fail the operation closed. The 30-minute deadline prevents starting another seed request but permits an already-started request to finish.

The committed summary contains counts and IDs needed for deterministic recovery, never raw provider bodies, prompts beyond the admitted dataset, or credentials.
