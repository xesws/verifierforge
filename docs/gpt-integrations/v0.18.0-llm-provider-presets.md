# v0.18.0 — OpenRouter/OpenAI presets

`VF_LLM_PROVIDER` selects `openrouter` (default) or `openai`. OpenRouter resolves
to `https://openrouter.ai/api/v1`, `OPENROUTER_API_KEY`, and
`z-ai/glm-5.2`. OpenAI resolves to `https://api.openai.com/v1`,
`OPENAI_API_KEY`, and `gpt-5.6-luna-xhigh`.

`VF_LLM_API_KEY`, `VF_LLM_BASE_URL`, and `VF_LLM_MODEL` remain explicit generic
overrides. `VF_AUGMENT_MODEL` is retained behind `VF_LLM_MODEL` for existing
augmentation and Copilot callers. The OpenAI policy rejects any resolved model
containing `sol` or `terra`; Forge Agent Gate C additionally requires the exact
Luna slug. Keys remain hidden from repr, errors, logs, and evidence.

The shared client grows an additive tool-turn API carrying structured tool calls
and token usage. Existing text and JSON methods keep their current behavior.
Live smokes are single, short calls with no raw response persisted.

The 2026-07-17 live check did not pass: OpenRouter returned an empty usable
completion and the Luna slug returned HTTP 404 from the OpenAI preset. These
are provider/model availability facts, not authorization to change the pinned
model. Later stages therefore use injected deterministic clients; Gate C live
remains unavailable until the exact Luna slug is accepted by its configured
endpoint.
