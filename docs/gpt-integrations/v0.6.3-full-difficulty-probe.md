# v0.6.3 — probe evaluation-client boundary

**Status:** implementation validated; pod launch pending O1/O2. Linked plan:
`docs/versions/v0.6.x/v0.6.3-full-difficulty-probe.md`.

The difficulty probe reuses only `EvalSettings`: `VF_EVAL_BASE_URL` and
`VF_EVAL_MODEL` are mandatory, while the absent eval key resolves to the
non-secret local placeholder. It does not load `.env` and cannot fall back to
`VF_LLM_*`, `VF_AUGMENT_MODEL`, or an OpenRouter credential.

The pod command clears those variables explicitly as a defense-in-depth check.
