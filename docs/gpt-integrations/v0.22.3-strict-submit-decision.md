# v0.22.3 — Strict Responses terminal decision

T1 found no formatting failure, so this is preventive protocol hardening rather
than a claimed root-cause fix. `submit_decision` remains a function call and
uses the OpenAI SDK's strict schema derived directly from the Pydantic contract;
the same Pydantic model parses the returned arguments. Parser failures retain
the complete bounded validation evidence.

The strict schema requires `additionalProperties: false`, marks every property
required, and represents optional `config` as nullable. This follows the
official Responses/function-calling strict-mode contract and avoids maintaining
a second handwritten decision schema. Reference:
`https://developers.openai.com/api/docs/guides/function-calling#strict-mode`.

No plain-text or JSON-prompt fallback is allowed. OpenRouter remains on Chat
Completions and OpenAI remains on Responses.
