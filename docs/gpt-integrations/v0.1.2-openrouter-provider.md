# v0.1.2 — GPT Integrations: OpenRouter Provider

## Provider Contract

`app/gpt/` will use the OpenAI Python SDK against `https://openrouter.ai/api/v1`, with `OPENROUTER_API_KEY` as the required credential. `VF_GPT_MODEL` selects the OpenRouter model slug; `x-ai/grok-4.5` is the explicit reproducible default. Optional `VF_APP_URL` and `VF_APP_TITLE` supply OpenRouter attribution headers.

Reference the [OpenRouter quickstart](https://openrouter.ai/docs/quickstart) and [OpenAI SDK integration guide](https://openrouter.ai/docs/guides/community/openai-sdk) when changing this adapter.

## Behavior and Failure Rules

Client construction fails clearly when the key is absent. A completion returns text or raises a provider-specific error for an empty response; it must not silently fall back to a direct OpenAI endpoint. The initial adapter is synchronous and non-streaming because no product caller exists yet.

## Validation

Tests use a fake injected OpenAI-compatible client and do not contact OpenRouter. A human-operated smoke test, after setting `OPENROUTER_API_KEY` outside Git, verifies a selected model slug and records only the status/model metadata.

Current automated validation: `pytest -q` passes without a key or network call. Before merge, record the model slug and non-secret outcome of one operator-run OpenRouter request here.
