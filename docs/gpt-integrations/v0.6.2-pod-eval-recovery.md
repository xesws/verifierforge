# v0.6.2 — eval-only client configuration

**Status:** in progress. Linked plan: `docs/versions/v0.6.x/v0.6.2-pod-eval-recovery.md`.

Gate A obtains endpoint and model exclusively from `VF_EVAL_BASE_URL` and
`VF_EVAL_MODEL`; neither may fall back to the augmentation/copilot variables.
It does not load `.env`. `VF_EVAL_API_KEY` is optional for authenticated future
endpoints; otherwise the OpenAI SDK receives the non-secret `vf-local-eval`
placeholder needed by an unauthenticated loopback vLLM server.

The existing generic OpenAI-compatible transport is reused, but provider
exceptions retain their Python cause and safe structured metadata. Error body
evidence is redacted and capped at 4 KiB; terminal output never contains a
credential, prompt, Authorization header, or raw provider body.
