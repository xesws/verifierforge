# v0.12.3 — Trainer Gate B replay diagnostic

C1 is a control experiment, not a training retry. It runs the exact
`grpo_v1_0p5b` file and trainer code from
`v0.10.2-p0-three-piece-freeze`, against the same frozen 50-row alias, while
using the fresh H100 runtime/cache. The current config cannot be substituted:
it contains later `enforce_eager` and vLLM-backend fields absent from the Gate
B source.

At three minutes, collect a full native `py-spy` stack from `WorkerDict`, then
wait no longer than fifteen minutes for one bridged JSONL metric. A metric
selects the future C2 configuration investigation; no metric selects the
future C3 environment investigation. Neither branch is an authorization to
modify the trainer or launch M3/M4.

**C1 result:** the exact 0.5B tag workload entered V1, updated weights, and
published eight metrics before W1 cleanup. Under the declared decision rule
this selects C2, but it cannot by itself prove the 1.5B config faulty: N6 was
stopped about two minutes before the control's first metric appeared. No C2
trainer change is authorized until a new documented plan names the comparison
and its stop conditions.
