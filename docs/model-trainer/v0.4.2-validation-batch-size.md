# v0.4.2 — Model Trainer: Positive Validation Batches

The D2 validation split contains ten cases. verl's default resolution produced
a zero validation batch size in the single-GPU preflight, so the command now
sets `data.val_batch_size=1` explicitly. This only makes the existing greedy
validation path constructible; it does not alter the 40/10 split or training
batch size.
