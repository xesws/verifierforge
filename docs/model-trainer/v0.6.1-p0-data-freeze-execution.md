# v0.6.1 — existing trainer consumption of frozen data

**Status:** in progress. Linked plan: `docs/versions/v0.6.x/v0.6.1-p0-data-freeze-execution.md`.

Tonight does not alter `trainer/` logic or configuration. The existing loader is fixed to the 50-row `trainer/data/nl2sql_v1.jsonl` fixture and its deterministic 40/10 split. S2 therefore replaces only the reviewed fixture contents with 50 enriched, provenance-bearing rows while retaining the required `id`, `question`, `reference_sql`, and `expected_results` fields; extra metadata is ignored by the loader.

After the pushed freeze tag, the existing `grpo_v1_0p5b` job is the only permitted Gate B command. Its metrics curve is a short-run smoke observation, not a claim of held-out improvement.
