# v0.28.5 — post-training checkpoint serviceability

The P-2 trainer uses `post_training` serving-gate timing. At steps 50 and 100,
the bridge writes atomic candidate artifacts through Storage and records their
manifest identities without starting vLLM. A candidate is neither a published
checkpoint nor sufficient evidence for lifecycle collection.

After verl exits successfully, `trainer/launch.sh` shuts down the job's Ray
runtime. A separate finalizer materializes the selected step-100 candidate,
runs the unchanged conversion plus loopback vLLM models/completion checks with
exclusive GPU use, uploads the serviceability evidence, and only then calls the
normal checkpoint publication and final-artifact paths. Failure remains
fail-closed and leaves the candidate evidence available.

Future mid-run validation, if required, must use **pause-verify**: deliberately
stop training, release the GPU, service-test the selected candidate, then resume
from the Storage checkpoint. That mode is documented only in this patch; it is
not implemented. The repository's existing resume proofs make it feasible, but
it needs its own version and tests before use.
