# v0.16.0 — storage-only resilience proof

The 0.5B 100-step proof deliberately reuses the frozen data and existing GRPO
configuration without editing `trainer/`. The `Storage` implementation provides
the existing LocalStorage-shaped materialization path to callers, while S3 is
the sole durable checkpoint/metric/artifact destination.

The acceptance event is a deliberately killed job at about step 60 whose new
process resumes from the highest committed S3 checkpoint and reaches step 100
with a continuous public curve. Failure to obtain cloud permission stops this
proof before a GPU job starts.
