# v0.28.4 — loopback vLLM free-memory fit

vLLM 0.10.2 rejects a utilization target larger than free memory before model
load. The `-r3` evidence measured 14.51/19.67 GiB free and rejected 0.90 because
it requested 17.71 GiB. The loopback smoke uses 0.70, or about 13.77 GiB, while
retaining BF16, eager mode, model length 64 and max sequence count one.

This changes only the checkpoint serving gate's resource ceiling. The GRPO
process, rewards, data, steps and checkpoint contents are unchanged.
