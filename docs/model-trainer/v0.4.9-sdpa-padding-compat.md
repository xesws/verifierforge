# v0.4.9 — Model Trainer: Padding Helpers without FlashAttention Kernels

The D2 actor intentionally runs with SDPA because FlashAttention 2 is not part
of the pinned CUDA runtime. verl 0.8 nevertheless imports FlashAttention's
small padding utility module during old-log-prob computation. That import is a
data-layout dependency, not a request to execute FlashAttention kernels.

The fix uses the semantically matching helpers already exposed by the pinned
Transformers version (`_index_first_axis`, `_unpad_input`, and `_pad_input`),
with a narrow PyTorch fallback only if those helpers are unavailable. It must
not shadow or pretend to provide FlashAttention CUDA attention kernels. The
test is an actual two-step GRPO run, not just an import check.
