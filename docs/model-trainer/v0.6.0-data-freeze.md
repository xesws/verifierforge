# v0.6.0 — frozen-data Gate B runbook

**Status:** in progress. Linked plan: `docs/versions/v0.6.x/v0.6.0-data-freeze.md`.

## Scope

This is an operations document only. v0.6.0 does not modify `trainer/` Python
training logic, reward calculation, checkpoint bridge, or the D2 runtime pins.
Once Gate A succeeds, the existing 0.5B GRPO path is launched through `vf` for
approximately one hour against the immutable tagged dataset.

## Required run record

The job configuration and `train.log` must name the frozen Git tag/dataset
digest. `runs/<job>/metrics.jsonl` is the source for the pass@1 curve, and the
final log must state whether the short run is merely a smoke signal or supports
a stronger conclusion. A stopped job may resume only through the existing
Storage-published checkpoint discipline.

## GPU prerequisites and validation

Use the already-validated 0.5B/verl runtime on RunPod; do not reinstall or
upgrade trainer dependencies during this gate. Confirm tmux detachment, inspect
with `vf status`, sync non-checkpoint evidence using `vf watch`, and preserve
the run log/curve for D4 planning.
