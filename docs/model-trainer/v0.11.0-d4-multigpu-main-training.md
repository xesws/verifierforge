# v0.11.0 — Trainer: D4 single-Blackwell main/control runs

The Gate B configuration is not reused as a main run: it hard-codes one GPU,
0.5B, and 100 steps. D4 adds operator-visible configurations for a 30-step
1.5B single-Blackwell smoke, a 400-step 1.5B main run, and a 200-step 0.5B random
reward control. Each preserves the frozen 50-row training-pool alias and
`rollout_n=8`.

The M1′ hardware decision is one 96 GB RTX PRO 6000 Blackwell GPU. The main
process therefore uses one GPU with FSDP strategy compatibility and a one-way
vLLM rollout/weight-update proof; it is not represented as a multi-card FSDP
claim. M2′ must show Blackwell compatibility, throughput, and a post-update
rollout before it is admitted. Main checkpoints publish through
Storage every 50 steps. An entropy-brake monitor observes append-only bridged
metrics and stops only after ten consecutive values below 25% of the initial
ten-step median, never before step 20.

The control uses a deterministic seed-derived Bernoulli(0.5) reward function;
it does not call the NL2SQL verifier. Because a single GPU cannot safely host
two independent Ray/vLLM jobs, it runs serially after M3 with its own job ID,
checkpoints, and metrics. The trainer may not select a main
checkpoint using its 40/10 training split: only the external frozen held-out
evaluation may mark a checkpoint as retained. Tests cover configuration GPU
overrides, random-reward determinism, entropy-brake boundaries, and checkpoint
selection artifacts before any pod launch.

**M2′ result:** the 30-step Blackwell smoke was launched but stopped at the
approximately 15-minute gate before the first recorded training metric. FSDP
rank 0 and NCCL `2.27.3+cuda12.9` initialized, and the vLLM server command was
constructed for the 1.5B model, but no rollout response, synchronized weight
update, throughput, `metrics.jsonl` line, or checkpoint appeared. The log has
no CUDA/OOM/Traceback diagnostic, so this document makes no causal claim. The
trainer path is not admitted for M3 or the serial M4 control until an approved
M2′ repair/retest meets every acceptance condition.
