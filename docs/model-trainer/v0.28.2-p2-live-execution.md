# v0.28.2 — P-2 live execution evidence

The training payload is unchanged: Qwen2.5-0.5B-Instruct, frozen 50-row pool,
100 steps, k=8, checkpoints at 50/100, verifier reward and S3-only persistence.
This patch does not edit `trainer/` logic or the frozen dataset.

P-2 completion still requires step-100 metrics, the step-100 checkpoint
manifest, final model marker, curve PNG and verified per-object SHA-256 before
the pod may be considered collected and terminated.

The first live attempt failed at the step-50 serving gate because the 0.5B raw
HF export is a valid single `model.safetensors` without an index. The converter
will accept exactly that single-file layout, create the standard output index,
and retain the vLLM load/completion smoke. This is export compatibility, not a
training algorithm or frozen-data change. The retry starts a clean `-r2` S3
job; the first attempt's 49 metrics remain failure evidence only.
