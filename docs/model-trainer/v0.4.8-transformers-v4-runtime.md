# v0.4.8 — Model Trainer: Keep vLLM on Transformers v4

`vllm==0.10.2` is used with `Qwen/Qwen2.5-0.5B-Instruct` in the D2 GRPO
preflight. The pod's unconstrained resolver selected Transformers 5.13.1,
whose Qwen tokenizer API no longer has the property vLLM 0.10.2 reads.

The trainer requirements will pin a tested Transformers v4 release (within
vLLM's declared `>=4.55.2` range). The acceptance check is not merely an
import: the detached two-step run must start vLLM and execute reward scoring.
