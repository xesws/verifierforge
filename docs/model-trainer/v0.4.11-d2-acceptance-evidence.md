# v0.4.11 — Model Trainer: D2 Acceptance Evidence

**Status:** Complete

## Scope

Record the real `grpo_v1_0p5b_preflight` and `grpo_v1_0p5b` executions on the
provisioned one-L4 pod. Evidence must cover the pinned runtime, deterministic
40/10 fixture path, native verl checkpoints, and the Storage-only resume
boundary.

## Required Evidence

- Two-step preflight with real validation, bridged metrics, a step-2 checkpoint,
  final manifest, and curve.
- A 100-step run that publishes independent `ckpt/step_50/global_step_50` and
  `ckpt/step_100/global_step_100` trees.
- `vf kill d2demo` after step 50, then a log line showing resume from the
  published Storage path and a first new bridged metric above step 50.

## Non-goals

This records an operational smoke only; it does not establish model quality or
claim that the observed validation curve generalizes.

## Observed Evidence — 2026-07-15

The two-step preflight ran through real vLLM rollout, NL-to-SQL reward, and
verl checkpoint writing, then published `step_2` through `LocalStorage`. The
full `d2demo` run published `step_50`, was intentionally stopped with `vf kill`,
and resumed only from `ckpt/step_50/global_step_50`; the first new public metric
was step 53. It completed at step 100 with an independently copied native
checkpoint containing model, optimizer, and extra state.

Pinned pod runtime: Python 3.12.3, torch 2.8.0+cu128, vLLM 0.10.2, verl 0.8.0,
Ray 2.56.0, and Transformers 4.57.6 on one L4. A shutdown-only
`DataLoader worker ... killed by signal: Killed` traceback appeared after both
normal native completions, but the launcher still returned through its
`Finished ...` path; cgroup `memory.events` reported `oom=0` and `oom_kill=0`.
