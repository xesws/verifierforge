# v0.15.1 — M6 checkpoint identity inventory

`trainer.m6_archive` will publish an additive schema-v2 identity manifest. For
each standard-HF `actor/serveable_huggingface` export it records the checkpoint
step, the full file inventory (`path`, `size_bytes`, `sha256`), and the tree
digest. The canonical digest concatenates sorted relative POSIX path UTF-8
bytes, `0x00`, the ASCII file SHA-256, and `0x0a`, then SHA-256s that stream.

The old manifest stays intact. A future archive must never publish a tree-only
identity: file-level data is required to locate a mismatch without guessing or
re-evaluating a model.

**Result:** the new M6 manifest inventories steps 50 through 400 in 50-step
increments; each export has 13 recorded files. Step 350 retains its canonical
`7bde…` identity.
