# v0.12.1 — Trainer: runtime provisioning gate

The H100 trainer cannot progress beyond N3 until the provisioned environment
imports the pinned runtime exactly: torch `2.8.0+cu128`, vLLM `0.10.2`, verl
`0.8.0`, and ray `2.56.0`. The provisioning change is infrastructure-only; it
does not change a model, dataset, verifier, reward, trainer configuration, or
training behavior.

After the explicit tmux-backed installation completes, N3 runs one import and
version command. A missing package, import error, or any version divergence
stops the whole N1–N7 sequence. Only a passing N3 may begin the two permitted
Qwen model snapshots in N4.

**H100 result:** the durable N3 probe passed all four imports at the exact
versions, so the trainer runtime gate is open for N4 only.
