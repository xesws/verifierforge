# v0.12.4 — Trainer H100 milestone smoke and D4

E1 keeps the existing `grpo_v1_1p5b_h100_smoke` values unchanged and adds only
the H100 diagnostic environment through the shell and Ray worker boundary. It
must be observed for 30 minutes using engine milestones, not an 11-minute
absence-of-metrics heuristic.

On E1 admission, M3 uses a visible 400-step 1.5B config with `k=8`, checkpoint
frequency 50, 0.45 rollout memory, and one GPU/TP 1. Its entropy brake is
tested and emits an evidence artifact when triggered. M4 is a separate serial
200-step 0.5B config whose reward function is deterministic Bernoulli(0.5),
seeded independently from verifier reward. Metrics/checkpoints/artifacts never
share a job namespace.

The current runtime captures `pip freeze`, driver, GPU, and the five diagnostic
environment variables before every GRPO child. This is an evidence change, not
a dependency change. M3/M4 do not make held-out claims; M5 is the only report
path permitted to compare against the frozen 60-row baseline.

Before the M3/M4 implementation, their data boundary is fixed: both train on
all 50 rows of the frozen pool; their deterministic 10-row monitoring slice is
also drawn only from that pool. The held-out file is never supplied to verl.
M3's 50-step checkpoints include an HF export for post-training serving as
well as native resume state. Its entropy brake writes `early_stopped` and an
atomic evidence artifact after step 20 only when ten consecutive bridged
entropies are below 25% of the first-ten median; it publishes no final-model
artifact. M4 selects a separate fixed-seed Bernoulli reward adapter.

**E1 implementation:** the environment is both exported by the H100 launch
target and represented as Ray runtime-environment overrides. Tests cover all
five worker-visible keys and the runtime evidence writer.

**First remote E1 result:** the shell exports were correct, but Hydra converted
unquoted `1` and `0` overrides to integers before Ray validated the worker
environment. The repair quotes the values; no smoke parameter or model setting
changes.

**E1 admission result:** rerun `e1-h100-1p5b-smoke-r1-v0124` passed every
milestone and completed 30 steps. It published Storage checkpoints at 10/20/30
and 30 bridged metrics. Stable steps 2--30 averaged 1198.881 tokens/s and
7.828 seconds per step; medians were 1302.839 tokens/s and 6.351 seconds. A
final DataLoader-worker `Killed` traceback appeared while teardown completed,
but the driver returned success, step 30 and final artifacts published, and
`nvidia-smi` returned 0 MiB. It is preserved as a nonfatal observation.
