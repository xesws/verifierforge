# v0.27.0 — Approved sample-source persistence

The cluster repository gains one nullable JSON metadata column for an approved
sample source. The stored object contains only governance and identity data:
source kind, repository-relative URI, SHA-256, row count, approver and approval
timestamp. Prompt/completion bodies remain outside SQLite and Supabase.

An Alembic migration upgrades existing SQLite/Postgres databases without
rewriting cluster rows. Repository round-trip tests run through the shared async
store contract. Source replacement is explicit, and its identity is available
to Agent cache invalidation and audit evidence.

Result: revision `20260719_0002` upgraded SQLite and Supabase. Supabase stores
only six metadata fields for the approved source; deterministic excerpts are
read from the repository file at tool-call time, and no sample body was added
to `clusters` or `traffic_requests`.
