# v0.29.0 — BYO provider credential boundary

Provider credentials use the existing `provider_credentials` table and
repository. Settings accepts a provider and API key but never returns key
material. Fernet protection is configured only through `VF_CRED_KEY`; decrypted
bytes exist only while constructing the provider adapter and are not cached.

For the local reviewer demonstration only, a missing stored RunPod credential
may fall back to `RUNPOD_API_KEY` from the process environment. The source is
reported as a non-secret label, never as a value.
