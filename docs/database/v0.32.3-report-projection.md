# v0.32.3 report projection

**Status:** complete — 2026-07-20

Supabase `jobs.summary_json` may hold the complete bounded Job presentation
projection, including the ten selected arena cards. The projection is never
hand-written: one deterministic builder derives it from committed/frozen
artifacts, records input hashes and source version, and idempotently backfills
the row.

The canonical JSON content hash excludes its own hash field. `generated_at`
comes from authoritative release evidence rather than wall-clock rebuild time,
so rebuilding the same sources is stable. Raw held-out samples, curves, and
evidence remain authoritative in artifacts/S3; any detected mismatch is fixed
by rebuilding the database projection from those sources.

Tests assert that the stored projection equals a fresh rebuild, fits the
64 KiB `summary_json` bound, and returns the same report as artifact and hybrid
modes.

Current ownership is explicit: Supabase stores traffic, clusters, routing,
Guardian points, the Jobs ledger, Agent decisions, and approvals. Artifacts/S3
store curves, raw arena samples, and evidence bundles. The current raw M5 arena
inputs are committed evidence and are not claimed to be present in S3; the
projection therefore records `s3_prefix: null` until that physical archive is
proved.
