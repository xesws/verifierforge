# v0.28.0 — Provision handle registry and audit

P2 extends existing repositories additively: approvals can be loaded by ID and
atomically bound to exactly one provider handle; active ownership is derived
from that binding plus terminal audit state. Jobs receive lifecycle summaries
and the S3 prefix. All create, transition, collect, terminate, billing and orphan
events remain append-only in `provision_events`.

No provider credential enters the database. The RunPod API key remains local,
while the pod receives only ephemeral S3 variables through tmux environment
injection. Repository tests cover idempotent binding, conflict rejection,
active-registry classification, and complete event order.
