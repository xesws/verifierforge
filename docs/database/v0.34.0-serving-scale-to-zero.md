# v0.34.0 Serving registry storage

**Status:** planned

Alembic adds `serving_endpoints` and append-only `serving_events` for SQLite and
Postgres. The endpoint row owns model/session identity, private URL and encrypted
API-key reference, lifecycle, provider handle, selected GPU/price, timestamps,
cost, and sanitized last error. An internal unique active slot enforces the
global concurrency limit at the database boundary.

The store provides atomic wake reservation, idempotent state observation,
compare-and-set transitions, active-session listing for restart recovery, and
append-only audit reads. The public API never returns `api_key_ref`, raw provider
bodies, or credential material. Store parity and concurrent wake behavior are
tested on SQLite and the existing Postgres parameterized framework.
