# v0.28.2 — Provision deletion and billing audit

Provision audit distinguishes `provider.deletion_confirmed` from the later
`billing.confirmed` event. Deletion evidence records the job/approval/provider
identity, target absence and zero raw `vf-auto-*` count. Billing is written only
when the provider returns a record; until then lifecycle summaries use
`billing_status: pending` and a separate conservative estimated-cost field.

Existing approval binding remains exact-once. The historical unbound approval
may resume only after its gold evidence matches and the live account prefix
inventory is empty.
