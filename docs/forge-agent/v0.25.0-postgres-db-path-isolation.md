# v0.25.0 — Agent data-source isolation after Postgres cutover

**Status:** complete — 2026-07-18

The DB-2 persistent-backend regression exposed a configuration precedence bug:
`ToolRegistry("real", db_path=...)` still read the global Postgres backend when
`VF_DB_BACKEND=postgres`. Explicit paths are test and operator-selected local
data sources, so they must bind to that SQLite file even after production moves
to Supabase. A missing explicit file remains read-only and must not be created.

This patch changes only Agent tool data-source resolution and focused tests. It
does not change contracts, database schema, frozen data, trainer code, feature
flags, or model/provider behavior. Validation requires the focused Agent tests,
the full suite with SQLite defaults, and the full suite with `.env` selecting
Postgres.

Result: explicit paths now bypass the process-wide backend selection, focused
Agent coverage passed `14 passed`, and the Postgres-selected full suite passed
`365 passed, 1 skipped`.
