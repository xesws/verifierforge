# v0.27.0 — Approved sample source

The real Agent binding must not infer prompt bodies from `traffic.db`: proxy
observability stores hashes, route labels, tokens, latency and verifier scores,
never request or response text. A cluster may instead reference one explicit,
user-approved sample source.

The first supported source is repository JSONL under `data/`. The API validates
the repository-relative path, file SHA-256, JSONL shape and row count before
persisting metadata. Agent inspection sorts rows deterministically by `id` and
returns at most the requested sample count. Verifiability is computed from the
approved rows' `reference_sql` fields and the NL2SQL verifier contract.

The source identity participates in the Agent evidence fingerprint, so attaching
or replacing a source invalidates a prior cached decision. A dedicated
`p2_gate_b` execution profile constrains the structured proposal to the approved
0.5B model, 100 steps, k=8, checkpoint interval 50 and RunPod; it does not alter
the normal 1.5B product default.

Acceptance preserves both sides of the boundary: a complete source-less
`need_more_data` trajectory and a new source-backed production trajectory. The
Agent still has no provisioner or paid-compute handle.
