# v0.22.3 — Gate C failure taxonomy

This wave separates three defects before editing behavior:

- formatting: a terminal config exists but strict contract parsing rejects it;
- intelligence: the final decision direction disagrees with the frozen gold;
- chain: a tool argument does not reuse an ID issued by an earlier tool.

Every change must map to one observed row. Replay must pass before each of at
most three live rounds. The feature flag remains owner-controlled.

The chain repair makes the four read-only calls sequential and binds each
downstream schema to the exact `cluster_id`, `analysis_id`, and `sample_set_id`
already issued by the runner. These are generation constraints, not post-hoc
argument correction: a provider that violates them is still rejected and
audited.

Live round 1 proved the chain repair (`1.0`) but exposed one remaining
intelligence error: an empty mock fixture overruled authoritative evaluator
facts. The general trust hierarchy was clarified, replay passed again, and live
round 2 passed every frozen metric at `1.0 / 1.0 / 0 / 1.0`.
