# v0.19.0 — Contracts and tool registry

The A-1 dependency graph is explicit:

`analyze_traffic -> analysis_id -> inspect_samples -> sample_set_id -> check_verifiability`

`estimate_economics` also consumes the issued `analysis_id`. IDs are SHA-256
hashes of canonical inputs plus observed facts. A registry accepts only IDs it
issued during that run, preventing a model from fabricating a plausible token.

The real binding opens `VF_PROXY_DB_PATH` through SQLite URI read-only mode. It
aggregates traffic metadata but returns `data_sufficient=false` for sample
inspection because the current schema intentionally stores no prompt or response
body. The mock binding provides deterministic, redacted fixtures for evaluator
and UI development with the exact same response models.
