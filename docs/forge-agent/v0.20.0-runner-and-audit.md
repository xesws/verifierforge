# v0.20.0 — Runner and audit trail

The runner permits one tool call per turn, at most eight turns, 12,000 combined
tokens, and 90 seconds. It does not request or retain hidden reasoning. Each
trace contains only structured tool input/output, public rationale, timestamps,
usage, errors, guard events, and the terminal decision.

`submit_decision` is not part of the extensible ToolRegistry. Its arguments are
validated twice: the strict Pydantic contract and a runtime policy whose budget
ceiling defaults to $100 and may only be lowered by
`VF_AGENT_MAX_TRAINING_BUDGET_USD`.

The agent package may not import trainer, control-plane scripts, subprocess,
provisioner modules, or GPU cloud SDKs. This is the physical decision/execution
boundary described in the design documents.
