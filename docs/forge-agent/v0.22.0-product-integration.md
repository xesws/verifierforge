# v0.22.0 — Discover integration

The flag-only product surface is:

- `POST /clusters/{cluster_id}/agent/analyze`
- `GET /clusters/{cluster_id}/agent/decision`
- `POST /agent-decisions/{decision_id}/approvals`
- `GET /discover`

Analysis responses contain the decision envelope and a `cached` flag. The UI
shows rationale/confidence and a config only for `forge`; Approve & Forge writes
an approval record and stops. Mock fixture decisions are deterministic:
data-pull-sql=forge, invoice=need_more_data, support=skip.

The demonstrated `data-pull-sql` path produced a completed `forge` decision,
reused it only for the identical evidence fingerprint, and accepted one
idempotent approval. The 3,766-byte full trace was present in S3 with SHA-256
metadata `311ea43b8fde714d192fcf7edc87111fc4389d1a6d00f3dada829dc290b29908`;
the queryable decision and approval remained in SQLite. These are mock-agent
semantics over the real audit stores, not Gate C admission evidence.
