# v0.23.0 — Forge Agent relational stores

Agent decision summaries and approvals move from SQLite-specific classes to
async `AgentDecisionStore` and `ApprovalStore` implementations. Full AgentTrace
objects remain in S3. Decision cache lookup, audited-forge approval gating, and
approval idempotence retain their current product behavior.

The real traffic-analysis tool reads through `TrafficStore`; mock binding stays
deterministic and database-free. The Agent runner becomes async only at the
relational boundary; tool order, guards, Gate C scenarios, and feature flags do
not change.
