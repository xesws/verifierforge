# v0.22.4 — Agent proposal presentation boundary

This patch does not change Agent reasoning or decision/config contracts. It
separates stable monthly cluster-profile facts from the short synthetic traffic
window and displays the existing decision in a clearer product sequence.

`Approve & Forge` continues to write one idempotent approval for a completed,
durably traced Forge decision. It does not provision, train, spend, create a
job, or alter routing. A future separately authorized execution service may
consume that record; no such handle is introduced here.

The optional analyze input is provenance/selection for the already configured
traffic database. The server validates it against `VF_PROXY_DB_PATH` and never
opens an arbitrary browser-supplied path.

## Validation result

Mock analysis remained deterministic and zero-cost. The approval write and
subsequent receipt read used SQLite-only services; tests make LLM service
construction fail deliberately on those routes, proving that approval cannot
cross into an execution or paid-inference path.
