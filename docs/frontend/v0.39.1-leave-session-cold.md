# v0.39.1 — Leave session shutdown behavior

The global Leave session control becomes an asynchronous reset boundary:

1. show `Closing demo…` and disable repeated clicks;
2. call the authenticated serving sleep endpoint;
3. require a returned `cold` state;
4. only then clear invitation, journey, and serving activity session storage.

If shutdown fails, the page remains authenticated and shows a retryable error
beside the button. This is intentional: hiding the app while a paid endpoint may
still run would be a false reset. Closing a tab or browser remains outside the
guarantee because teardown requests are not reliable.
