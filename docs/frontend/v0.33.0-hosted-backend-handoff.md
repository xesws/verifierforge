# v0.33.0 hosted frontend handoff

**Status:** complete; fixed reviewer base URL accepted — 2026-07-20

The frontend will set `VITE_VF_API_BASE_URL` to the Railway public subdomain.
Product requests use the separately shared Basic Auth invitation; health is
public. CORS permits configured localhost origins, Vercel preview subdomains,
and the final Vercel URL without opening a wildcard origin.

The hosted service keeps `VF_AGENT_BINDING=mock` and
`VF_AUTOPROVISION=false`, so Analyze is deterministic and Start Forge returns
the documented disabled response. The frontend contract remains the 19
operations frozen by `frontend-api-v1.1`.

For local development:

```text
VITE_VF_API_BASE_URL=http://127.0.0.1:8010
```

The accepted production value is
`https://verifierforge-production.up.railway.app` (no trailing slash).
Every product request carries the separately shared Basic Authorization header;
`GET /healthz` is public. Vercel preview origins are accepted by the configured
regex, while the final production Vercel origin should also be added explicitly
to `VF_CORS_ORIGINS` in Railway.

If the tuned GPU route is down, the frontend should continue rendering report
artifacts. `/healthz` may report `degraded`; proxy completions fall back rather
than returning an application error. This is a supported degraded state, not a
signal to replace the API base URL.
