# v0.35.1 — Discover Analyze source semantics

Discover exposes two different data concepts and must not conflate them:

- **Input** is the human-approved repository JSONL that defines the bounded
  training/evaluation sample source. The UI submits it only through
  `PUT /clusters/{id}/sample-source` with its expected SHA-256 and row count.
- **Analyze** examines production traffic through the server-configured proxy
  store. The reviewer UI sends no filesystem path in
  `POST /clusters/{id}/agent/analyze`; the backend resolves
  `VF_PROXY_DB_PATH` inside its own trust boundary.

The optional `data_source` request field remains available for controlled
operator tests, and the backend must continue rejecting any path that differs
from its configured traffic store. A failed Analyze does not unlock Forge.

Acceptance requires the default Input to remain visible and identity-checked,
Analyze to return a forgeable decision in the deterministic public sandbox,
and the Forge navigation item to unlock immediately after that success.

The deployed candidate met this gate: its Analyze body was `{}`, Railway
returned HTTP 200 with `decision=forge` and a config, and the journey regression
test observed the resulting **Continue to Forge** handoff.
