# v0.29.0 — P-4 double-confirmation demo

The FastAPI-hosted Discover page keeps `Approve & Forge` as a no-spend action.
Once its receipt exists, the page may show a separate `Start Forge` control,
an explicit statement that a provider resource may incur cost, and a second
confirmation. The page then polls visible lifecycle state through completion.

With `VF_AUTOPROVISION=false`, Start is unavailable and existing product
behavior is unchanged.
