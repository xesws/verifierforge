# v0.32.2 frontend run snapshot

The repository snapshot publishes historical metrics, reports, curves,
evaluation samples, and SQLite demo fixtures from `runs/` for frontend work.
Consumers should treat them as immutable demo/evidence inputs rather than live
application state. Large `.safetensors` model weights are intentionally absent;
the frontend has no dependency on those bytes.

Validation requires the staged snapshot to pass the credential scanner and the
GitHub single-file size gate before push.
