# v0.35.3 — Ship readiness gate

The SQL playground must always state its serving prerequisite beside the Run
action:

- `cold`: model is asleep; confirm spend and Wake above;
- `provisioning` / `loading`: Wake is in progress; Run unlocks automatically
  only when the registry reports Ready;
- `ready`: tuned endpoint is verified and Run is available;
- `draining`: endpoint is shutting down; wait for Cold before another Wake;
- failed cold state: show the recorded failure and direct the reviewer to retry
  from Wake.

A premature **Run tuned completion** click is intentionally handled rather
than natively disabled: it displays an alert, scrolls to the Wake section, and
does not call either Wake or completion. The control exposes `aria-disabled`
until Ready. Globally disabled controls must look disabled and must not animate
as hoverable primary actions.

Registry polling pauses while the page is hidden, then immediately reloads on
`visibilitychange` when the reviewer returns. A background tab must never stay
permanently stuck on a stale Loading state.
