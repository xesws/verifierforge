# Frontend Documentation

This area owns the future `app/web/` UI, mock-server consumption, onboarding, charts, report views, and visual API compatibility. It must not alter trainer or storage internals.

Before frontend work, create a versioned document covering the user flow, API/mock contract used, loading/error states, visual acceptance criteria, and screenshots or recording requirements. Contract changes must be coordinated through `docs/backend-api/` before implementation.
