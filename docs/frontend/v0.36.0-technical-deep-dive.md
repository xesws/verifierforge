# v0.36.0 public technical deep dive

**Status:** complete

The React frontend will expose a public `/tech` route outside `InviteGate`.
It renders a byte-checked generated mirror of
`docs/blog/technical-deep-dive.md`, supports GFM and KaTeX, and presents a
sticky table of contents plus collapsible level-two sections.
The page makes no API request and never advances reviewer journey state.

The authenticated navigation gains an always-available Tech link without
changing the five product-stage locks. The page and favicon use a small SVG
brand mark derived from the existing V/F visual language and current graphite,
blue, and green tokens. Desktop and 390px layouts must remain readable; images
must have alt text and figures must retain visible captions.

Validation covers direct unauthenticated `/tech` access, article/figure/math
rendering, navigation behavior, the existing product flow tests, lint, and the
Vite production build.

Production: `https://verifierforge-web.vercel.app/tech`. The route is outside
`InviteGate`, lazy-loads its Markdown renderer, performs no API request, and
links back to the protected product. Because the Vercel project root is
`frontend/`, a generated content mirror is committed for deployment and is
required to equal the canonical blog, figures, and wordmark in pytest.
