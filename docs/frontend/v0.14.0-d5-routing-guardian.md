# v0.14.0 — Routing controls for the frontend

The D5 frontend control can use the same routes against mock and real API:
`GET` and `PUT /clusters/{cluster_id}/routing`, plus
`GET /clusters/{cluster_id}/live-pass-rate`. The PUT body and every routing
response are the existing `RoutingState` shape; the switch is `enabled` and
the canary slider is `canary_percent` from 0 through 100.

`target_model` is a stable logical target label (`tuned` in tonight's fake
flow), not a deployed URL. The proxy resolves that label through the operator's
environment, so the frontend never needs an endpoint address or secret. Live
pass-rate points are asynchronous operational telemetry. They are not a
training score and may remain empty until sampled tuned traffic is observed.

The local acceptance proved both switch states through the real API: a 50%
canary split produced 113 tuned and 87 default requests across 200 calls, then
a 0% update produced zero additional tuned requests across the next 20 calls.
