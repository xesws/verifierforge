# v0.34.0 Reviewer serving controls

**Status:** complete; public reviewer flow accepted

The Discover demo gains a separate **Wake model** panel. It is not part of the
Approve/Start Forge training flow. The user confirms provider spend once, the
button disables after submission, and the page polls serving status until it
shows cold, provisioning, loading, ready, or draining.

Ready state may display the public endpoint, selected GPU, hourly price, and
measured cold-start duration. Error text is sanitized. When the feature flag is
off the existing page behavior is unchanged; when the model is cold, flagship
reports and arena cards remain available from frozen artifacts.

The frontend contract grows additively from 19 to 21 operations and the mock
server must return the same response models as the real API.

The hosted button is invitation-protected and uses a separate default-off code
flag from training autoprovision. Live acceptance observed the full progress
sequence twice; the measured cold-start expectation for the selected fallback
is about 4.5 minutes. Reports remain visible while state is cold or loading.
