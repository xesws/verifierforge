# v0.39.0 — live Guardian auto-traffic

The Ship routing form and Guardian panel become one continuous reviewer action.
Once `Save routing policy` succeeds, the page starts or adopts the existing
200-request demo traffic task without requiring a second click.

While traffic runs, the chart switches from the compressed all-history view to
a run-local x-axis from request 0 to request 200. The green line advances as
the backend `sent` counter advances. Its y-value is the latest rolling Guardian
pass rate returned by the API: unsampled/default requests carry that value
forward, and only persisted sampled tuned SQL scores change it. The UI labels
traffic progress and new Guardian sample count separately so 200 requests are
never presented as 200 verifier scores.

The completed run remains visible with its final request count, success/failure
totals, Guardian sample delta, and latest pass rate. The manual simulation
button remains available for deliberate reruns, and cold guidance remains
visible because saving a policy does not Wake a model.

The chart auto-zooms its labelled y-axis to the visible rolling-rate range and
shows only the latest 40 points outside a run, avoiding the previous 151-point
compression while keeping the complete API count in the header.

Validation passed with 34 frontend tests, ESLint, and the production Vite
build. The browser runtime had no available browser instance, so validation
does not claim a manual animation inspection or screenshot.
