## Required first-screen story

The README must let a judge answer three questions before reading deployment
details:

1. What business workflow did VerifierForge optimize?
2. What does one real input/output example look like?
3. Did the selected small model improve on unseen data?

The scenario is the frozen enterprise NL→SQL workload over employees,
departments, projects, and assignment hours. The central result is held-out
pass@1 `0.5833 → 0.7833` across 60 unseen prompts, with step 350 selected from
eight checkpoints. The canonical figure is
`docs/blog/figures/05-heldout-selection.svg`.

4. Hosted reviewer entry.

The README will reference canonical `docs/blog/figures/` assets, never the
generated frontend mirror. Infrastructure instructions remain available but
are secondary to evidence for this submission-facing repository.

## Truth boundaries

- Pass@1 is calculated over all 60 held-out prompts; the UI displays ten arena
  examples but those ten are not the metric denominator.
- The selected step is 350, not the final step 400.
- The random-reward run is a falsification reference with a smaller model and
  shorter duration, not a controlled causal estimate.
- The SQL shown in README must use columns present in the frozen schema.
