# v0.6.3 — pod-local full difficulty probe

**Status:** implementation validated; pod launch pending O1/O2. Linked plan:
`docs/versions/v0.6.x/v0.6.3-full-difficulty-probe.md`.

The probe runs in a new detached tmux session on RunPod and connects only to
the existing loopback vLLM endpoint at `http://127.0.0.1:8000/v1`. It explicitly
unsets `VF_LLM_*` and `VF_EVAL_API_KEY`, then sets
`VF_EVAL_MODEL=Qwen2.5-1.5B-Instruct`. Outputs live under the persistent
`/workspace/verifierforge/runs/p0-gate-a/` directory and are synced locally
after the background run finishes.

No polling faster than two minutes is needed. Starting the probe is the last
authorized pod action tonight; it is not a training job.
