# v0.6.1 — freeze and Gate B operations

**Status:** in progress. Linked plan: `docs/versions/v0.6.x/v0.6.1-p0-data-freeze-execution.md`.

All live augmentation and Gate A requests remain on the laptop. Once a passing freeze manifest is committed, the annotated `v0.6.1-p0-data-freeze` tag and `main` are pushed before RunPod is asked to pull. This lets the existing `vf train` command consume the frozen repository data without credentials or environment-file transfer.

Gate B starts in detached tmux with the existing 0.5B configuration. External checks occur at a 120-second cadence; checkpoints and weights stay on the `/workspace` volume and are not synchronized. Metrics, final artifact, curve, and a factual smoke conclusion are synchronized after completion. A failed remote job is evidence, not authority to alter parameters or retry automatically.
