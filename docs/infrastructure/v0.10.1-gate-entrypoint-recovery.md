# v0.10.1 — reliable remote Gate A entrypoint

`scripts/` becomes an explicit Python package so every repository-root caller
uses `/workspace/verifierforge/.venv/bin/python -m scripts.gate_a`. This is the
chosen fix for the prior pod-only import failure; no path-dependent alternative
launcher is introduced.

Before `vf train` creates a tmux session, its remote shell runs
`python -c "import scripts.gate_a"` from `/workspace/verifierforge`. A failed
import exits synchronously under `set -e`, so a job cannot appear detached while
its Python entrypoint is unusable. Direct Gate A launches apply the identical
preflight before tmux creation.

RunPod evaluation continues to use only `VF_EVAL_BASE_URL=http://127.0.0.1:8000/v1`
and `VF_EVAL_MODEL=Qwen2.5-1.5B-Instruct`; all `VF_LLM_*` variables are unset.
