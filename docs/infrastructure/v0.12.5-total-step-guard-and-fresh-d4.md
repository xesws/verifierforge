# v0.12.5 — H100 rerun operations

The H100 run keeps the established offline/diagnostic launch environment:
`HF_HUB_OFFLINE=1`, `TRANSFORMERS_OFFLINE=1`, `VLLM_LOGGING_LEVEL=INFO`,
`RAY_DEDUP_LOGS=0`, and `PYTHONFAULTHANDLER=1`, propagated to Ray workers.
Each job captures `pip freeze` and `nvidia-smi` under `evidence/` before the
child starts.

Use a new detached tmux job and monitor only append-only Storage metrics. At
the first 50 metrics, calculate actual steps/minute against the 3.5 baseline;
outside 1.75--5.25 triggers an operator update, not a stop. M3's old job is
never deleted, resumed, or used for a held-out claim. On M3 completion the GPU
must be released before serial M4 and each local loopback M5 evaluator start.
M6 records artifact SHA-256 values and verifies the laptop sync.

Stop and preserve evidence for entropy brake, any training/evaluation/storage
gate rejection, or held-out after pass@1 below 0.5833333333333334. No secret or
`.env` value is copied to the pod.
