# v0.30.0 — Full reviewer sandbox operations

`scripts/start_reviewer_sandbox.sh` retains its default loopback artifact/fake
mode. `--mode full` starts one authenticated composite API on loopback, points
its repositories at Supabase, selects the configured real tuned endpoint,
keeps Agent and provisioner bindings deterministic/mock, and creates a
Cloudflare quick tunnel. The invite code is never printed or committed; the
launcher reports only the ignored `0600` file path.

The full launcher is allowed to set a temporary canary for its proxy and must
restore the prior routing state on shutdown. Quick-tunnel URLs are evidence,
not durable infrastructure.

An HTTP tuned endpoint has two independent identities: base URL/key and its
served model ID. Full mode requires the latter through
`VF_PROXY_TUNED_MODEL` or the existing `VF_ENDPOINT_MODEL`; the proxy maps only
the tuned request copy so frontend-friendly labels remain valid.

Live quick URL at the gate:
`https://functioning-become-revelation-meanwhile.trycloudflare.com`. It remains
active only while tmux session `vf-reviewer-full` and the laptop are running.
The Basic Auth invite is intentionally absent from this document and Git.
