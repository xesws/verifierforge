# v0.4.8 — Infrastructure: Reproducible Pod Dependency Resolution

The RunPod venv is persistent under `/workspace`, but `pip` may resolve newer
major versions when a package only supplies a lower bound. D2 therefore pins
the tested Transformers major in `requirements-trainer.txt` rather than
relying on the current resolver result.

After installation, record the installed versions in the preflight log and
keep all changes inside `/workspace/verifierforge/.venv`; no host-level Python
or non-persistent filesystem is used.
