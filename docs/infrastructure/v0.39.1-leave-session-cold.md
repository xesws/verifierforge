# v0.39.1 — reviewer session lifecycle

The reviewer lifecycle now has two explicit provider boundaries: Wake allocates
one budget-capped endpoint; Leave session synchronously drains it. Successful
Leave means the serving registry is cold and the provider deletion gate has
passed. The existing 30-minute idle reaper remains the fallback for abandoned
tabs and network loss.

Before implementation, the ready recording endpoint was manually reset through
the same provider deletion primitive. An initial local invocation used the
local mock binding and therefore proved only the registry transition; it was
not accepted as provider evidence. A second invocation targeted the persisted
RunPod handle with the real runtime and confirmed both `target_absent=True` and
managed `vf-auto-*` inventory `0`. Railway then returned `state=cold`.

The deployed endpoint repeated the idempotent cold response with HTTP 200 and
kept invitation authentication mandatory. Commit `ab47298` is live on Railway;
Vercel deployment `dpl_GBLSy7zaYewCJXW7UB4Ao13FEeyP` is the matching reviewer
UI release.
