# v0.7.1 — routing-artifact handling

The routing command is local/offline: it reads synced subset evidence and
publishes one JSON artifact with a temporary file plus atomic replace. The
result is stored under ignored `runs/p0-gate-a/`, then its content hash is
checked against the immutable source sample/evidence files before a branch
decision is recorded in the run sheet. No secret, model endpoint, or prompt is
introduced by this step; the existing completion evidence remains restricted to
the local run artifact.
