# v0.4.11 — Infrastructure: D2 Acceptance Evidence

**Status:** Complete

## Scope

Record the laptop/RunPod separation during the real D2 run: detached tmux
execution, persistent `/workspace` checkpoints, `vf kill`, and `vf watch`
without copying checkpoint/staging/weight artifacts to the laptop.

## Required Evidence

- `vf train` returns while the remote tmux job continues.
- `vf kill` terminates the remote session after a completed step-50 Storage
  publish, and no GPU process remains before resume.
- `vf watch` gives the same SHA-256 for remote and local `metrics.jsonl` while
  excluding `ckpt/`, `.verl-staging/`, and `*.safetensors`.

## Non-goals

No change to the deploy-key handling, pod provisioning, or storage backend is
part of this documentation release.
