# v0.15.1 — Transfer identity gate

The D5 transfer gate uses the M6 canonical tree algorithm, not an ad hoc
reimplementation. A full file inventory accompanies the aggregate digest so a
partial transfer, copy error, or changed export can be pinpointed. The
step-350 `7bde…` value remains the serving eligibility identity; the previous
`22b9…` result is documented as a rejected literal-separator diagnostic hash.

No serving pod is started by this patch. The new identity manifest is verified
on the training pod before the existing E4 serving procedure resumes.
