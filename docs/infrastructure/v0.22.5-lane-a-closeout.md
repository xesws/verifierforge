# v0.22.5 — Serving endpoint closeout

`vfserve` is inspected non-invasively through an SSH tunnel before any public
request. Secrets move only through environment variables or stdin; they never
appear in command arguments, logs, evidence, or Git. Public proof is a single
bounded SDK attempt with automatic retries disabled.

An HTTP tuned upstream requires `VF_PROXY_TUNED_API_KEY`. Failure to load that
key is a startup/configuration error, not a reason to reuse `VF_LLM_API_KEY`.
Canary evidence uses a fresh ignored SQLite file and restores canary to zero in
a `finally` path even when traffic or guardian validation fails.

The downloaded `tq/` wheel is not a source dependency and will be removed and
ignored; TransferQueue remains supplied by the pinned trainer requirement.

**Result:** SSH to the recorded serving host timed out before any model or
public request, so S6 remained disabled and requires owner endpoint recovery.
The dedicated-key and isolated-traffic prerequisites are implemented and
covered by zero-network tests.
