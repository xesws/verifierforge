# v0.6.2 — pod-local eval server and data flow

**Status:** in progress. Linked plan: `docs/versions/v0.6.x/v0.6.2-pod-eval-recovery.md`.

The pod serves the cached Qwen 1.5B snapshot through the validated vLLM 0.10.2
venv on loopback `127.0.0.1:8000`, in detached tmux. Offline cache flags prevent
downloads. A raw `/v1/models` response is mandatory before Gate A begins.

Gate A runs in a second pod tmux session with only `VF_EVAL_*` configuration;
no `.env` is copied or forwarded. Evidence and logs synchronize back after each
finished run. Before GRPO, the server session is stopped and GPU/port release
is verified. The freeze tag message records the serving model name.
