# v0.23.0 — Database foundation

The database runtime selects exactly one backend. SQLite uses
`VF_PROXY_DB_PATH`; Postgres is enabled only by `VF_DB_BACKEND=postgres` and
requires `SUPABASE_DB_URL`. Connection strings and driver exceptions are never
rendered into application logs or API errors.

Alembic owns all schema evolution. The initial revision covers traffic,
clusters, routing, guardian/live points, jobs, Agent decisions, credentials,
approvals, and append-only provision audit events. Temporary SQLite migration
cycles and repository contract tests are the DB-1 gate; true Supabase work is
reserved for DB-2.

The migration entrypoint refuses any SQLite file that has DB-1 table names but
no Alembic revision. This protects the legacy traffic database from a partial
DDL collision; the idempotent DB-2 importer is the only supported bridge.
