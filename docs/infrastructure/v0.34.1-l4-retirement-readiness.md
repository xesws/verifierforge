# v0.34.1 L4 retirement readiness

**Status:** complete

Railway must not contain `VF_PROXY_TUNED_UPSTREAM`,
`VF_PROXY_TUNED_API_KEY`, `VF_PROXY_TUNED_MODEL`, or `VF_ENDPOINT_MODEL` after
this patch. The only production model selector is `VF_SERVING_MODEL_ID`; the
physical URL and encrypted endpoint-key reference are read from Supabase only
while `serving_endpoints.state=ready`.

The old L4 is read-only and out of operational scope. Historical endpoint URLs
remain valid evidence when their surrounding document explicitly labels them
as past observations; they are not configuration.

Sanitized inventory after deployment: all four legacy Railway variables are
absent; the same names are absent from the ignored local `.env`;
`VF_SERVING_MODEL_ID` remains present. Supabase reports `vf-demo` cold with
`url=NULL`, and RunPod's complete `vf-auto-*` inventory count is zero. The old
L4 was not contacted during this audit.
