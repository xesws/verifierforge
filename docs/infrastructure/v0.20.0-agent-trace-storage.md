# v0.20.0 — Agent trace storage

Full traces are single JSON objects at
`<VF_S3_PREFIX>/agent-traces/<trace_id>.json` with SHA-256 metadata. SQLite at
`VF_PROXY_DB_PATH` stores queryable summaries only. S3 is published before the
summary so a successful decision always has a durable audit object.

Unit tests use moto and temporary SQLite. Real credentials are never required
by CI, printed, copied into trace data, or stored in commits. This storage path
does not reuse checkpoint/training methods and performs no GPU operation.
