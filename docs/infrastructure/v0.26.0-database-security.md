# v0.26.0 — Database security and failure behavior

Credential plaintext exists only at the service boundary. `CredentialStore`
receives Fernet ciphertext, and `VF_CRED_KEY` is read from the process
environment without logging or persistence. Missing/invalid keys produce a
short generation instruction rather than a weak fallback.

Database disconnects are explicit: control-plane APIs return a sanitized 503,
Postgres mode never falls back to SQLite, and proxy observability failures are
reported without changing the upstream response. CI and pytest both scan for
connection strings, passwords, provider keys, and known sentinel secrets.
