# v0.10.0 — local-only data gates and frozen artifacts

Population/count reuse and 50/60 splitting run locally with no provider call.
The only live evaluation before freeze is the new `k=8` training-pool Gate A on
the existing loopback RunPod vLLM; it unsets all `VF_LLM_*` variables and keeps
sample/evidence artifacts under `runs/` for hash-checked sync.

If the three U2 predicates pass, the freeze workflow must bind and push the
training pool, held-out pool, and verifier provenance before the pod receives a
training command. The 60-row held-out baseline can run on the same local vLLM,
but its metrics remain separate from the training-pool gate report.
