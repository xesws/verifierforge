# v0.25.0 — Supabase migration operations

No command, exception, evidence file, or commit may contain a database URL or
password. Alembic and the importer obtain `SUPABASE_DB_URL` only from their
process environment; the checked-in Alembic configuration contains no URL.

The import starts from an ignored immutable SQLite/runs metadata snapshot with
SHA-256 and counts. Each table migrates in one transaction and verify mode
compares counts plus canonical row digests. A mismatch prevents backend switch
and milestone tag. Permission failures are not retried.

Resume after the owner supplies a resolvable pooler DSN:

```bash
VF_DB_BACKEND=postgres alembic upgrade head
VF_TEST_POSTGRES_URL="$SUPABASE_DB_URL" pytest -q tests/test_repository_contracts.py
VF_DB_BACKEND=postgres python -m scripts.import_legacy_database apply
VF_DB_BACKEND=postgres python -m scripts.import_legacy_database verify
```

The URL remains environment-only; do not put it on argv or in a log.
