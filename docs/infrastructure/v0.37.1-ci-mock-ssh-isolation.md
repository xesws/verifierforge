# v0.37.1 clean-runner mock SSH isolation

GitHub Actions runs without an operator SSH key. The P4 product test selected
the mock provider but still let `training_config_to_spec` resolve its public key
from the host default. That made CI depend on whether
`~/.ssh/id_ed25519.pub` happened to exist, while local full suites passed.

The repair belongs only in the test fixture: set
`VF_PROVISION_SSH_PUBLIC_KEY` to a syntactically representative public test
value before calling Start Forge. Production continues to fail closed when a
real RunPod execution has no configured public key.

Acceptance requires the exact GitHub security selection and full local suite,
followed by a green pushed workflow. No CI retry is accepted as the fix.

Local acceptance is green: the missing-path reproduction passes only after the
fixture injection, the security selection has 23 passing tests, and the full
suite has `492 passed, 1 skipped`. Pushed workflow evidence remains pending.
