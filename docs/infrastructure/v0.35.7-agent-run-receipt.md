# v0.35.7 — Reviewer Agent binding disclosure

The hosted reviewer UI must disclose whether Analyze used `mock` or a real LLM
provider. The existing public Railway sandbox is known to have been designed
with `VF_AGENT_BINDING=mock`; it must not be marketed as a live model run.

Production may switch to `real` only when the Gate C receipt and required
provider credentials already exist. No secret value may appear in deployment
logs, trace payloads, UI metadata, or commits.

## Production result

Railway initially reported `VF_AGENT_BINDING=mock` and no provider credential.
The repository tag `agent-gate-c-pass` identified `gpt-5.6-luna` as the passed
model, and a fresh OpenAI `/v1/models` read confirmed that exact ID. The key was
transferred from ignored local environment state into Railway through stdin;
no value appeared in command output. Railway now resolves `real / openai /
gpt-5.6-luna`, while training autoprovision remains disabled.

The first hosted run completed all four bounded read-only tools and persisted
its full trace to S3. An attempted OpenRouter transport probe failed closed at
a bound-argument guard and was not promoted.
