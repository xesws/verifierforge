# v0.35.0 — Vercel frontend and Railway API rollout

The frontend deploys as a Vite SPA on Vercel while the existing Railway
reviewer remains the API, proxy, Supabase, Agent mock, and scale-to-zero
control plane. The reviewer-journey correction adds one small Railway API
operation, `POST /serving/tuned-completion`; it must deploy before the matching
Vercel build.

## Build and environment

- Vercel project: `verifierforge-web`
- Root Directory: `frontend`
- Install: `npm ci`
- Build: `npm run build`
- Output: `dist`
- Public build variable:
  `VITE_VF_API_BASE_URL=https://verifierforge-production.up.railway.app`

Set this variable in the Vercel project dashboard or CLI; repository policy
does not permit tracked `.env` example files, even when their value is public.

The SPA rewrite serves `index.html` for direct page loads. The production build
must not read `../data/demo-artifacts`, because a Vercel project rooted at
`frontend` cannot rely on parent-directory files. Runtime API data replaces
that build-time fixture.

Vercel authentication uses the CLI device flow and remains outside the repo.
`.vercel/` is ignored. After the preview URL is created, the owner adds its
exact origin to Railway CORS; preview regex access is optional and remains an
owner-controlled Railway setting.

## One-wake acceptance

Before any separately authorized live wake, prove cold state, zero managed pods,
and an operable drain path. Start recording only after the invitation has been
entered. Require visible provisioning/loading/ready registry activity, one
tuned-only NL→SQL completion with a complete result, immediate drain, cold
registry, target absence, and zero `vf-auto-*` inventory. The probe bypasses
canary without mutating it, so there is no route restoration step.

## Promotion and rollback

The feature candidate is published to the stable Vercel alias so the owner can
perform the required visual acceptance against the matching Railway API.
Acceptance is still required before merging main and tagging
`frontend-v1-live`. A failed public smoke rolls back the Vercel deployment and
blocks that merge/tag without touching scale-to-zero serving state.
