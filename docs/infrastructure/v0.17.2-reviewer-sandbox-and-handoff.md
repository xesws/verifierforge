# v0.17.2 — reviewer sandbox infrastructure

The existing `vfserve` L4 is serving-only and must remain running. This patch
does not restart vLLM or expose a model endpoint. A reviewer sandbox uses the
committed artifact API and fake proxy locally, so it has no GPU, cloud account,
or model-provider dependency.

The only external experiment is a 30-minute maximum Cloudflare quick-tunnel
check. It may proceed only with a reviewable authorization mechanism already
present in the repository or platform. The repository currently has no invite
authentication implementation, so an unauthenticated public quick tunnel is
not an acceptable delivery path. If that remains true, the output is a
one-command local launcher and a precise owner-action note, not a fabricated
public demo.

The launcher uses `VF_API_DATA_MODE=artifacts` and `VF_PROXY_UPSTREAM=fake`.
It writes no secrets, traffic database, checkpoints, or weights to Git and
leaves service lifecycle control with its parent shell.
