# v0.28.5 — fifth P-2 live attempt

The fifth 0.5B P-2 attempt corrects resource sequencing rather than tuning a
memory fraction. Training and serving use one GPU serially: 100-step training
first, trainer/Ray shutdown second, one selected-checkpoint vLLM gate third,
then S3 collection and provider deletion.

Admission requires a clean pushed revision, a new empty `-r5` S3 prefix, zero
raw `vf-auto-*` inventory and conservative cumulative spend below `$4.5` of the
unchanged `$5` fuse. Monitor at five-minute intervals. Training-time candidate
manifests are evidence only. `COLLECTING` is forbidden until the service-tested
step-100 checkpoint, final model marker and curve are all visible.

Success requires SHA verification for metrics, candidate and published
checkpoint objects, a real completion response, accepted DELETE, absent target
and a fresh raw `/pods` list with zero `vf-auto-*`. Billing is reconciled at
`+1h` and `+6h` asynchronously. Failure ends P-2 without a sixth attempt or
semantic tag.
