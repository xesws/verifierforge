# v0.28.0 — Disposable RunPod live infrastructure

P2 pods are community on-demand, non-interruptible, one GPU, 80 GB container
disk, no network volume, public SSH only, using
`runpod/pytorch:1.0.2-cu1281-torch280-ubuntu2404`. The local public key is the
only create-time access material; `RUNPOD_API_KEY` stays in the local HTTP
Authorization header.

The exact pushed HEAD is delivered as a Git bundle over SSH, then cloned and
checked out remotely. S3 credentials use the existing stdin-to-tmux injection
path, are never written to a credential file, and are cleared on job exit.
Progress and collection use S3; SSH is limited to bootstrap, launch and bounded
diagnostics.

The live evidence records pod ID/name/GPU/hourly price, refreshed SSH endpoint,
state transitions, cost observations, delete time, billing-stop confirmation,
S3 object inventory and hashes. It never records credentials.
