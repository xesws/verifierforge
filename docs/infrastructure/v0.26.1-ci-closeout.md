# v0.26.1 — Security CI closeout

The first `security` push run failed inside `actions/setup-python@v5` before
install or tests: no default `requirements.txt` or `pyproject.toml` existed.
VerifierForge intentionally uses `requirements-app.txt`, so the workflow must
set `cache-dependency-path` to that exact file. This is a CI cache-discovery
fix; dependency versions and runtime behavior do not change.

The next run passed install and secret scanning, then failed with
`pytest: command not found` (exit 127). The repository previously relied on an
already-populated developer environment and had no dev dependency file.
`requirements-dev.txt` now includes the app requirements plus bounded pytest
and pytest-asyncio versions; CI installs that file while production continues
to install `requirements-app.txt` only.
