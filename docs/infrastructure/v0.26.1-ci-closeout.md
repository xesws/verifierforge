# v0.26.1 — Security CI closeout

The first `security` push run failed inside `actions/setup-python@v5` before
install or tests: no default `requirements.txt` or `pyproject.toml` existed.
VerifierForge intentionally uses `requirements-app.txt`, so the workflow must
set `cache-dependency-path` to that exact file. This is a CI cache-discovery
fix; dependency versions and runtime behavior do not change.
