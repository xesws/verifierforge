# v0.15.0 — L4 serving pod

`vfserve` is intentionally independent from the historical `runpod` trainer
host. It points to the empty serving-only L4 pod and uses a dedicated
`/workspace/vfserve-venv`. The only Python package installed there is
`vllm==0.10.2`, which brings its compatible torch runtime; no repository clone
or trainer dependencies belong on this pod.

The pod receives the selected step-350 standard-HF directory by rsync and must
match the M6 tree SHA-256 before serving. vLLM runs detached in tmux on port
8000 with bfloat16 and `--max-model-len 2048`. The public RunPod proxy URL,
not localhost, is the endpoint acceptance route. The secret comes from local
ignored `.env` and is passed to the process without being logged.

The endpoint is a disposable service executor. Its URL, command, GPU identity,
hourly price, and verification result are retained in `NOTES.md` and the run
sheet, not embedded in source code.
