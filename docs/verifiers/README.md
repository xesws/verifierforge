# Verifier Documentation

This area owns `core/verifier_base.py`, `core/rewards/`, tiered scoring, verifier tests, safety boundaries, and future Verifier Copilot output.

Before changing a verifier, create a versioned document with task inputs/outputs, rubric tiers, reward-hack cases, sandbox constraints, expected-result semantics, and unit-test cases. A reward or contract change must identify trainer and API consumers and update their area documents when affected.
