# v0.10.0 — verifier-bound split invariants

U1 changes difficulty selection, not `NL2SQLVerifier` behavior. The training
and held-out rows must independently score exactly `1.0` under verifier v2.
The split manifest records verifier identity, explicit version, source digest,
and Git blob identity so the frozen three-piece dataset cannot silently change
its reward definition.

Selection prefers an observed model count closest to two in `[1,4]`, then the
lowest count in `[1,6]`; only a seed with no eligible `[1,6]` row uses the
existing deterministic discard/backfill path. That distinction must be visible
per row in provenance.
