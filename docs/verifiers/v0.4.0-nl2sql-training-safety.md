# v0.4.0 — Verifiers: V1 Training Safety

## Scope

The existing NL2SQL rubric remains parse 0.2, execute 0.5, exact result 1.0,
with a 0.05 penalty above 400 characters. For model-generated SQL, execution is
limited to exactly one read-only `SELECT` or `WITH` statement; schema-changing,
write, pragma, attach, and multi-statement completions retain only parse credit.

## Fixture

Fifty hand-reviewable employee/department/project cases are committed as JSONL.
Stable case IDs, sorted order, and seed 42 produce 40 training and 10 validation
examples. Remote Parquet conversion is generated under ignored `runs/<job>/input`.

## Validation

Unit tests cover valid exact/executable/invalid cases plus rejected write and
multi-statement SQL. A human must review all generated seeds before GPU use.
