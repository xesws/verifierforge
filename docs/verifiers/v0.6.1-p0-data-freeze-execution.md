# v0.6.1 — verifier admission and frozen provenance

**Status:** in progress. Linked plan: `docs/versions/v0.6.x/v0.6.1-p0-data-freeze-execution.md`.

`NL2SQLVerifier` is unchanged and remains the sole admission authority. An augmentation variant, selected projection row, or fallback seed is eligible only when it scores exactly `1.0`; parse- or execution-only tiers are insufficient.

Projection rechecks candidates rather than trusting S1 output. The freeze manifest records the verifier class identity, source-file SHA-256, and Git blob identity along with hashes of the full and 50-row datasets. This freezes data and verifier behavior together without changing scoring semantics.
