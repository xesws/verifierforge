# v0.11.0 — Verifier boundary for the D4 spurious control

The frozen NL2SQL verifier v2 and its exact-result reward remain unchanged for
the 1.5B main job. No scoring tier, schema handling, expected result, or
training/held-out dataset is modified in D4.

The 0.5B control deliberately bypasses that verifier with a deterministic,
fixed-seed Bernoulli(0.5) reward. The implementation must make that choice
explicit in the job configuration and artifact metadata, so no one can mistake
control reward for verifier evidence. The draw is reproducible from the fixed
control seed and completion identity, and focused tests must prove that the
same input produces the same control reward while the main adapter retains its
existing score behavior.
