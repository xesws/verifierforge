# v0.6.0 — verifier-screened NL2SQL augmentation

**Status:** in progress. Linked plan: `docs/versions/v0.6.x/v0.6.0-data-freeze.md`.

## Scope

The existing `NL2SQLVerifier` remains the source of truth. Generated prompt
variants are admitted only when their proposed SQL receives a full result-set
match (`score == 1.0`) against the corresponding SQLite schema and expected
result. Parse-only or executable-but-wrong outputs are rejected.

## Boundaries

This release does not alter scoring tiers, expected-result comparison, schema
execution, sandbox policy, or verifier contracts. Candidate records retain the
prompt, schema, expected result, and accepted reference completion needed for
reproducible gate evaluation. SQL and prompts are treated as data, not executable
shell input.

## Validation

Focused tests cover an accepted full-match expansion plus parse, execution, and
wrong-result rejections. The frozen manifest hashes both candidate JSONL and the
verifier source revision so a future training job cannot silently drift.
