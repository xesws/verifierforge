# v0.7.1 — verifier taxonomy interpretation

The legacy verifier reports a fenced SQL completion as parsed (`0.2`) but not
as a single read-only statement, because the Markdown fence is its first
significant token. That raw `execution_error` remains correct for verifier
version 1 and remains preserved in the sample artifact.

For runbook routing only, v0.7.1 treats the narrow conjunction of a leading
Markdown fence and `not_single_read_only_statement` as a format/SQL-extraction
failure. It does not relabel SQLite errors, wrong result sets, raw prose, or
unfenced multi-statement SQL. The subsequent Branch A verifier change is a
separate versioned implementation and must be revalidated against all
candidates.
