# v0.8.0 — verifier v2 fenced-SQL extraction

`NL2SQLVerifier.VERSION` becomes `2`. The raw scoring algorithm remains the
legacy v1 algorithm: parse credit, lexical read-only gate, SQLite execution,
Counter result comparison, and the existing over-400-character penalty all
remain unchanged.

The v2 wrapper recognizes only a Markdown code block tagged `sql` (case
insensitive) or untagged. It removes the fence and uses `sqlparse.split` to
pass the first extracted statement to that unchanged scorer. An unfenced
completion remains raw, including multi-statement input, so the existing
single-statement and authorizer safety checks continue to reject it. Evidence
keeps the original completion and records the scored/extracted completion plus
the extraction kind when normalization occurred.
