# v0.7.0 — NL2SQL diagnostic score breakdown

**Status:** in progress. Linked plan:
`docs/versions/v0.7.x/v0.7.0-subset-sample-evidence.md`.

The verifier exposes an additive diagnostic breakdown whose final score is
identical to `NL2SQLVerifier.score()`: parse tier `0.2`, execution tier `0.5`,
result-match tier `1.0`, and the existing over-400-character penalty. No
extraction, normalization, new permission, or scoring relaxation is introduced
by this version.

For O1, `parse_failure` means the existing parser tier is zero. `execution_error`
means the parser tier was reached but execution was not. `executable_not_full`
means execution completed but the final score is not a full pass; its detail
records whether rows actually matched but the existing length penalty applied.
`D` is always parse failures divided by all samples whose final score is below
`1.0`.
