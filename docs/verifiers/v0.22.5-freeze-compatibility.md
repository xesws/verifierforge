# v0.22.5 — Legacy freeze command compatibility

The untracked `freeze_nl2sql.py` implementation predates the authoritative U3
three-piece freeze. Its lexicographic selection rule, missing held-out set, and
missing pass@8 ceiling make it unsafe to publish as a second freezer.

The filename is retained as a deprecated compatibility entrypoint that delegates
argument parsing and publication to `scripts.freeze_three_piece`. Tests protect
that delegation and prove the obsolete rule is absent. The frozen datasets,
manifest, verifier v2, and `v0.10.2-p0-three-piece-freeze` tag are unchanged.
